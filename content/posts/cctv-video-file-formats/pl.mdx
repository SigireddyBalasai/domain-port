Wybór odpowiedniego formatu pliku dla nagrań CCTV wpływa na jakość wideo, wymagania dotyczące przechowywania, kompatybilność odtwarzania i przydatność kryminalistyczną. Ten przewodnik obejmuje główne formaty kontenerów i kodeki używane w systemach nadzoru.

[Format kontenera](https://en.wikipedia.org/wiki/Container_format) (taki jak MP4 lub AVI) łączy wideo, audio i metadane. [Kodek](https://en.wikipedia.org/wiki/Video_codec) (taki jak H.264 lub H.265) określa, jak wideo jest kompresowane wewnątrz tego kontenera. Nowoczesne systemy CCTV zazwyczaj używają wideo H.264 lub H.265 w kontenerze MP4.

[AVI](https://en.wikipedia.org/wiki/Audio_Video_Interleave) został wprowadzony przez Microsoft w 1992 roku. Przechowuje dane przy minimalnej kompresji, tworząc duże pliki o wysokiej jakości.

**Najlepsze do:** Krótkoterminowych klipów kryminalistycznych, gdzie szczegóły są ważniejsze niż miejsce.

| Zalety                                       | Wady                        |
| -------------------------------------------- | --------------------------- |
| Wysoka jakość, szerokie wsparcie odtwarzania | Bardzo duże rozmiary plików |
| Brak złożonego kodowania                     | Słaba wydajność kompresji   |

[H.264](https://en.wikipedia.org/wiki/H.264/MPEG-4_AVC) jest najczęściej używanym standardem kompresji wideo w CCTV. Może zmniejszyć rozmiar pliku nawet o 50% w porównaniu do MJPEG, zachowując porównywalną jakość.

**Najlepsze do:** Nagrywania HD i ciągłego nadzoru 24/7 na istniejącym sprzęcie.

| Zalety                                    | Wady                                               |
| ----------------------------------------- | -------------------------------------------------- |
| Szerokie wsparcie dekoderów sprzętowych   | Ograniczenia licencyjne (większość OEM pokrywa je) |
| Dojrzały, dobrze przetestowany w nadzorze | Wypierany przez H.265                              |

**Najlepsze do:** Zastosowań wymagających niskiego opóźnienia kodowania lub łatwego wyodrębniania klatek.

Wybór kodeka bezpośrednio wpływa na planowanie przechowywania. System 4-kanałowy 1080p nagrywający 24/7 wymaga około:

| Kodek | Dzienne przechowywanie (4× 1080p @ 15 fps) |
| ----- | ------------------------------------------ |
| MJPEG | ~300 GB                                    |
| H.264 | ~80 GB                                     |
| H.265 | ~40 GB                                     |
| RAW   | ~2+ TB                                     |

Zapoznaj się z [implementacją referencyjną H.264](https://en.wikipedia.org/wiki/X264) lub [projektem x265](https://en.wikipedia.org/wiki/X265) aby uzyskać więcej informacji technicznych.

选择合适的 CCTV 录像文件格式会影响视频质量、存储需求、播放兼容性和取证可用性。本指南涵盖监控系统中使用的主要容器格式和编解码器。

[容器格式](https://en.wikipedia.org/wiki/Container_format)（如 MP4 或 AVI）将视频、音频和元数据封装在一起。[编解码器](https://en.wikipedia.org/wiki/Video_codec)（如 H.264 或 H.265）决定视频在该容器内的压缩方式。现代 CCTV 系统通常使用 MP4 容器中的 H.264 或 H.265 视频。

[AVI](https://en.wikipedia.org/wiki/Audio_Video_Interleave) 由微软于 1992 年推出。它以最小压缩存储数据，生成大文件但质量高。

**最适合：** 画质比存储更重要的短期取证剪辑。

| 优点                   | 缺点           |
| ---------------------- | -------------- |
| 高质量，广泛的播放支持 | 文件体积非常大 |
| 无需复杂编码           | 压缩效率差     |

[H.264](https://en.wikipedia.org/wiki/H.264/MPEG-4_AVC) 是 CCTV 中部署最广泛的视频压缩标准。与 MJPEG 相比，它可在保持相同画质的同时将文件大小减少多达 50%。

**最适合：** 在现有硬件上进行高清录像和 24/7 持续监控。

| 优点                       | 缺点                              |
| -------------------------- | --------------------------------- |
| 广泛的硬件解码器支持       | 受许可限制（但大多数 OEM 已覆盖） |
| 成熟，在监控中经过充分测试 | 正被 H.265 取代                   |

[MJPEG](https://en.wikipedia.org/wiki/Motion_JPEG) 将每一帧编码为独立的 JPEG 图像。这使得逐帧分析变得简单，但由于没有帧间压缩，文件体积很大。

**最适合：** 需要低编码延迟或易于提取单帧的应用。

| 优点                                 | 缺点                        |
| ------------------------------------ | --------------------------- |
| 每帧都是完整图像（易于提取静止图像） | 相比 H.264/H.265 压缩效率差 |
| 低编码延迟                           | 持续录像的文件体积大        |

| H.264    | ~80 GB                          |
| H.265    | ~40 GB                          |
| RAW      | ~2+ TB                          |

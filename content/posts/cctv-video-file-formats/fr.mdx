Le choix du bon format de fichier pour les images de vidéosurveillance affecte la qualité vidéo, les besoins de stockage, la compatibilité de lecture et l'utilisabilité médico-légale. Ce guide couvre les principaux formats de conteneur et codecs utilisés dans les systèmes de surveillance.

Un [format conteneur](https://en.wikipedia.org/wiki/Container_format) (comme MP4 ou AVI) regroupe la vidéo, l'audio et les métadonnées ensemble. Un [codec](https://en.wikipedia.org/wiki/Video_codec) (comme H.264 ou H.265) détermine comment la vidéo est compressée à l'intérieur de ce conteneur. Les systèmes CCTV modernes utilisent généralement de la vidéo H.264 ou H.265 dans un conteneur MP4.

[AVI](https://en.wikipedia.org/wiki/Audio_Video_Interleave) a été introduit par Microsoft en 1992. Il stocke les données avec une compression minimale, produisant des fichiers volumineux de haute qualité.

**Meilleur pour :** Clips médico-légaux à court terme où le détail prime sur le stockage.

| Avantages                               | Inconvénients                    |
| --------------------------------------- | -------------------------------- |
| Haute qualité, large support de lecture | Très grandes tailles de fichiers |
| Aucun encodage complexe nécessaire      | Faible efficacité de compression |

**Meilleur pour :** Enregistrement à long terme, visualisation à distance et flux de travail multiplateformes.

| Avantages                             | Inconvénients                               |
| ------------------------------------- | ------------------------------------------- |
| Excellent rapport compression/qualité | Encodage légèrement plus complexe que l'AVI |

[H.264](https://en.wikipedia.org/wiki/H.264/MPEG-4_AVC) est la norme de compression vidéo la plus déployée en CCTV. Il peut réduire la taille des fichiers jusqu'à 50% par rapport au MJPEG tout en maintenant une qualité comparable.

**Meilleur pour :** Enregistrement HD et surveillance continue 24/7 sur du matériel existant.

| Avantages                             | Inconvénients                                                     |
| ------------------------------------- | ----------------------------------------------------------------- |
| Large support des décodeurs matériels | Contraintes de licence (bien que la plupart des OEM les couvrent) |
| Mature, bien testé en surveillance    | Remplacé par H.265                                                |

| Idéal pour les résolutions 4K+                      | Support limité dans les NVR plus anciens  |

[MJPEG](https://en.wikipedia.org/wiki/Motion_JPEG) encode chaque image comme une image JPEG indépendante. Cela rend l'analyse image par image simple mais produit de gros fichiers car aucune compression inter-image n'est utilisée.

**Meilleur pour :** Applications nécessitant une faible latence d'encodage ou une extraction facile d'images.

| Avantages                                              | Inconvénients                                              |
| ------------------------------------------------------ | ---------------------------------------------------------- |
| Chaque image est une image complète (captures faciles) | Mauvaise compression comparée à H.264/H.265                |
| Faible latence d'encodage                              | Grandes tailles de fichiers pour un enregistrement continu |

| Avantages                     | Inconvénients                               |
| ----------------------------- | ------------------------------------------- |

Le choix du codec affecte directement la planification du stockage. Un système 4 canaux 1080p enregistrant 24/7 nécessite environ :

| Codec | Stockage quotidien (4× 1080p @ 15 ips) |
| ----- | -------------------------------------- |
| MJPEG | ~300 Go                                |
| H.264 | ~80 Go                                 |
| H.265 | ~40 Go                                 |
| RAW   | ~2+ To                                 |

Reportez-vous à [l'implémentation de référence H.264](https://en.wikipedia.org/wiki/X264) ou au [projet x265](https://en.wikipedia.org/wiki/X265) pour plus de détails techniques sur ces codecs.

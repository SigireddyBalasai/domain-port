Escolher o formato de arquivo correto para imagens CCTV afeta a qualidade do vídeo, requisitos de armazenamento, compatibilidade de reprodução e usabilidade forense. Este guia cobre os principais formatos de contêiner e codecs usados em sistemas de vigilância.

Um [formato contêiner](https://en.wikipedia.org/wiki/Container_format) (como MP4 ou AVI) agrupa vídeo, áudio e metadados juntos. Um [codec](https://en.wikipedia.org/wiki/Video_codec) (como H.264 ou H.265) determina como o vídeo é comprimido dentro desse contêiner. Sistemas CCTV modernos normalmente usam vídeo H.264 ou H.265 dentro de um contêiner MP4.

[AVI](https://en.wikipedia.org/wiki/Audio_Video_Interleave) foi introduzido pela Microsoft em 1992. Armazena dados com compressão mínima, produzindo arquivos grandes com alta qualidade.

**Melhor para:** Clipes forenses de curto prazo onde o detalhe é mais importante que o armazenamento.

| Prós                                        | Contras                           |
| ------------------------------------------- | --------------------------------- |
| Alta qualidade, amplo suporte de reprodução | Tamanhos de arquivo muito grandes |
| Nenhuma codificação complexa necessária     | Eficiência de compressão ruim     |

[H.264](https://en.wikipedia.org/wiki/H.264/MPEG-4_AVC) é o padrão de compressão de vídeo mais amplamente implantado em CCTV. Pode reduzir o tamanho do arquivo em até 50% em comparação com MJPEG, mantendo qualidade comparável.

**Melhor para:** Gravação HD e vigilância contínua 24/7 em hardware existente.

| Prós                                         | Contras                                                  |
| -------------------------------------------- | -------------------------------------------------------- |
| Amplo suporte de decodificadores de hardware | Com licenciamento (embora a maioria dos OEMs cubra isso) |
| Maduro, bem testado em vigilância            | Sendo substituído por H.265                              |

[MJPEG](https://en.wikipedia.org/wiki/Motion_JPEG) codifica cada quadro como uma imagem JPEG independente. Isso torna a análise quadro a quadro simples, mas produz arquivos grandes, pois nenhuma compressão entre quadros é usada.

**Melhor para:** Aplicações que exigem baixa latência de codificação ou extração fácil de quadros.

| Prós                                                                 | Contras                                            |
| -------------------------------------------------------------------- | -------------------------------------------------- |
| Cada quadro é uma imagem completa (fácil capturar imagens estáticas) | Compressão ruim comparada a H.264/H.265            |
| Baixa latência de codificação                                        | Tamanhos de arquivo grandes para gravação contínua |

**Melhor para:** Reprodução contínua dentro do ecossistema do fabricante.

A escolha do codec afeta diretamente o planejamento de armazenamento. Um sistema de 4 canais 1080p gravando 24/7 requer aproximadamente:

| Codec | Armazenamento diário (4× 1080p @ 15 fps) |
| ----- | ---------------------------------------- |
| MJPEG | ~300 GB                                  |
| H.264 | ~80 GB                                   |
| H.265 | ~40 GB                                   |
| RAW   | ~2+ TB                                   |

Consulte a [implementação de referência H.264](https://en.wikipedia.org/wiki/X264) ou o [projeto x265](https://en.wikipedia.org/wiki/X265) para aprofundamento técnico sobre esses codecs.

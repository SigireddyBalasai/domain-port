选择正确的摄像头分辨率是规划监控系统时最重要的决策之一。3MP、5MP和8MP（4K）是2026年最常见的选项，但更高的百万像素并不自动意味着更好的安全录像。

本指南比较了三个分辨率级别的每个相关类别：像素数量、识别距离、存储需求、夜视适用性和每台摄像头的成本。

<Callout type="info" title="关键点 upfront">

**没有普遍最佳的分辨率。** 8MP提供最多的细节，但需要更多的存储和光线。5MP在细节和存储使用之间提供了最佳的折衷。3MP对于大多数室内空间完全足够，并且最具成本效益。

</Callout>

## 分辨率对比

| 规格                 | 3MP         | 5MP         | 8MP (4K)    |
| -------------------- | ----------- | ----------- | ----------- |
| 分辨率               | 2304 × 1296 | 2560 × 1920 | 3840 × 2160 |
| 像素（总计）         | ~300万      | ~500万      | ~830万      |
| 宽高比               | 16:9        | 4:3         | 16:9        |
| 每英尺像素（12米处） | 48          | 64          | 96          |
| 识别距离 (DORI)      | 6–9 米      | 9–15 米     | 15–25 米    |
| 存储/天 (H.265)      | 8–12 GB     | 25–35 GB    | 30–50 GB    |
| 相对成本             | 1×          | 1.3–1.5×    | 1.5–2.5×    |

### 3MP (2304 × 1296) — 坚实的基准

3MP是1080p（2MP）的继任者，提供约多50%的像素。此分辨率非常适合：

- **室内空间** — 办公室、走廊、仓库
- **短距离监控** 可达6–9米
- **预算有限的安装** 使用许多摄像头
- **带宽有限的Wi-Fi摄像头**

优点：低存储使用，得益于更大的像素的良好夜视能力，低成本。
缺点：有限的变焦能力，较短的识别距离。

### 5MP (2560 × 1920) — 全能型

5MP使用4:3宽高比，提供比16:9更多的垂直覆盖。这使得5MP特别适合：

- **户外区域** — 入口、停车场、车道
- **走廊和过道** — 垂直覆盖重要的地方
- **中等识别距离区域**（9–15米）
- **光线良好的场景**

优点：出色的4:3格式用于垂直场景，良好的细节，合理的存储使用。
缺点：相同传感器尺寸下夜视能力比3MP弱；没有用于宽屏显示器的16:9。

### 8MP (3840 × 2160 / 4K) — 旗舰级

4K（8MP）提供最高的细节密度，并允许在较低分辨率下会模糊的数字变焦。非常适合：

- **宽阔的户外区域** — 大型停车场、堆场、入口区域
- **远距离车牌和面部识别**（15–25米）
- **法医细节捕捉** — 事后放大图像
- **必须无间隙覆盖的区域**

优点：最大细节，最佳数字变焦，面向未来。
缺点：最高的存储使用，小传感器上夜视能力较弱，成本更高。

<Callout type="warning" title="注意4K夜视">

许多廉价的4K摄像头使用小传感器（1/3"或1/2.7"），这会在黄昏时造成严重的图像噪点。配备1/1.8"传感器的5MP摄像头通常在弱光下提供更清晰的图像。始终检查传感器尺寸，而不仅仅是百万像素数！

</Callout>

## 存储消耗详情

存储使用通常是选择分辨率时的决定因素：

| 场景                | 3MP        | 5MP         | 8MP         |
| ------------------- | ---------- | ----------- | ----------- |
| 连续 H.265          | 8–12 GB/天 | 25–35 GB/天 | 30–50 GB/天 |
| 连续 H.265+         | 5–8 GB/天  | 15–22 GB/天 | 20–35 GB/天 |
| 运动 H.265          | 2–4 GB/天  | 7–12 GB/天  | 10–18 GB/天 |
| 8摄像头系统（30天） | 0.7–1.1 TB | 2.1–3.0 TB  | 2.5–4.2 TB  |

<Callout type="tip" title="使用H.265+节省存储">

H.265+（也称为Smart H.265+）分析场景并仅以全分辨率编码运动区域。静态背景以降低的比特率存储。这样可以节省30-50%的存储空间，且没有可见的质量损失。在您的NVR或录像机上启用H.265+ — 大多数制造商默认提供此功能。

</Callout>

## 夜视和感光度

传感器尺寸通常比百万像素数对夜视性能更重要：

| 传感器         | 3MP  | 5MP    | 8MP    |
| -------------- | ---- | ------ | ------ |
| 1/3"（标准）   | 好   | 可接受 | 弱     |
| 1/2.8"         | 很好 | 好     | 可接受 |
| 1/1.8"（星光） | 优秀 | 很好   | 好     |

经验法则：**每提高一个分辨率级别，您需要大约提高一个传感器尺寸级别**才能获得相同的夜视性能。配备1/3"传感器的3MP摄像头的夜视效果与配备1/2.8"传感器的5MP摄像头大致相同。

## 按地点推荐

| 地点              | 推荐分辨率 | 原因                         |
| ----------------- | ---------- | ---------------------------- |
| 房屋入口 / 门     | 5MP 或 8MP | 短距离面部识别               |
| 车道 / 停车场     | 5MP 或 8MP | 10–20米车牌识别              |
| 客厅 / 办公室     | 3MP        | 完全足够，存储低             |
| 走廊 / 过道       | 5MP (4:3)  | 最佳垂直覆盖                 |
| 仓库 / 车间       | 3MP 或 5MP | 概览，大像素以获得更好的夜视 |
| 大型停车场 / 院子 | 8MP (4K)   | 最大细节覆盖                 |

<Callout type="note" title="实用建议">

对于大多数家庭，3MP室内摄像头和5MP户外摄像头的组合完全足够。8MP（4K）在您需要在15米以上识别面部或车牌时才有意义 — 或者当存储预算不是问题时。

</Callout>

## 结论

3MP、5MP和8MP各有其用途。根据DORI原则规划您的监控系统：**根据您需要的识别距离选择分辨率**，而不是根据可用的最高数字。

- **3MP** — 室内空间，短距离，经济实惠
- **5MP** — 户外区域，全能型，垂直覆盖
- **8MP (4K)** — 广阔区域，最大细节，高存储需求

最好的监控系统通常在这三个分辨率中各取其长。

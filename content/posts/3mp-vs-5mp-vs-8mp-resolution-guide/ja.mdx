監視システムを計画する際、適切なカメラ解像度を選ぶことは最も重要な決定の1つです。3MP、5MP、8MP（4K）は2026年で最も一般的なオプションですが、メガピクセルが多いほど必ずしも優れたセキュリティ映像になるとは限りません。

このガイドでは、3つの解像度階層をすべての関連カテゴリで比較します：画素数、識別距離、保存需要、ナイトビジョン適性、カメラあたりのコスト。

<Callout type="info" title="最初に結論">

**普遍的に最良の解像度はありません。**
8MPは最大の詳細を提供しますが、より多くの保存容量と光を必要とします。5MPは詳細と保存使用の最良の妥協点を提供します。3MPはほとんどの屋内スペースに十分で、最もコスト効率が高いです。

</Callout>

## 解像度の比較

| 仕様                               | 3MP         | 5MP         | 8MP（4K）   |
| ---------------------------------- | ----------- | ----------- | ----------- |
| 解像度                             | 2304 × 1296 | 2560 × 1920 | 3840 × 2160 |
| 画素数（合計）                     | 約300万     | 約500万     | 約830万     |
| アスペクト比                       | 16:9        | 4:3         | 16:9        |
| 1フィートあたりのピクセル（12m時） | 48          | 64          | 96          |
| 識別距離（DORI）                   | 6〜9 m      | 9〜15 m     | 15〜25 m    |
| 保存量/日（H.265）                 | 8〜12 GB    | 25〜35 GB   | 30〜50 GB   |
| 相対コスト                         | 1×          | 1.3〜1.5×   | 1.5〜2.5×   |

### 3MP（2304 × 1296）— 確かなベースライン

3MPは1080p（2MP）の後継であり、約50%多くのピクセルを提供します。この解像度は以下に最適です：

- **屋内スペース** — オフィス、廊下、倉庫
- **近距離監視** 最大6〜9メートル
- **予算重視の設置** 多数のカメラを使用
- **帯域幅が限られたWi-Fiカメラ**

利点：保存量が少ない、大きなピクセルによる良好なナイトビジョン、低コスト。
欠点：ズーム能力が限定的、識別距離が短い。

### 5MP（2560 × 1920）— 万能型

5MPは4:3のアスペクト比を使用し、16:9より多くの垂直カバレッジを提供します。これにより5MPは特に以下に適しています：

- **屋外エリア** — 入口、駐車場、私道
- **廊下や通路** — 垂直カバレッジが重要な場所
- **中程度の識別距離のエリア**（9〜15メートル）
- **明るいシーン**

利点：垂直シーンに優れた4:3フォーマット、良好な詳細、適度な保存使用。
欠点：同じセンサーサイズでは3MPよりナイトビジョンが弱い；ワイドスクリーンモニターには16:9非対応。

### 8MP（3840 × 2160 / 4K）— フラッグシップ

4K（8MP）は最高の詳細密度を提供し、低解像度ではぼやけるデジタルズームを可能にします。以下に最適：

- **広い屋外エリア** — 大規模駐車場、保管ヤード、入口ゾーン
- **遠距離でのナンバープレートおよび顔認識**（15〜25メートル）
- **フォレンジック詳細キャプチャ** — 事後的な画像へのズーム
- **隙間なくカバーする必要があるエリア**

利点：最大の詳細、最良のデジタルズーム、将来性。
欠点：最大の保存使用、小型センサーでの弱いナイトビジョン、高コスト。

<Callout type="warning" title="4Kナイトビジョンに注意">

多くの安価な4Kカメラは小型センサー（1/3型または1/2.7型）を使用しており、夕暮れ時に大きな画質ノイズを引き起こします。1/1.8型センサー搭載の5MPカメラは、低照度でより鮮明な画像を提供することがよくあります。メガピクセル数だけでなく、必ずセンサーサイズを確認してください！

</Callout>

## 保存消費量の詳細

保存量は解像度選択の決定的要因となることがよくあります：

| シナリオ              | 3MP         | 5MP          | 8MP          |
| --------------------- | ----------- | ------------ | ------------ |
| 連続H.265             | 8〜12 GB/日 | 25〜35 GB/日 | 30〜50 GB/日 |
| 連続H.265+            | 5〜8 GB/日  | 15〜22 GB/日 | 20〜35 GB/日 |
| 動体検知H.265         | 2〜4 GB/日  | 7〜12 GB/日  | 10〜18 GB/日 |
| 8台システム（30日間） | 0.7〜1.1 TB | 2.1〜3.0 TB  | 2.5〜4.2 TB  |

<Callout type="tip" title="H.265+で保存容量を節約">

H.265+（Smart
H.265+とも呼ばれる）はシーンを分析し、動いているエリアのみをフル解像度でエンコードします。静止背景は低ビットレートで保存されます。これにより、目に見える品質低下なしに保存容量を30〜50%節約できます。NVRまたはレコーダーでH.265+を有効にしてください—この機能はほとんどのメーカーでデフォルトで利用可能です。

</Callout>

## ナイトビジョンと感度

ナイトビジョン性能には、メガピクセル数よりもセンサーサイズが重要であることがよくあります：

| センサー                | 3MP        | 5MP        | 8MP  |
| ----------------------- | ---------- | ---------- | ---- |
| 1/3型（標準）           | 良好       | 普通       | 弱い |
| 1/2.8型                 | 非常に良好 | 良好       | 普通 |
| 1/1.8型（スターライト） | 優れている | 非常に良好 | 良好 |

経験則：**解像度を1段階上げるごとに、同じナイトビジョン性能を達成するにはセンサーサイズも約1段階大きくする必要があります**。1/3型センサーの3MPカメラは、1/2.8型センサーの5MPカメラと夜間の視認性がほぼ同等です。

## 場所別の推奨

| 場所                      | 推奨解像度 | 理由                                           |
| ------------------------- | ---------- | ---------------------------------------------- |
| 住宅玄関 / ドア           | 5MP or 8MP | 近距離での顔認識                               |
| 私道 / 駐車場             | 5MP or 8MP | 10〜20 mでのプレート                           |
| リビングルーム / オフィス | 3MP        | 十分、保存量が少ない                           |
| 廊下 / 通路               | 5MP（4:3） | 最適な垂直カバレッジ                           |
| 倉庫 / 作業場             | 3MP or 5MP | 概要監視、大きなピクセルで良好なナイトビジョン |
| 大規模駐車場 / 庭         | 8MP（4K）  | 最大詳細カバレッジ                             |

<Callout type="note" title="実用的な推奨">

ほとんどの家庭では、屋内カメラに3MP、屋外カメラに5MPを混在させることで十分です。8MP（4K）は、15メートルを超える距離で顔やプレートを識別する必要がある場合、または保存予算が問題にならない場合に意味があります。

</Callout>

## 結論

3MP、5MP、8MPにはそれぞれの役割があります。DORI原則に基づいて監視システムを計画してください：**利用可能な最高の数値ではなく、必要な識別距離に基づいて解像度を選択してください**。

- **3MP** — 屋内スペース、近距離、予算に優しい
- **5MP** — 屋外エリア、万能型、垂直カバレッジ
- **8MP（4K）** — 広いエリア、最大詳細、高保存需要

最良の監視システムは、多くの場合、各解像度が最も効果を発揮する場所で3つすべてを使用します。

## 平台深度解析

### Amazon Alexa

**原生支持:** Ring、Blink、Arlo、Echo Show (作为显示屏)  
**需 Skill:** Reolink、Eufy、UniFi、Wyze、Tapo、Amcrest、TP-Link Kasa

<Accordion type="single" collapsible>
  <AccordionItem value="alexa-features">
    <AccordionTrigger>Alexa 能用摄像头做什么</AccordionTrigger>
    <AccordionContent>

      **实时查看:** "Alexa, show [camera name]" → Echo Show、Fire TV、Alexa App
      **广播通知:** "Alexa, announce [camera] detected a person" → 所有 Echo
      设备 **常规 (自动化):** - 触发器: 运动 / 人 / 包裹 / 门铃按下 - 动作:
      灯开/关、锁开/关、说自定义短语、播放声音、等待、通知 **双向对讲:** "Alexa,
      talk to [camera]" (半双工 PTT) **录制:** Ring/Blink/Arlo → 云端
      (订阅制)。其他 → **无云端录制**。 **限制:** 无本地流 (全云中转); Echo Show
        10 分钟超时; 无 4K (最高 1080p 转码); 常规限 1 触发 + 1 动作
      (用多个常规)。

    </AccordionContent>

  </AccordionItem>
  <AccordionItem value="alexa-setup">
    <AccordionTrigger>设置技巧</AccordionTrigger>
    <AccordionContent>

        1. **启用 Skill** → 关联账号 → 发现设备 2. **重命名** Alexa App
      中的摄像头为自然语言 ("front door", "driveway cam") 3. **创建组**
      ("Outside" = front + back + driveway) 以便 "show outside" 4. **常规示例:**
      - **门廊灯:** 前门动作日落 30 分钟后 → 开门廊灯 (5 分钟) - **威慑:**
      车道检测到人 → Echo 宣播 "Security camera recording" + 投光灯开 -
      **欢迎:** 门铃按下 → Kitchen Echo 播报 "Someone's at the front door" -
      **地理围栏:** "When I leave" → 布防摄像头 (若 Skill 支持)


    </AccordionContent>

  </AccordionItem>
</Accordion>

### Google Home (Google Assistant)

**原生支持:** Nest、Arlo、Blink、Ring (有限)  
**需 Action:** Eufy、Reolink、UniFi、Wyze、Tapo、Amcrest

      **广播:** "Hey Google, broadcast [camera] sees a person" → 所有 Nest 音箱

      动作、人、车辆、动物、包裹、门铃、声音 - 动作:
      灯、插座、锁、恒温器、音箱、电视、脚本 **双向对讲:** "Hey Google, talk to
      [camera]" (仅 Nest; 其他经 App) **录制:** Nest Aware (云端)。其他 →
      **无云端录制**。 **优于 Alexa:** Nest Hub Max 多摄像头网格视图更好;
      运动类型更细; 内置 Home/Away 常规; 脚本编辑器支持复杂逻辑。

    </AccordionContent>

      **自动化示例:** - **夜灯:** 人在前门日落后 → 走廊灯 50% (5 分钟) -
      **包裹提醒:** 检测到包裹 → 所有音箱播报 "Package delivered" + 记录到
      Google Sheet - **离家:** 手机位置离开 → 布防摄像头 (若 Action 支持) + 锁门
      - **宠物提醒:** 院子检测到动物 → Kitchen 音箱播报 "Pet in yard"

### Apple HomeKit (HomeKit Secure Video)

**原生支持:** Eufy (HomeBase 3)、Arlo (Pro 3/4/5S, Ultra)、Logitech Circle View、Netatmo、Eve、Aqara、TP-Link Tapo (Matter)、UniFi Protect (3.0+)、Onvis、Wemo  
**需桥接:** Reolink (经 Homebridge/Scrypted)、Ring (经 Homebridge)、Wyze (经 Homebridge)、Dahua/Hikvision (经 Homebridge/Scrypted)

<Callout type="info">

**HomeKit Secure Video (HSV) = 隐私最佳。** 视频 **设备端分析** (Home Hub:
Apple TV 4K、HomePod、iPad)。**端到端加密** — Apple 无法查看。10 天滚动存储
**含在 iCloud+** (50GB = 1 摄像头, 200GB = 5 摄像头, 2TB = 无限)。无额外订阅。

</Callout>

<Accordion type="single" collapsible>
  <AccordionItem value="hsv-features">
    <AccordionTrigger>HSV 功能与要求</AccordionTrigger>
    <AccordionContent>

**支持分辨率:** 1080p, 2K, 4K (分析降为 1080p, 存源分辨率)
**活动区域:** 每摄像头, Home App 设置
**人脸识别:** "检测到人" → "检测到 John" (若在 People 相册)
**录制选项:** 关 / 检测活动 / 流式和录制
**存储:** iCloud+ (50GB/$0.99, 200GB/$2.99, 2TB/$9.99) — **非每摄像头**
**Home Hub 必需:** Apple TV 4K (2021+), HomePod (mini/2代), iPad (常开)
**本地流:** 同一 Wi-Fi 的 iPhone/iPad → 直连 (无互联网)。远程 → 经 iCloud 中转 (加密)。
**限制:** 200GB 套餐最多 5 摄像头 (除非 2TB); 无 24/7 连续 (仅事件); 无 Web 视图 (仅 iOS/macOS); 无原生 Android。

    </AccordionContent>

  </AccordionItem>
  <AccordionItem value="homebridge">
    <AccordionTrigger>桥接非 HomeKit 摄像头 (Homebridge / Scrypted / Hoobs)</AccordionTrigger>
    <AccordionContent>

**Homebridge** (Node.js, 2000+ 插件): 跑在 Pi、Docker、NAS、迷你主机

- `homebridge-camera-ffmpeg` → RTSP → HSV (转码, 加动作)
- `homebridge-reolink` → Reolink 原生 API (动作、PTZ、快照)
- `homebridge-dahua` / `homebridge-hikvision` → ONVIF/ISAPI
- `homebridge-ring` → Ring 摄像头/门铃 (轮询, ~30s 延迟)

**Scrypted** (TypeScript, 更新更快): 原生 HomeKit、Google、Alexa 插件

- 更好 RTSP 处理、更低延迟、插件市场
- 跑在同硬件上如 Homebridge

**桥接硬件:** Raspberry Pi 4 (4GB+) / 5, Intel N100/N305 迷你主机 ($120), 旧笔记本, Synology/QNAP Docker。
**延迟:** Homebridge FFmpeg ~2–5s; Scrypted ~1–3s; 原生 HSV ~0.5–1s。
**经桥接 HSV:** 摄像头须支持 RTSP。桥接转码 → HSV → iCloud。 **计入 HSV 摄像头上限。**

</AccordionContent>

  </AccordionItem>
</Accordion>

### Matter (新标准)

**Matter 1.3 (2024)** 增加 **Camera** 设备类型。**Matter 1.4 (2025)** 增加 **Doorbell** 类型。

<Accordion type="single" collapsible>
  <AccordionItem value="matter-status">
    <AccordionTrigger>当前 Matter 摄像头支持 (2026)</AccordionTrigger>
    <AccordionContent>

**已认证 Matter 摄像头:**

- **TP-Link Tapo C120 / C220 / C320WS** (Matter over Wi-Fi)
- **Eve **Nanoleaf Smart Cam\*\* (Matter)
- **Eve Cam** (Thread + Matter)
- **Google Nest Cam (wired, 2024)** / **Nest Doorbell (wired, 2024)** — 经 Google Home Matter 桥接
- **EufyCam 3 / SoloCam S340** — Matter 固件更新 (2024)

**Matter 给您带来:**

- **多管理员:** 同时加入 Apple Home、Google Home、Alexa、SmartThings
- **本地控制:** 基础开/关/流无云中转 (Thread/Wi-Fi 直连)
- **标准化命令:** 实时流、动作事件、隐私模式

**Matter 目前不做:**

- 4K 流 (规格最高 1080p)
- PTZ 控制 (1.3/1.4 无)
- 人脸识别 / 包裹检测 (仅厂商扩展)
- 连续录制 (仅事件片段)
- NVR 集成

**结论 (2026):** Matter 摄像头适合 **基础多平台实时查看 + 动作提醒**。录制、AI、PTZ、4K 仍需原生集成或桥接。

## 本地控制 / 无云: 进阶路径

<Callout type="tip">

**本地 = 快、私、离线工作。** RTSP/ONVIF/WebSocket → Frigate/Scrypted/Blue
Iris → Home Assistant → 面板/自动化。无厂商云见过您的视频。

</Callout>

### 完全本地摄像头栈架构

```
Cameras (PoE/RTSP)
    │
    ▼
┌─────────────────────────────────────┐
│  Frigate (Docker)                   │
│  - RTSP ingest                      │
│  - Motion detection (CPU)           │
│  - AI object detection (Coral TPU)  │
│  - Recording (MP4, segmented)       │
│  - MQTT events (motion, person, etc)│
│  - WebRTC live view (sub-second)    │
└─────────────────────────────────────┘
    │                    │
    ▼                    ▼
┌─────────────┐    ┌─────────────┐
│ Home Asst.  │    │  Scrypted   │
│ (Automation)│    │ (HomeKit/   │
│  - Lights   │    │  Google/    │
│  - Locks    │    │  Alexa)     │
│  - Alerts   │    │             │
└─────────────┘    └─────────────┘
    │                    │
    ▼                    ▼
Dashboard           Apple Home /
                    Google Home /
                    Alexa App
```

### 本地栈硬件

| 组件          | 预算            | 推荐                                   | 高端                           |
| ------------- | --------------- | -------------------------------------- | ------------------------------ |
| **服务器**    | RPi 4 4GB ($80) | Intel N100/N305 迷你主机 ($130)        | i5-13400 / Ryzen 5700G ($400)  |
| **AI 加速器** | 仅 CPU (慢)     | **Google Coral USB ($60)** / M.2 ($40) | Coral PCIe x4 / Hailo-8 ($150) |
| **存储**      | 1× 4TB HDD      | 2× 8TB RAID 1                          | 4× 14TB RAID 6 / ZFS           |
| **网络**      | 1 Gbps          | 2.5 Gbps 交换机 + 网卡                 | 10 Gbps 主干                   |

### 本地栈摄像头要求

- **RTSP 支持** (标准: `rtsp://user:pass@ip:554/stream1`)
- **ONVIF Profile S/T/G** (发现、PTZ、事件)
- **子流** (D1/720p 给 AI, 4K 给录制)
- **无云依赖** 获取流 (部分摄像头需云鉴权令牌 — 避开)

**最佳 ONVIF/RTSP 摄像头 (2026):**

- Reolink (所有 PoE, 大多 Wi-Fi): `rtsp://ip:554/h264Preview_01_main`
- Dahua / Hikvision / Hanwha / Axis / Uniview / Vivotek / Milesight
- UniFi G4/G5 (RTSP 经 Protect 设置)
- Amcrest / Lorex / Annke / Montavue (OEM Dahua/Hik)
- **避开:** Ring、Blink、Nest、Arlo (2024 前)、Wyze (受限)、Eufy (电池款无 RTSP)

## 自动化配方 (可直接复制)

### Home Assistant (YAML)

```yaml
# 日落后人在前门亮门廊灯
automation:
  - alias: "Porch Light - Person at Front Door"
    trigger:
      - platform: mqtt
        topic: frigate/front_door/person
        payload: "on"
    condition:
      - condition: sun
        after: sunset
        after_offset: "-00:30:00"
    action:
      - service: light.turn_on
        target:
          entity_id: light.porch
        data:
          brightness_pct: 80
      - delay: "00:05:00"
      - service: light.turn_off
        target:
          entity_id: light.porch

  # 夜间车道人触发警报 + 灯
  - alias: "Deterrence - Driveway Person Night"
    trigger:
      - platform: mqtt
        topic: frigate/driveway/person
        payload: "on"
    condition:
      - condition: time
        after: "22:00:00"
        before: "06:00:00"
    action:
      - service: siren.turn_on
        target:
          entity_id: siren.outdoor
      - service: light.turn_on
        target:
          entity_id: light.floodlights
        data:
          brightness_pct: 100
      - service: notify.mobile_app
        data:
          title: "🚨 Driveway Intrusion"
          message: "Person detected at driveway at {{ now().strftime('%H:%M') }}"
```

### Alexa 常规 (App → Routines)

```
WHEN: Smart Home Device → [Front Door Camera] → Motion Detected → Person
IF: Time → Between Sunset and Sunrise
THEN:
    1. Smart Home → [Porch Light] → Turn On → 80% → 5 minutes
    2. Alexa Says → "Person at front door" → Device: [Kitchen Echo]
    3. Smart Home → [Floodlight] → Turn On → 100% → 3 minutes
```

### Google Home 自动化 (Home App → Automations)

```
STARTER: Device → [Driveway Camera] → Event → Person detected
CONDITION: Time → Between 10:00 PM and 6:00 AM
ACTIONS:
    1. Device → [Floodlights] → Turn On → Brightness 100%
    2. Device → [All Speakers] → Broadcast → "Person detected at driveway"
    3. Script → Log to Google Sheets (via Make/Zapier/webhook)
```

### Apple HomeKit (Home App → Automation)

```
WHEN: A Sensor Detects Something → [Front Door Camera] → People Detected
CONDITION: Time → After Sunset
DO:
    1. Scenes → [Porch Lights On] (预建场景: porch 80%, entry 50%)
    2. Wait → 5 minutes
    3. Scenes → [Porch Lights Off]
```

## 平台对比: 选哪个?

| 优先级              | 最佳平台                                | 理由                                      |
| ------------------- | --------------------------------------- | ----------------------------------------- |
| **隐私 / 无云**     | **HomeKit (HSV) + 本地桥接**            | 端到端加密, iCloud+ 含, 设备端 AI         |
| **语音 + 显示屏**   | **Alexa** (Echo Show 生态)              | 最多 Skill, 最好常规 UI, 便宜显示屏       |
| **自动化能力**      | **Google Home** (脚本编辑器)            | 最佳启动器类型, 复杂逻辑, Nest Hub 网格   |
| **本地 + 多平台**   | **Home Assistant + Frigate + Scrypted** | 完全控制, Matter 桥接, 无厂商锁定         |
| **最易上手**        | **Ring / Blink / Nest** (单品牌)        | 原生, 无桥接, 但订阅 + 云端               |
| **商业 / 多站点**   | **UniFi Protect**                       | 集中化, 无云费, HSV + Alexa/Google Skills |
| **Matter 未来兼容** | **TP-Link Tapo / Eve / Nanoleaf**       | Thread + Matter, 处处可用                 |

## 迁移路径: 从云到本地

1. **保留云端摄像头** 暂时 (Ring 门铃, Blink 户外)
2. **添加 PoE 摄像头** (Reolink, Dahua) → NVR (UniFi/Reolink) 或 Frigate 服务器
3. **桥接云端摄像头** 经 Homebridge/Scrypted → HomeKit/Google/Alexa
4. **迁移自动化** 到 Home Assistant (或 HA + Alexa/Google 常规)
5. **禁用云端录制** 桥接摄像头; 用本地 NVR/Frigate
6. **取消订阅** 本地录制验证后
7. **未来购买:** 仅买 ONVIF/RTSP + Matter

Le telecamere sicurezza diventano potenti quando parlano con la tua casa intelligente: luci lampeggiano su movimento, altoparlanti annunciano visitatori, serrature si sbloccano per volti riconosciuti, geofencing arma/disarma automaticamente. Ma non tutte le telecamere funzionano bene con tutte le piattaforme. Questa guida mappa ogni percorso di integrazione per il 2026.

**Supporto Nativo:** Ring, Blink, Arlo, Echo Show (come display)  
**Skill Richiesta:** Reolink, Eufy, UniFi, Wyze, Tapo, Amcrest, TP-Link Kasa

### Apple HomeKit (HomeKit Secure Video)

**Supporto Nativo:** Eufy (HomeBase 3), Arlo (Pro 3/4/5S, Ultra), Logitech Circle View, Netatmo, Eve, Aqara, TP-Link Tapo (Matter), UniFi Protect (3.0+), Onvis, Wemo  
**Bridge Richiesto:** Reolink (via Homebridge/Scrypted), Ring (via Homebridge), Wyze (via Homebridge), Dahua/Hikvision (via Homebridge/Scrypted)

<Callout type="info">

**HomeKit Secure Video (HSV) = Migliore Privacy.** Video analizzato **sul dispositivo**
(Home Hub: Apple TV 4K, HomePod, iPad). **Crittografato end-to-end** — Apple
non può visualizzare. Archiviazione rotativa 10 giorni **inclusa con iCloud+** (50GB = 1 telecamera,
200GB = 5 telecamere, 2TB = illimitato). Nessun abbonamento separato.

</Callout>

<Accordion type="single" collapsible>
  <AccordionItem value="hsv-features">
    <AccordionTrigger>Funzionalità HSV e Requisiti</AccordionTrigger>
    <AccordionContent>

**Risoluzioni Supportate:** 1080p, 2K, 4K (ridimensionato a 1080p per analisi, archiviato a risoluzione originale)
**Zone Attività:** Per telecamera, impostate in app Home
**Riconoscimento Volti:** "Persona rilevata" → "Mario rilevato" (se nell'album Persone)
**Opzioni Registrazione:** Spento / Rileva Attività / Streaming e Registrazione
**Archiviazione:** iCloud+ (50GB/€0.99, 200GB/€2.99, 2TB/€9.99) — **non per telecamera**
**Home Hub Richiesto:** Apple TV 4K (2021+), HomePod (mini/2a gen), iPad (sempre acceso)
**Streaming Locale:** iPhone/iPad sulla stessa Wi-Fi → diretto (nessun internet). Remoto → relay via iCloud (crittografato).
**Limitazioni:** Max 5 telecamere su piano 200GB (a meno di 2TB); nessuna registrazione 24/7 continua (solo eventi); nessuna vista web (solo iOS/macOS); nessun Android nativo.

    </AccordionContent>

  </AccordionItem>
  <AccordionItem value="homebridge">
    <AccordionTrigger>Bridging Telecamere Non-HomeKit (Homebridge / Scrypted / Hoobs)</AccordionTrigger>
    <AccordionContent>

**Homebridge** (Node.js, 2000+ plugin): Funziona su Pi, Docker, NAS, mini PC

- `homebridge-camera-ffmpeg` → RTSP → HSV (transcodifica, aggiunge movimento)
- `homebridge-reolink` → API nativa Reolink (movimento, PTZ, istantanee)
- `homebridge-dahua` / `homebridge-hikvision` → ONVIF/ISAPI
- `homebridge-ring` → Telecamere/campanelli Ring (polling, ~30s latenza)

**Scrypted** (TypeScript, più recente, più veloce): Plugin nativi HomeKit, Google, Alexa

- Migliore gestione RTSP, latenza inferiore, marketplace plugin
- Funziona sullo stesso hardware di Homebridge

**Hardware per Bridge:** Raspberry Pi 4 (4GB+) / 5, mini PC Intel N100/N305 (€120), vecchio laptop, Synology/QNAP Docker.
**Latenza:** Homebridge FFmpeg ~2–5s; Scrypted ~1–3s; HSV nativo ~0.5–1s.
**HSV via Bridge:** La telecamera deve supportare RTSP. Il bridge transcodifica → HSV → iCloud. **Conta verso il limite telecamere HSV.**

</AccordionContent>

  </AccordionItem>
</Accordion>

### Matter (Il Nuovo Standard)

**Matter 1.3 (2024)** aggiunge tipo dispositivo **Telecamera**. **Matter 1.4 (2025)** aggiunge tipo **Campanello**.

<Accordion type="single" collapsible>
  <AccordionItem value="matter-status">
    <AccordionTrigger>Supporto Telecamera Matter Attuale (2026)</AccordionTrigger>
    <AccordionContent>

**Telecamere Matter Certificate:**

- **TP-Link Tapo C120 / C220 / C320WS** (Matter su Wi-Fi)
- **Eve Cam** (Thread + Matter)
- **Google Nest Cam (cablata, 2024)** / **Nest Doorbell (cablata, 2024)** — bridge Matter via Google Home
- **EufyCam 3 / SoloCam S340** — Aggiornamento firmware Matter (2024)

**Cosa Ti Dà Matter:**

- **Multi-amministratore:** Aggiungi simultaneamente a Apple Home, Google Home, Alexa, SmartThings
- **Controllo Locale:** Nessun relay cloud per base on/off/stream (Thread/Wi-Fi diretto)
- **Comandi Standardizzati:** Stream live, eventi movimento, modalità privacy

**Cosa Matter NON Fa Ancora:**

- Streaming 4K (max 1080p nella specifica)
- Controllo PTZ (non in 1.3/1.4)
- Riconoscimento volti / rilevamento pacchi (solo estensioni fornitore)
- Registrazione continua (solo clip eventi)
- Integrazione NVR

**Verdetto (2026):** Le telecamere Matter funzionano per **visione live multi-piattaforma base + avvisi movimento**. Per registrazione, AI, PTZ, 4K → servono ancora integrazione nativa o bridge.

## Controllo Locale / Senza Cloud: Il Percorso Avanzato

<Callout type="tip">

**Locale = Veloce, Privato, Funziona Offline.** RTSP/ONVIF/WebSocket →
Frigate/Scrypted/Blue Iris → Home Assistant → Dashboard/Automazioni. Nessun
cloud fornitore vede mai il tuo video.

</Callout>

### Requisiti Telecamera per Stack Locale

- **Supporto RTSP** (standard: `rtsp://user:pass@ip:554/stream1`)
- **ONVIF Profile S/T/G** (scoperta, PTZ, eventi)
- **Sub-stream** (D1/720p per AI, 4K per registrazione)
- **Nessuna dipendenza cloud** per lo stream (alcune telecamere richiedono token auth cloud — evita)

**Migliori Telecamere ONVIF/RTSP (2026):**

- Reolink (tutte PoE, la maggior parte Wi-Fi): `rtsp://ip:554/h264Preview_01_main`
- Dahua / Hikvision / Hanwha / Axis / Uniview / Vivotek / Milesight
- UniFi G4/G5 (RTSP via impostazioni Protect)
- Amcrest / Lorex / Annke / Montavue (OEM Dahua/Hik)
- **Evita:** Ring, Blink, Nest, Arlo (pre-2024), Wyze (limitato), Eufy (nessun RTSU su telecamere a batteria)

### Home Assistant (YAML)

```yaml
# Luce Portico su Persona (dopo tramonto)
automazione:
  - alias: "Luce Portico - Persona alla Porta"
    trigger:
      - platform: mqtt
        topic: frigate/front_door/person
        payload: "on"
    condition:
      - condition: sun
        after: sunset
        after_offset: "-00:30:00"
    action:
      - service: light.turn_on
        target:
          entity_id: light.porch
        data:
          brightness_pct: 80
      - delay: "00:05:00"
      - service: light.turn_off
        target:
          entity_id: light.porch

  # Deterrenza: Sirena + Luci su Persona in Vialetto (notte)
  - alias: "Deterrenza - Persona Vialetto Notte"
    trigger:
      - platform: mqtt
        topic: frigate/driveway/person
        payload: "on"
    condition:
      - condition: time
        after: "22:00:00"
        before: "06:00:00"
    action:
      - service: siren.turn_on
        target:
          entity_id: siren.outdoor
      - service: light.turn_on
        target:
          entity_id: light.floodlights
        data:
          brightness_pct: 100
      - service: notify.mobile_app
        data:
          title: "🚨 Intrusione Vialetto"
          message: "Persona rilevata al vialetto alle {{ now().strftime('%H:%M') }}"
```

## Confronto Piattaforme: Quale Scegliere?

| Priorità                       | Migliore Piattaforma                    | Perché                                                       |
| ------------------------------ | --------------------------------------- | ------------------------------------------------------------ |
| **Privacy / Senza Cloud**      | **HomeKit (HSV) + Bridge Locale**       | Crittografia E2E, iCloud+ incluso, AI su dispositivo         |
| **Vocale + Display**           | **Alexa** (ecosistema Echo Show)        | Più skill, migliore UI routine, display economici            |
| **Potenza Automazione**        | **Google Home** (editor script)         | Migliori tipi attivatore, logica complessa, griglia Nest Hub |
| **Locale + Multi-Piattaforma** | **Home Assistant + Frigate + Scrypted** | Controllo totale, bridge Matter, nessun vendor lock-in       |
| **Configurazione Più Facile**  | **Ring / Blink / Nest** (marca singola) | Nativo, nessun bridge, ma abbonamento + cloud                |
| **Business / Multi-Sede**      | **UniFi Protect**                       | Centralizzato, nessuna tassa cloud, HSV + skill Alexa/Google |
| **Future-Proof Matter**        | **TP-Link Tapo / Eve / Nanoleaf**       | Thread + Matter, funziona ovunque                            |

## Percorso Migrazione: Da Cloud a Locale

1. **Tieni telecamere cloud** per ora (campanello Ring, Blink esterno)
2. **Aggiungi telecamere PoE** (Reolink, Dahua) → NVR (UniFi/Reolink) o server Frigate
3. **Bridge telecamere cloud** via Homebridge/Scrypted → HomeKit/Google/Alexa
4. **Sposta automazioni** su Home Assistant (o HA + routine Alexa/Google)
5. **Disabilita registrazione cloud** su telecamere bridgate; usa NVR/Frigate locale
6. **Cancella abbonamenti** quando registrazione locale verificata
7. **Future telecamere:** Compra solo ONVIF/RTSP + Matter

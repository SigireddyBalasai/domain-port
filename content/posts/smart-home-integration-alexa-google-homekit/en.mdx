## Platform Deep Dives

### Amazon Alexa

**Native Support:** Ring, Blink, Arlo, Echo Show (as display)  
**Skill Required:** Reolink, Eufy, UniFi, Wyze, Tapo, Amcrest, TP-Link Kasa

<Accordion type="single" collapsible>
  <AccordionItem value="alexa-features">
    <AccordionTrigger>What Alexa Can Do with Cameras</AccordionTrigger>
    <AccordionContent>
**Live View:** "Alexa, show [camera name]" → Echo Show, Fire TV, Alexa app  
**Announcements:** "Alexa, announce [camera] detected a person" → All Echo devices  
**Routines (Automation):**
- Trigger: Motion / Person / Package / Doorbell press
- Actions: Lights on/off, lock/unlock, speak custom phrase, play sound, wait, notification
**Two-Way Talk:** "Alexa, talk to [camera]" (half-duplex PTT)
**Recording:** Ring/Blink/Arlo → Cloud (subscription). Others → **No cloud recording via Alexa**.
**Limitations:** No local streaming (all cloud-relayed); 10-min timeout on Echo Show; no 4K (max 1080p transcoded); routines limited to 1 trigger + 1 action per routine (use multiple routines).
    </AccordionContent>
  </AccordionItem>
  <AccordionItem value="alexa-setup">
    <AccordionTrigger>Setup Tips</AccordionTrigger>
    <AccordionContent>
1. **Enable Skill** → Link account → Discover devices
2. **Rename** cameras in Alexa app to natural language ("front door", "driveway cam")
3. **Create Groups** ("Outside" = front + back + driveway) for "show outside"
4. **Routine Examples:**
   - **Porch Light:** Motion at front door 30 min after sunset → Turn on porch light (5 min)
   - **Deterrence:** Person detected at driveway → "Security camera recording" on Echo + Floodlight on
   - **Welcome:** Doorbell press → "Someone's at the front door" on Kitchen Echo
   - **Geofence:** "When I leave" → Arm cameras (if supported by skill)
    </AccordionContent>
  </AccordionItem>
</Accordion>

### Google Home (Google Assistant)

**Native Support:** Nest, Arlo, Blink, Ring (limited)  
**Action Required:** Eufy, Reolink, UniFi, Wyze, Tapo, Amcrest

**Announcements:** "Hey Google, broadcast [camera] sees a person" → All Nest speakers

- Actions: Lights, plugs, locks, thermostats, speakers, TV, scripts
**Two-Way Talk:** "Hey Google, talk to [camera]" (Nest only; others via app)
**Recording:** Nest Aware (cloud). Others → **No cloud recording via Google**.

### Apple HomeKit (HomeKit Secure Video)

**Native Support:** Eufy (HomeBase 3), Arlo (Pro 3/4/5S, Ultra), Logitech Circle View, Netatmo, Eve, Aqara, TP-Link Tapo (Matter), UniFi Protect (3.0+), Onvis, Wemo  
**Bridge Required:** Reolink (via Homebridge/Scrypted), Ring (via Homebridge), Wyze (via Homebridge), Dahua/Hikvision (via Homebridge/Scrypted)

<Callout type="info">
**HomeKit Secure Video (HSV) = Best Privacy.** Video analyzed **on-device** (Home Hub: Apple TV 4K, HomePod, iPad). **Encrypted end-to-end** — Apple cannot view. 10-day rolling storage **included with iCloud+** (50GB = 1 cam, 200GB = 5 cams, 2TB = unlimited). No separate subscription.
</Callout>

<Accordion type="single" collapsible>
  <AccordionItem value="hsv-features">
    <AccordionTrigger>HSV Features & Requirements</AccordionTrigger>
    <AccordionContent>
**Supported Resolutions:** 1080p, 2K, 4K (downscaled to 1080p for analysis, stored at source res)
**Activity Zones:** Per-camera, set in Home app
**Face Recognition:** "Person detected" → "John detected" (if in People album)
**Recording Options:** Off / Detect Activity / Stream & Record
**Storage:** iCloud+ (50GB/$0.99, 200GB/$2.99, 2TB/$9.99) — **not per camera**
**Home Hub Required:** Apple TV 4K (2021+), HomePod (mini/2nd gen), iPad (always on)
**Local Streaming:** iPhone/iPad on same Wi-Fi → direct (no internet). Remote → relayed via iCloud (encrypted).
**Limitations:** Max 5 cameras on 200GB plan (unless 2TB); no 24/7 continuous (event-only); no web view (iOS/macOS only); no native Android.
    </AccordionContent>
  </AccordionItem>
  <AccordionItem value="homebridge">
    <AccordionTrigger>Bridging Non-HomeKit Cameras (Homebridge / Scrypted / Hoobs)</AccordionTrigger>
    <AccordionContent>
**Homebridge** (Node.js, 2000+ plugins): Runs on Pi, Docker, NAS, mini PC
- `homebridge-camera-ffmpeg` → RTSP → HSV (transcodes, adds motion)
- `homebridge-reolink` → Reolink native API (motion, PTZ, snapshots)
- `homebridge-dahua` / `homebridge-hikvision` → ONVIF/ISAPI
- `homebridge-ring` → Ring cameras/doorbells (polling, ~30s latency)

**Scrypted** (TypeScript, newer, faster): Native HomeKit, Google, Alexa plugins
- Better RTSP handling, lower latency, plugin marketplace
- Runs on same hardware as Homebridge

**Hardware for Bridge:** Raspberry Pi 4 (4GB+) / 5, Intel N100/N305 mini PC ($120), old laptop, Synology/QNAP Docker.
**Latency:** Homebridge FFmpeg ~2–5s; Scrypted ~1–3s; Native HSV ~0.5–1s.
**HSV via Bridge:** Camera must support RTSP. Bridge transcodes → HSV → iCloud. **Counts toward HSV cam limit.**
    </AccordionContent>
  </AccordionItem>
</Accordion>

### Matter (The New Standard)

**Matter 1.3 (2024)** adds **Camera** device type. **Matter 1.4 (2025)** adds **Doorbell** type.

<Accordion type="single" collapsible>
  <AccordionItem value="matter-status">
    <AccordionTrigger>Current Matter Camera Support (2026)</AccordionTrigger>
    <AccordionContent>
**Certified Matter Cameras:**
- **TP-Link Tapo C120 / C220 / C320WS** (Matter over Wi-Fi)
- **Eveo **Nanoleaf Smart Cam** (Matter)
- **Eve Cam** (Thread + Matter)
- **Google Nest Cam (wired, 2024)** / **Nest Doorbell (wired, 2024)** — Matter bridge via Google Home
- **EufyCam 3 / SoloCam S340** — Matter firmware update (2024)

**What Matter Gives You:**
- **Multi-admin:** Add to Apple Home, Google Home, Alexa, SmartThings simultaneously
- **Local Control:** No cloud relay for basic on/off/stream (Thread/Wi-Fi direct)
- **Standardized Commands:** Live stream, motion events, privacy mode

**What Matter Does NOT Yet Do:**
- 4K streaming (max 1080p in spec)
- PTZ control (not in 1.3/1.4)
- Face recognition / package detection (vendor extensions only)
- Continuous recording (event clips only)
- NVR integration

**Verdict (2026):** Matter cameras work for **basic multi-platform live view + motion alerts**. For recording, AI, PTZ, 4K → still need native integration or bridge.

### Hardware for Local Stack

|-----------|--------|-------------|----------|
| **Server** | RPi 4 4GB ($80) | Intel N100/N305 Mini PC ($130) | i5-13400 / Ryzen 5700G ($400) |
| **AI Accelerator** | CPU only (slow) | **Google Coral USB ($60)** / M.2 ($40) | Coral PCIe x4 / Hailo-8 ($150) |
| **Storage** | 1× 4TB HDD | 2× 8TB RAID 1 | 4× 14TB RAID 6 / ZFS |
| **Network** | 1 Gbps | 2.5 Gbps switch + NIC | 10 Gbps backbone |

### Camera Requirements for Local Stack

- **RTSP Support** (standard: `rtsp://user:pass@ip:554/stream1`)
- **ONVIF Profile S/T/G** (discovery, PTZ, events)
- **Sub-streams** (D1/720p for AI, 4K for recording)
- **No cloud dependency** for stream (some cams require cloud auth token — avoid)

**Best ONVIF/RTSP Cameras (2026):**
- Reolink (all PoE, most Wi-Fi): `rtsp://ip:554/h264Preview_01_main`
- Dahua / Hikvision / Hanwha / Axis / Uniview / Vivotek / Milesight
- UniFi G4/G5 (RTSP via Protect settings)
- Amcrest / Lorex / Annke / Montavue (OEM Dahua/Hik)
- **Avoid:** Ring, Blink, Nest, Arlo (pre-2024), Wyze (limited), Eufy (no RTSP on battery cams)

## Automation Recipes (Copy-Paste Ready)

### Home Assistant (YAML)

```yaml
# Porch Light on Person (after sunset)
automation:
  - alias: "Porch Light - Person at Front Door"
    trigger:
      - platform: mqtt
        topic: frigate/front_door/person
        payload: "on"
    condition:
      - condition: sun
        after: sunset
        after_offset: "-00:30:00"
    action:
      - service: light.turn_on
        target:
          entity_id: light.porch
        data:
          brightness_pct: 80
      - delay: "00:05:00"
      - service: light.turn_off
        target:
          entity_id: light.porch

# Deterrence: Siren + Lights on Person at Driveway (night)
  - alias: "Deterrence - Driveway Person Night"
    trigger:
      - platform: mqtt
        topic: frigate/driveway/person
        payload: "on"
    condition:
      - condition: time
        after: "22:00:00"
        before: "06:00:00"
    action:
      - service: siren.turn_on
        target:
          entity_id: siren.outdoor
      - service: light.turn_on
        target:
          entity_id: light.floodlights
        data:
          brightness_pct: 100
      - service: notify.mobile_app
        data:
          title: "🚨 Driveway Intrusion"
          message: "Person detected at driveway at {{ now().strftime('%H:%M') }}"
```

## Platform Comparison: Which Should You Choose?

| Priority | Best Platform | Why |
|----------|---------------|-----|
| **Privacy / No Cloud** | **HomeKit (HSV) + Local Bridge** | E2E encryption, iCloud+ included, on-device AI |
| **Voice + Displays** | **Alexa** (Echo Show ecosystem) | Most skills, best routine UI, cheap displays |
| **Automation Power** | **Google Home** (script editor) | Best starter types, complex logic, Nest Hub grid |
| **Local + Multi-Platform** | **Home Assistant + Frigate + Scrypted** | Total control, Matter bridge, no vendor lock-in |
| **Easiest Setup** | **Ring / Blink / Nest** (single brand) | Native, no bridge, but subscription + cloud |
| **Business / Multi-Site** | **UniFi Protect** | Centralized, no cloud fee, HSV + Alexa/Google skills |
| **Matter Future-Proof** | **TP-Link Tapo / Eve / Nanoleaf** | Thread + Matter, works everywhere |

## Migration Path: From Cloud to Local

1. **Keep cloud cams** for now (Ring doorbell, Blink outdoor)
2. **Add PoE cams** (Reolink, Dahua) → NVR (UniFi/Reolink) or Frigate server
3. **Bridge cloud cams** via Homebridge/Scrypted → HomeKit/Google/Alexa
4. **Move automations** to Home Assistant (or HA + Alexa/Google routines)
5. **Disable cloud recording** on bridged cams; use local NVR/Frigate
6. **Cancel subscriptions** when local recording verified
7. **Future cams:** Buy ONVIF/RTSP + Matter only

As câmeras de segurança se tornam poderosas quando conversam com sua casa inteligente: luzes piscam ao detectar movimento, alto-falantes anunciam visitantes, fechaduras destravam para rostos reconhecidos, cerca geográfica arma/desarma automaticamente. Mas nem todas as câmeras se integram bem com todas as plataformas. Este guia mapeia cada caminho de integração para 2026.

## Análises Aprofundadas das Plataformas

### Amazon Alexa

**Suporte Nativo:** Ring, Blink, Arlo, Echo Show (como display)  
**Skill Necessária:** Reolink, Eufy, UniFi, Wyze, Tapo, Amcrest, TP-Link Kasa

### Apple HomeKit (HomeKit Secure Video)

**Suporte Nativo:** Eufy (HomeBase 3), Arlo (Pro 3/4/5S, Ultra), Logitech Circle View, Netatmo, Eve, Aqara, TP-Link Tapo (Matter), UniFi Protect (3.0+), Onvis, Wemo  
**Bridge Necessária:** Reolink (via Homebridge/Scrypted), Ring (via Homebridge), Wyze (via Homebridge), Dahua/Hikvision (via Homebridge/Scrypted)

<Callout type="info">

**HomeKit Secure Video (HSV) = Melhor Privacidade.** Vídeo analisado **no
dispositivo** (Home Hub: Apple TV 4K, HomePod, iPad). **Criptografado ponta a
ponta** — Apple não pode visualizar. Armazenamento contínuo de 10 dias
**incluído com iCloud+** (50GB = 1 câmera, 200GB = 5 câmeras, 2TB =
ilimitado). Sem assinatura separada.

</Callout>

<Accordion type="single" collapsible>
  <AccordionItem value="hsv-features">
    <AccordionTrigger>Recursos e Requisitos do HSV</AccordionTrigger>
    <AccordionContent>

**Resoluções Suportadas:** 1080p, 2K, 4K (reduzido para 1080p para análise, armazenado na resolução original)
**Zonas de Atividade:** Por câmera, definidas no app Home
**Reconhecimento Facial:** "Pessoa detectada" → "João detectado" (se estiver no álbum Pessoas)
**Opções de Gravação:** Desligado / Detectar Atividade / Transmitir e Gravar
**Armazenamento:** iCloud+ (50GB/$0.99, 200GB/$2.99, 2TB/$9.99) — **não por câmera**
**Home Hub Necessário:** Apple TV 4K (2021+), HomePod (mini/2ª geração), iPad (sempre ligado)
**Transmissão Local:** iPhone/iPad no mesmo Wi-Fi → direto (sem internet). Remoto → retransmitido via iCloud (criptografado).
**Limitações:** Máx. 5 câmeras no plano de 200GB (a menos que 2TB); sem contínuo 24/7 (apenas eventos); sem visualização web (apenas iOS/macOS); sem Android nativo.

    </AccordionContent>

  </AccordionItem>
  <AccordionItem value="homebridge">
    <AccordionTrigger>Integrando Câmeras Não HomeKit (Homebridge / Scrypted / Hoobs)</AccordionTrigger>
    <AccordionContent>

**Homebridge** (Node.js, 2000+ plugins): Roda em Pi, Docker, NAS, mini PC

- `homebridge-camera-ffmpeg` → RTSP → HSV (transcodifica, adiciona movimento)
- `homebridge-reolink` → API nativa Reolink (movimento, PTZ, snapshots)
- `homebridge-dahua` / `homebridge-hikvision` → ONVIF/ISAPI
- `homebridge-ring` → Câmeras/campainhas Ring (polling, ~30s de latência)

**Scrypted** (TypeScript, mais novo, mais rápido): Plugins nativos HomeKit, Google, Alexa

- Melhor manipulação RTSP, menor latência, mercado de plugins
- Roda no mesmo hardware que o Homebridge

**Hardware para Bridge:** Raspberry Pi 4 (4GB+) / 5, Intel N100/N305 mini PC ($120), laptop antigo, Docker Synology/QNAP.
**Latência:** Homebridge FFmpeg ~2–5s; Scrypted ~1–3s; HSV nativo ~0.5–1s.
**HSV via Bridge:** A câmera deve suportar RTSP. A bridge transcodifica → HSV → iCloud. **Conta para o limite de câmeras HSV.**

</AccordionContent>
  </AccordionItem>
</Accordion>

### Matter (O Novo Padrão)

**Matter 1.3 (2024)** adiciona o tipo de dispositivo **Câmera**. **Matter 1.4 (2025)** adiciona o tipo **Campainha**.

<Accordion type="single" collapsible>
  <AccordionItem value="matter-status">
    <AccordionTrigger>Suporte Atual para Câmeras Matter (2026)</AccordionTrigger>
    <AccordionContent>

**Câmeras Matter Certificadas:**

- **TP-Link Tapo C120 / C220 / C320WS** (Matter sobre Wi-Fi)
- **Eve Cam** (Thread + Matter)
- **Google Nest Cam (com fio, 2024)** / **Nest Doorbell (com fio, 2024)** — bridge Matter via Google Home
- **EufyCam 3 / SoloCam S340** — atualização de firmware Matter (2024)

**O Que o Matter Oferece:**

- **Multi-administrador:** Adicione ao Apple Home, Google Home, Alexa, SmartThings simultaneamente
- **Controle Local:** Sem retransmissão em nuvem para ligar/desligar/transmitir básico (Thread/Wi-Fi direto)
- **Comandos Padronizados:** Transmissão ao vivo, eventos de movimento, modo privacidade

**O Que o Matter AINDA Não Faz:**

- Transmissão 4K (máx. 1080p na especificação)
- Controle PTZ (não no 1.3/1.4)
- Reconhecimento facial / detecção de pacotes (apenas extensões do fornecedor)
- Gravação contínua (apenas clipes de eventos)
- Integração NVR

**Veredito (2026):** Câmeras Matter funcionam para **visualização ao vivo básica multiplataforma + alertas de movimento**. Para gravação, IA, PTZ, 4K → ainda precisa de integração nativa ou bridge.

## Controle Local / Sem Nuvem: O Caminho Avançado

<Callout type="tip">

**Local = Rápido, Privado, Funciona Offline.** RTSP/ONVIF/WebSocket →
Frigate/Scrypted/Blue Iris → Home Assistant → Painéis/Automações. Nenhuma
nuvem do fornecedor vê seu vídeo.

</Callout>

### Arquitetura: Pilha de Câmeras Totalmente Local

```
Câmeras (PoE/RTSP)
    │
    ▼
┌─────────────────────────────────────┐
│  Frigate (Docker)                   │
│  - Ingestão RTSP                      │
│  - Detecção de movimento (CPU)           │
│  - Detecção de objetos IA (Coral TPU)  │
│  - Gravação (MP4, segmentada)       │
│  - Eventos MQTT (movimento, pessoa, etc)│

└─────────────────────────────────────┘
    │                    │
    ▼                    ▼
┌─────────────┐    ┌─────────────┐
│ Home Asst.  │    │  Scrypted   │
│ (Automação)│    │ (HomeKit/   │
│  - Luzes   │    │  Google/    │
│  - Fechaduras    │  Alexa)     │
│  - Alertas   │    │             │
└─────────────┘    └─────────────┘
    │                    │
```

### Hardware para Pilha Local

| Componente           | Econômico          | Recomendado                            | Alto Padrão                    |
| -------------------- | ------------------ | -------------------------------------- | ------------------------------ |
| **Servidor**         | RPi 4 4GB ($80)    | Intel N100/N305 Mini PC ($130)         | i5-13400 / Ryzen 5700G ($400)  |
| **Acelerador de IA** | Apenas CPU (lento) | **Google Coral USB ($60)** / M.2 ($40) | Coral PCIe x4 / Hailo-8 ($150) |
| **Armazenamento**    | 1× 4TB HDD         | 2× 8TB RAID 1                          | 4× 14TB RAID 6 / ZFS           |
| **Rede**             | 1 Gbps             | Switch 2.5 Gbps + NIC                  | Backbone 10 Gbps               |

### Requisitos de Câmera para Pilha Local

- **Suporte RTSP** (padrão: `rtsp://user:pass@ip:554/stream1`)
- **ONVIF Profile S/T/G** (descoberta, PTZ, eventos)
- **Sub-fluxos** (D1/720p para IA, 4K para gravação)
- **Sem dependência de nuvem** para transmissão (algumas câmeras exigem token de autenticação na nuvem — evite)

**Melhores Câmeras ONVIF/RTSP (2026):**

- Reolink (todas PoE, a maioria Wi-Fi): `rtsp://ip:554/h264Preview_01_main`
- Dahua / Hikvision / Hanwha / Axis / Uniview / Vivotek / Milesight
- UniFi G4/G5 (RTSP via configurações Protect)
- Amcrest / Lorex / Annke / Montavue (OEM Dahua/Hik)
- **Evite:** Ring, Blink, Nest, Arlo (pré-2024), Wyze (limitado), Eufy (sem RTSP em câmeras de bateria)

## Receitas de Automação (Prontas para Copiar)

### Home Assistant (YAML)

```yaml
# Luz do Alpendre ao detectar Pessoa (após o pôr do sol)
automation:
  - alias: "Luz do Alpendre - Pessoa na Porta da Frente"
    trigger:
      - platform: mqtt
        topic: frigate/front_door/person
        payload: "on"
    condition:
      - condition: sun
        after: sunset
        after_offset: "-00:30:00"
    action:
      - service: light.turn_on
        target:
          entity_id: light.porch
        data:
          brightness_pct: 80
      - delay: "00:05:00"
      - service: light.turn_off
        target:
          entity_id: light.porch

  # Dissuasão: Sirene + Luzes ao detectar Pessoa na Garagem (noite)
  - alias: "Dissuasão - Pessoa na Garagem à Noite"
    trigger:
      - platform: mqtt
        topic: frigate/driveway/person
        payload: "on"
    condition:
      - condition: time
        after: "22:00:00"
        before: "06:00:00"
    action:
      - service: siren.turn_on
        target:
          entity_id: siren.outdoor
      - service: light.turn_on
        target:
          entity_id: light.floodlights
        data:
          brightness_pct: 100
      - service: notify.mobile_app
        data:
          title: "🚨 Intrusão na Garagem"
          message: "Pessoa detectada na garagem às {{ now().strftime('%H:%M') }}"
```

### Rotina Alexa (App → Rotinas)

```
QUANDO: Dispositivo Smart Home → [Câmera da Porta da Frente] → Movimento Detectado → Pessoa
SE: Horário → Entre o Pôr e o Nascer do Sol
ENTÃO:
    1. Smart Home → [Luz do Alpendre] → Ligar → 80% → 5 minutos
    2. Alexa Diz → "Pessoa na porta da frente" → Dispositivo: [Echo da Cozinha]
    3. Smart Home → [Holofote] → Ligar → 100% → 3 minutos
```

### Automação Google Home (App Home → Automações)

```
INICIADOR: Dispositivo → [Câmera da Garagem] → Evento → Pessoa detectada
CONDIÇÃO: Horário → Entre 22:00 e 06:00
AÇÕES:
    1. Dispositivo → [Holofotes] → Ligar → Brilho 100%
    2. Dispositivo → [Todos os Alto-falantes] → Transmitir → "Pessoa detectada na garagem"
    3. Script → Registrar no Google Planilhas (via Make/Zapier/webhook)
```

### Apple HomeKit (App Home → Automação)

```
QUANDO: Um Sensor Detecta Algo → [Câmera da Porta da Frente] → Pessoas Detectadas
CONDIÇÃO: Horário → Após o Pôr do Sol
FAZER:
    1. Cenas → [Luzes do Alpendre Ligadas] (cena pré-criada: alpendre 80%, entrada 50%)
    2. Aguardar → 5 minutos
    3. Cenas → [Luzes do Alpendre Desligadas]
```

## Comparação de Plataformas: Qual Escolher?

| Prioridade                   | Melhor Plataforma                       | Porquê                                                       |
| ---------------------------- | --------------------------------------- | ------------------------------------------------------------ |
| **Privacidade / Sem Nuvem**  | **HomeKit (HSV) + Bridge Local**        | Criptografia E2E, iCloud+ incluído, IA no dispositivo        |
| **Voz + Displays**           | **Alexa** (ecossistema Echo Show)       | Mais skills, melhor interface de rotina, displays baratos    |
| **Poder de Automação**       | **Google Home** (editor de script)      | Melhores tipos de iniciador, lógica complexa, grade Nest Hub |
| **Local + Multiplataforma**  | **Home Assistant + Frigate + Scrypted** | Controle total, bridge Matter, sem dependência de fornecedor |
| **Configuração Mais Fácil**  | **Ring / Blink / Nest** (marca única)   | Nativo, sem bridge, mas assinatura + nuvem                   |
| **Empresarial / Multilocal** | **UniFi Protect**                       | Centralizado, sem taxa de nuvem, skills HSV + Alexa/Google   |
| **À Prova de Futuro Matter** | **TP-Link Tapo / Eve / Nanoleaf**       | Thread + Matter, funciona em todos os lugares                |

## Caminho de Migração: Da Nuvem para o Local

1. **Mantenha câmeras na nuvem** por enquanto (Ring campainha, Blink externa)
2. **Adicione câmeras PoE** (Reolink, Dahua) → NVR (UniFi/Reolink) ou servidor Frigate
3. **Faça bridge de câmeras na nuvem** via Homebridge/Scrypted → HomeKit/Google/Alexa
4. **Mova automações** para o Home Assistant (ou HA + rotinas Alexa/Google)
5. **Desative a gravação em nuvem** nas câmeras com bridge; use NVR/Frigate local
6. **Cancele assinaturas** quando a gravação local for verificada
7. **Futuras câmeras:** Compre apenas ONVIF/RTSP + Matter

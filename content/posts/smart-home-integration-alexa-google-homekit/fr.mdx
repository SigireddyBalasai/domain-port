Les caméras de sécurité deviennent puissantes quand elles communiquent avec votre maison connectée : les lumières s'allument au mouvement, les enceintes annoncent les visiteurs, les serrures se déverrouillent pour les visages reconnus, le géorepérage arme/désarme automatiquement. Mais toutes les caméras ne fonctionnent pas avec toutes les plateformes. Ce guide cartographie chaque chemin d'intégration pour 2026.

### Apple HomeKit (HomeKit Secure Video)

**Compatibilité Native :** Eufy (HomeBase 3), Arlo (Pro 3/4/5S, Ultra), Logitech Circle View, Netatmo, Eve, Aqara, TP-Link Tapo (Matter), UniFi Protect (3.0+), Onvis, Wemo  
**Pont Requis :** Reolink (via Homebridge/Scrypted), Ring (via Homebridge), Wyze (via Homebridge), Dahua/Hikvision (via Homebridge/Scrypted)

<Callout type="info">

**HomeKit Secure Video (HSV) = Meilleure Confidentialité.** Vidéo analysée
**sur l'appareil** (Hub Domotique : Apple TV 4K, HomePod, iPad). **Chiffrée de
bout en bout** — Apple ne peut pas la voir. Stockage avec rolling de 10 jours
**inclus avec iCloud+** (50 Go = 1 caméra, 200 Go = 5 caméras, 2 To =
illimité). Pas d'abonnement séparé.

</Callout>

<Accordion type="single" collapsible>
  <AccordionItem value="hsv-features">
    <AccordionTrigger>Fonctionnalités et Exigences HSV</AccordionTrigger>
    <AccordionContent>

**Résolutions Supportées :** 1080p, 2K, 4K (réduite à 1080p pour l'analyse, stockée en résolution source)
**Zones d'Activité :** Par caméra, définies dans l'app Home
**Reconnaissance Faciale :** "Personne détectée" → "John détecté" (si dans l'album Personnes)
**Options d'Enregistrement :** Désactivé / Détecter l'Activité / Streamer & Enregistrer
**Stockage :** iCloud+ (50 Go/0,99€, 200 Go/2,99€, 2 To/9,99€) — **pas par caméra**
**Hub Domotique Requis :** Apple TV 4K (2021+), HomePod (mini/2e génération), iPad (toujours allumé)
**Streaming Local :** iPhone/iPad sur le même Wi-Fi → direct (pas d'internet). À distance → relayé via iCloud (chiffré).
**Limitations :** Max 5 caméras sur le forfait 200 Go (sauf 2 To) ; pas de continu 24/7 (événements uniquement) ; pas de vue web (iOS/macOS uniquement) ; pas d'Android natif.

    </AccordionContent>

  </AccordionItem>
  <AccordionItem value="homebridge">
    <AccordionTrigger>Pont pour Caméras Non-HomeKit (Homebridge / Scrypted / Hoobs)</AccordionTrigger>
    <AccordionContent>

**Homebridge** (Node.js, 2000+ plugins) : Tourne sur Pi, Docker, NAS, mini PC

- `homebridge-camera-ffmpeg` → RTSP → HSV (transcode, ajoute mouvement)
- `homebridge-reolink` → API native Reolink (mouvement, PTZ, instantanés)
- `homebridge-dahua` / `homebridge-hikvision` → ONVIF/ISAPI
- `homebridge-ring` → Caméras/sonnettes Ring (polling, ~30s de latence)

**Scrypted** (TypeScript, plus récent, plus rapide) : Plugins HomeKit, Google, Alexa natifs

- Meilleure gestion RTSP, latence plus faible, marché de plugins
- Tourne sur le même matériel que Homebridge

**Matériel pour le Pont :** Raspberry Pi 4 (4 Go+) / 5, Intel N100/N305 mini PC (120$), vieux portable, Synology/QNAP Docker.
**Latence :** Homebridge FFmpeg ~2–5s ; Scrypted ~1–3s ; HSV natif ~0.5–1s.
**HSV via Pont :** La caméra doit supporter RTSP. Le pont transcode → HSV → iCloud. **Compte dans la limite de caméras HSV.**

</AccordionContent>

  </AccordionItem>
</Accordion>

### Matter (Le Nouveau Standard)

**Matter 1.3 (2024)** ajoute le type d'appareil **Caméra**. **Matter 1.4 (2025)** ajoute le type **Sonnette**.

<Accordion type="single" collapsible>
  <AccordionItem value="matter-status">
    <AccordionTrigger>Support Actuel des Caméras Matter (2026)</AccordionTrigger>
    <AccordionContent>

**Caméras Matter Certifiées :**

- **TP-Link Tapo C120 / C220 / C320WS** (Matter over Wi-Fi)
- **Nanoleaf Smart Cam** (Matter)
- **Eve Cam** (Thread + Matter)
- **Google Nest Cam (filaire, 2024)** / **Nest Doorbell (filaire, 2024)** — Pont Matter via Google Home
- **EufyCam 3 / SoloCam S340** — Mise à jour firmware Matter (2024)

**Ce que Matter Vous Apporte :**

- **Multi-administrateur :** Ajouter à Apple Home, Google Home, Alexa, SmartThings simultanément
- **Contrôle Local :** Pas de relayage cloud pour l'allumage/extinction/stream de base (Thread/Wi-Fi direct)
- **Commandes Standardisées :** Flux en direct, événements de mouvement, mode confidentialité

**Ce que Matter NE FAIT PAS Encore :**

- Streaming 4K (max 1080p dans la spécification)
- Contrôle PTZ (pas dans 1.3/1.4)
- Reconnaissance faciale / détection de colis (extensions fabricant uniquement)
- Enregistrement continu (clips d'événements uniquement)
- Intégration NVR

**Verdict (2026) :** Les caméras Matter fonctionnent pour **la vue en direct multi-plateforme de base + les alertes de mouvement**. Pour l'enregistrement, l'IA, le PTZ, la 4K → encore besoin d'intégration native ou d'un pont.

</AccordionContent>

</Accordion>

## Contrôle Local / Sans Cloud : La Voie Avancée

<Callout type="tip">

**Local = Rapide, Privé, Fonctionne Hors Ligne.** RTSP/ONVIF/WebSocket →
Frigate/Scrypted/Blue Iris → Home Assistant → Tableaux de
bord/Automatisations. Aucun cloud de fournisseur ne voit jamais votre vidéo.

</Callout>

### Architecture : Stack Caméra Entièrement Locale

```
Caméras (PoE/RTSP)
    │
    ▼
┌─────────────────────────────────────┐
│  Frigate (Docker)                   │
│  - RTSP ingest                      │
│  - Détection de mouvement (CPU)     │
│  - Détection d'objets IA (Coral TPU)│
│  - Enregistrement (MP4, segmenté)   │
│  - Événements MQTT (mouvement, etc.)│
│  - Vue en direct WebRTC (sub-seconde)│
└─────────────────────────────────────┘
    │                    │
    ▼                    ▼
┌─────────────┐    ┌─────────────┐
│ Home Asst.  │    │  Scrypted   │
│ (Automat.)  │    │ (HomeKit/   │
│  - Lumières │    │  Google/    │
│  - Serrures │    │  Alexa)     │
│  - Alertes  │    │             │
└─────────────┘    └─────────────┘
    │                    │
    ▼                    ▼
Tableau de bord      Apple Home /
                     Google Home /
                     App Alexa
```

### Matériel pour Stack Local

| Composant           | Budget                | Recommandé                             | Haut de Gamme                  |
| ------------------- | --------------------- | -------------------------------------- | ------------------------------ |
| **Serveur**         | RPi 4 4 Go (80$)      | Intel N100/N305 Mini PC (130$)         | i5-13400 / Ryzen 5700G (400$)  |
| **Accélérateur IA** | CPU uniquement (lent) | **Google Coral USB (60$)** / M.2 (40$) | Coral PCIe x4 / Hailo-8 (150$) |
| **Stockage**        | 1× 4 To HDD           | 2× 8 To RAID 1                         | 4× 14 To RAID 6 / ZFS          |
| **Réseau**          | 1 Gbps                | Switch 2.5 Gbps + NIC                  | Backbone 10 Gbps               |

### Exigences Caméra pour Stack Local

- **Support RTSP** (standard : `rtsp://utilisateur:motdepasse@ip:554/stream1`)
- **ONVIF Profile S/T/G** (découverte, PTZ, événements)
- **Sous-flux** (D1/720p pour l'IA, 4K pour l'enregistrement)
- **Pas de dépendance cloud** pour le flux (certaines caméras nécessitent un token d'authentification cloud — à éviter)

**Meilleures Caméras ONVIF/RTSP (2026) :**

- Reolink (tous PoE, plupart Wi-Fi) : `rtsp://ip:554/h264Preview_01_main`
- Dahua / Hikvision / Hanwha / Axis / Uniview / Vivotek / Milesight
- UniFi G4/G5 (RTSP via les paramètres Protect)
- Amcrest / Lorex / Annke / Montavue (OEM Dahua/Hik)
- **À éviter :** Ring, Blink, Nest, Arlo (avant 2024), Wyze (limité), Eufy (pas de RTSP sur caméras à batterie)

## Recettes d'Automatisation (Prêtes à Copier)

### Home Assistant (YAML)

```yaml
# Lumière Porche sur Personne (après le coucher du soleil)
automation:
  - alias: "Porche Lumière - Personne à la Porte d'Entrée"
    trigger:
      - platform: mqtt
        topic: frigate/front_door/person
        payload: "on"
    condition:
      - condition: sun
        after: sunset
        after_offset: "-00:30:00"
    action:
      - service: light.turn_on
        target:
          entity_id: light.porch
        data:
          brightness_pct: 80
      - delay: "00:05:00"
      - service: light.turn_off
        target:
          entity_id: light.porch

  # Dissuasion : Sirène + Lumières sur Personne dans l'Allée (nuit)
  - alias: "Dissuasion - Allée Personne Nuit"
    trigger:
      - platform: mqtt
        topic: frigate/driveway/person
        payload: "on"
    condition:
      - condition: time
        after: "22:00:00"
        before: "06:00:00"
    action:
      - service: siren.turn_on
        target:
          entity_id: siren.outdoor
      - service: light.turn_on
        target:
          entity_id: light.floodlights
        data:
          brightness_pct: 100
      - service: notify.mobile_app
        data:
          title: "🚨 Intrusion Allée"
          message: "Personne détectée dans l'allée à {{ now().strftime('%H:%M') }}"
```

### Routine Alexa (App → Routines)

```
QUAND : Appareil Domotique → [Caméra Porte d'Entrée] → Mouvement Détecté → Personne
SI : Heure → Entre le Coucher et le Lever du Soleil
ALORS :
    1. Appareil Domotique → [Lumière Porche] → Allumer → 80% → 5 minutes
    2. Alexa Dit → "Personne à la porte d'entrée" → Appareil : [Echo Cuisine]
    3. Appareil Domotique → [Projecteur] → Allumer → 100% → 3 minutes

```

## Comparaison des Plateformes : Laquelle Choisir ?

| Priorité                           | Meilleure Plateforme                    | Pourquoi                                                           |
| ---------------------------------- | --------------------------------------- | ------------------------------------------------------------------ |
| **Confidentialité / Pas de Cloud** | **HomeKit (HSV) + Pont Local**          | Chiffrement E2E, iCloud+ inclus, IA sur l'appareil                 |
| **Voix + Écrans**                  | **Alexa** (écosystème Echo Show)        | Plus de Skills, meilleure UI des routines, écrans bon marché       |
| **Puissance d'Automatisation**     | **Google Home** (éditeur de scripts)    | Meilleurs types de déclencheurs, logique complexe, grille Nest Hub |
| **Local + Multi-Plateforme**       | **Home Assistant + Frigate + Scrypted** | Contrôle total, pont Matter, pas de verrouillage fournisseur       |
| **Configuration la Plus Facile**   | **Ring / Blink / Nest** (marque unique) | Natif, pas de pont, mais abonnement + cloud                        |
| **Professionnel / Multi-Site**     | **UniFi Protect**                       | Centralisé, pas de frais cloud, HSV + Skills Alexa/Google          |
| **Matter Prêt pour l'Avenir**      | **TP-Link Tapo / Eve / Nanoleaf**       | Thread + Matter, fonctionne partout                                |

## Chemin de Migration : Du Cloud au Local

1. **Gardez les caméras cloud** pour l'instant (sonnette Ring, Blink extérieur)
2. **Ajoutez des caméras PoE** (Reolink, Dahua) → NVR (UniFi/Reolink) ou serveur Frigate
3. **Pont les caméras cloud** via Homebridge/Scrypted → HomeKit/Google/Alexa
4. **Déplacez les automatisations** vers Home Assistant (ou HA + routines Alexa/Google)
5. **Désactivez l'enregistrement cloud** sur les caméras pontées ; utilisez le NVR/Frigate local
6. **Annulez les abonnements** une fois l'enregistrement local vérifié
7. **Futures caméras :** Achetez ONVIF/RTSP + Matter uniquement

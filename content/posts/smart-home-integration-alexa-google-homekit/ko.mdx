**기본 지원:** Ring, Blink, Arlo, Echo Show (디스플레이로)

### Apple HomeKit (HomeKit Secure Video)

**기본 지원:** Eufy (HomeBase 3), Arlo (Pro 3/4/5S, Ultra), Logitech Circle View, Netatmo, Eve, Aqara, TP-Link Tapo (Matter), UniFi Protect (3.0+), Onvis, Wemo  
**브리지 필요:** Reolink (Homebridge/Scrypted 통해), Ring (Homebridge 통해), Wyze (Homebridge 통해), Dahua/Hikvision (Homebridge/Scrypted 통해)

<Callout type="info">

**HomeKit Secure Video (HSV) = 최고의 프라이버시.** 비디오는 **장치 내에서** 분석 (Home Hub: Apple TV 4K, HomePod, iPad). **종단간 암호화** — Apple도 볼 수 없음. 10일 롤링 스토리지가 iCloud+에 **포함** (50GB = 1캠, 200GB = 5캠, 2TB = 무제한). 별도 구독 불필요.

</Callout>

<Accordion type="single" collapsible>
  <AccordionItem value="hsv-features">
    <AccordionTrigger>HSV 기능 및 요구사항</AccordionTrigger>
    <AccordionContent>

**지원 해상도:** 1080p, 2K, 4K (분석을 위해 1080p로 다운스케일, 원본 해상도로 저장)
**활동 영역:** 카메라별, Home 앱에서 설정
**얼굴 인식:** "사람 감지됨" → "John 감지됨" (People 앨범에 있을 경우)
**녹화 옵션:** 꺼짐 / 활동 감지 / 스트림 및 녹화
**스토리지:** iCloud+ (50GB/$0.99, 200GB/$2.99, 2TB/$9.99) — **카메라당 아님**
**Home Hub 필요:** Apple TV 4K (2021+), HomePod (mini/2세대), iPad (항상 켜짐)
**로컬 스트리밍:** 동일 Wi-Fi의 iPhone/iPad → 직접 (인터넷 불필요). 원격 → iCloud 통해 중계 (암호화).
**제한 사항:** 200GB 요금제에서 최대 5개 카메라 (2TB면 무제한); 24/7 연속 녹화 없음 (이벤트 전용); 웹 보기 없음 (iOS/macOS 전용); 기본 Android 없음.

    </AccordionContent>

  </AccordionItem>
  <AccordionItem value="homebridge">
    <AccordionTrigger>비HomeKit 카메라 연결 (Homebridge / Scrypted / Hoobs)</AccordionTrigger>
    <AccordionContent>

**Homebridge** (Node.js, 2000+ 플러그인): Pi, Docker, NAS, 미니 PC에서 실행

- `homebridge-camera-ffmpeg` → RTSP → HSV (트랜스코딩, 모션 추가)
- `homebridge-reolink` → Reolink 네이티브 API (모션, PTZ, 스냅샷)
- `homebridge-dahua` / `homebridge-hikvision` → ONVIF/ISAPI
- `homebridge-ring` → Ring 카메라/초인종 (폴링, ~30초 지연)

**Scrypted** (TypeScript, 최신, 더 빠름): 네이티브 HomeKit, Google, Alexa 플러그인

- 더 나은 RTSP 처리, 낮은 지연 시간, 플러그인 마켓플레이스
- Homebridge와 동일한 하드웨어에서 실행

**브리지 하드웨어:** Raspberry Pi 4 (4GB+) / 5, Intel N100/N305 미니 PC ($120), 구형 노트북, Synology/QNAP Docker.
**지연 시간:** Homebridge FFmpeg ~2–5초; Scrypted ~1–3초; 네이티브 HSV ~0.5–1초.
**브리지를 통한 HSV:** 카메라는 RTSP를 지원해야 함. 브리지가 트랜스코딩 → HSV → iCloud. **HSV 캠 제한에 포함됨.**

</AccordionContent>

  </AccordionItem>
</Accordion>

**Matter 1.3 (2024)** 에 **카메라** 장치 유형 추가. **Matter 1.4 (2025)** 에 **초인종** 유형 추가.

**인증된 Matter 카메라:**

- **TP-Link Tapo C120 / C220 / C320WS** (Wi-Fi over Matter)
- **Eve / Nanoleaf Smart Cam** (Matter)
- **Eve Cam** (Thread + Matter)
- **Google Nest Cam (유선, 2024)** / **Nest Doorbell (유선, 2024)** — Google Home 통해 Matter 브리지
- **EufyCam 3 / SoloCam S340** — Matter 펌웨어 업데이트 (2024)

**Matter가 제공하는 것:**

- **멀티어드민:** Apple Home, Google Home, Alexa, SmartThings에 동시 추가
- **로컬 제어:** 기본 on/off/스트림에 클라우드 중계 불필요 (Thread/Wi-Fi 직접)
- **표준화된 명령:** 라이브 스트림, 모션 이벤트, 프라이버시 모드

**Matter가 아직 지원하지 않는 것:**

- 4K 스트리밍 (사양상 최대 1080p)
- PTZ 제어 (1.3/1.4에 없음)
- 얼굴 인식 / 패키지 감지 (공급업체 확장만 가능)
- 연속 녹화 (이벤트 클립만)
- NVR 통합

**결론 (2026):** Matter 카메라는 **기본 멀티플랫폼 라이브 뷰 + 모션 알림**에 적합. 녹화, AI, PTZ, 4K는 여전히 네이티브 통합 또는 브리지 필요.

## 로컬 제어 / 노클라우드: 고급 경로

<Callout type="tip">

**로컬 = 빠름, 프라이빗, 오프라인 작동.** RTSP/ONVIF/WebSocket → Frigate/Scrypted/Blue Iris → Home Assistant → 대시보드/자동화. 공급업체 클라우드가 비디오를 보지 않음.

</Callout>

### 아키텍처: 완전 로컬 카메라 스택

```
Cameras (PoE/RTSP)
    │
    ▼

│  Frigate (Docker)                   │
│  - RTSP 수집                        │
│  - 모션 감지 (CPU)                  │

    │                    │
    ▼                    ▼
┌─────────────┐    ┌─────────────┐
│ Home Asst.  │    │  Scrypted   │
```

### 로컬 스택을 위한 카메라 요구사항

- **RTSP 지원** (표준: `rtsp://user:pass@ip:554/stream1`)
- **ONVIF Profile S/T/G** (검색, PTZ, 이벤트)
- **서브스트림** (D1/720p for AI, 4K for 녹화)
- **스트림에 클라우드 의존성 없음** (일부 카메라는 클라우드 인증 토큰 필요 — 피할 것)

**최고의 ONVIF/RTSP 카메라 (2026):**

- Reolink (모든 PoE, 대부분 Wi-Fi): `rtsp://ip:554/h264Preview_01_main`
- Dahua / Hikvision / Hanwha / Axis / Uniview / Vivotek / Milesight
- UniFi G4/G5 (Protect 설정에서 RTSP)
- Amcrest / Lorex / Annke / Montavue (OEM Dahua/Hik)
- **피할 것:** Ring, Blink, Nest, Arlo (2024 이전), Wyze (제한적), Eufy (배터리 캠에 RTSP 없음)

## 자동화 레시피 (복사-붙여넣기 준비)

### Home Assistant (YAML)

```yaml
# 해진 후 현관등 - 사람 감지 시
automation:
  - alias: "Porch Light - Person at Front Door"
    trigger:
      - platform: mqtt
        topic: frigate/front_door/person
        payload: "on"
    condition:
      - condition: sun
        after: sunset
        after_offset: "-00:30:00"
    action:
      - service: light.turn_on
        target:
          entity_id: light.porch
        data:
          brightness_pct: 80
      - delay: "00:05:00"
      - service: light.turn_off
        target:
          entity_id: light.porch

  # 억지력: 야간 진입로 사람 감지 시 사이렌 + 조명
  - alias: "Deterrence - Driveway Person Night"
    trigger:
      - platform: mqtt
        topic: frigate/driveway/person
        payload: "on"
    condition:
      - condition: time
        after: "22:00:00"
        before: "06:00:00"
    action:
      - service: siren.turn_on
        target:
          entity_id: siren.outdoor
      - service: light.turn_on
        target:
          entity_id: light.floodlights
        data:
          brightness_pct: 100
      - service: notify.mobile_app
        data:
          title: "🚨 진입로 침입"
          message: "진입로에서 {{ now().strftime('%H:%M') }}에 사람 감지됨"
```

### Alexa 루틴 (앱 → 루틴)

```
WHEN: Smart Home Device → [Front Door Camera] → Motion Detected → Person
IF: Time → Between Sunset and Sunrise
THEN:
    1. Smart Home → [Porch Light] → Turn On → 80% → 5 minutes
    2. Alexa Says → "현관문에 사람이 있습니다" → Device: [Kitchen Echo]
    3. Smart Home → [Floodlight] → Turn On → 100% → 3 minutes
```

### Google Home 자동화 (Home 앱 → 자동화)

```
STARTER: Device → [Driveway Camera] → Event → Person detected
CONDITION: Time → Between 10:00 PM and 6:00 AM
ACTIONS:
    1. Device → [Floodlights] → Turn On → Brightness 100%
    2. Device → [All Speakers] → Broadcast → "진입로에서 사람 감지됨"
    3. Script → Google Sheets 기록 (Make/Zapier/webhook 통해)
```

### Apple HomeKit (Home 앱 → 자동화)

```
WHEN: A Sensor Detects Something → [Front Door Camera] → People Detected
CONDITION: Time → After Sunset
DO:
    1. Scenes → [Porch Lights On] (미리 생성된 씬: 현관 80%, 입구 50%)
    2. Wait → 5 minutes
    3. Scenes → [Porch Lights Off]
```

## 플랫폼 비교: 어떤 것을 선택해야 할까?

| 우선순위                    | 최적 플랫폼                             | 이유                                                      |
| --------------------------- | --------------------------------------- | --------------------------------------------------------- |
| **프라이버시 / 노클라우드** | **HomeKit (HSV) + 로컬 브리지**         | E2E 암호화, iCloud+ 포함, 온디바이스 AI                   |
| **음성 + 디스플레이**       | **Alexa** (Echo Show 생태계)            | 가장 많은 Skill, 최고의 루틴 UI, 저렴한 디스플레이        |
| **자동화 성능**             | **Google Home** (스크립트 편집기)       | 최고의 시작 조건 유형, 복잡한 로직, Nest Hub 그리드       |
| **로컬 + 멀티플랫폼**       | **Home Assistant + Frigate + Scrypted** | 완전한 제어, Matter 브리지, 공급업체 종속 없음            |
| **가장 쉬운 설정**          | **Ring / Blink / Nest** (단일 브랜드)   | 네이티브, 브리지 불필요, 하지만 구독 + 클라우드           |
| **비즈니스 / 멀티사이트**   | **UniFi Protect**                       | 중앙 집중식, 클라우드 요금 없음, HSV + Alexa/Google Skill |
| **Matter 미래 대비**        | **TP-Link Tapo / Eve / Nanoleaf**       | Thread + Matter, 모든 곳에서 작동                         |

## 마이그레이션 경로: 클라우드에서 로컬로

1. **클라우드 캠 유지** (Ring 초인종, Blink 야외)
2. **PoE 캠 추가** (Reolink, Dahua) → NVR (UniFi/Reolink) 또는 Frigate 서버
3. **클라우드 캠 브리지** Homebridge/Scrypted → HomeKit/Google/Alexa
4. **자동화를 Home Assistant로 이동** (또는 HA + Alexa/Google 루틴)
5. **브리지된 캠의 클라우드 녹화 비활성화**; 로컬 NVR/Frigate 사용
6. **로컬 녹화 확인 시 구독 취소**
7. **향후 캠:** ONVIF/RTSP + Matter만 구매

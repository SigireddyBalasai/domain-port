**ネイティブサポート:** Ring、Blink、Arlo、Echo Show（表示端末として）  
**スキル必須:** Reolink、Eufy、UniFi、Wyze、Tapo、Amcrest、TP-Link Kasa

**ネイティブサポート:** Nest、Arlo、Blink、Ring（制限あり）  
**アクション必須:** Eufy、Reolink、UniFi、Wyze、Tapo、Amcrest

      スマホ **アナウンス:** 「OK Google、[カメラ]が人を見つけたと放送」→
      全Nestスピーカー **自動化（Homeアプリ → 自動化）:** - トリガー:

      ライト、プラグ、ロック、サーモスタット、スピーカー、TV、スクリプト **双方向通話:**
      「OK Google、[カメラ]と通話」（Nestのみ; その他はアプリ経由） **録画:**
      Nest Aware（クラウド）。その他 → **Google経由のクラウド録画なし**。

### Apple HomeKit（HomeKit Secure Video）

**ネイティブサポート:** Eufy（HomeBase 3）、Arlo（Pro 3/4/5S、Ultra）、Logitech Circle View、Netatmo、Eve、Aqara、TP-Link Tapo（Matter）、UniFi Protect（3.0+）、Onvis、Wemo  
**ブリッジ必須:** Reolink（Homebridge/Scrypted経由）、Ring（Homebridge経由）、Wyze（Homebridge経由）、Dahua/Hikvision（Homebridge/Scrypted経由）

<Callout type="info">

**HomeKit Secure Video (HSV) = 最高のプライバシー。** 映像は**デバイス上で**分析（Home Hub: Apple TV 4K、HomePod、iPad）。**エンドツーエンド暗号化** — Appleも閲覧不可。iCloud+で**10日間のロリングストレージが含まれます**（50GB＝1台、200GB＝5台、2TB＝無制限）。別途サブスクリプション不要。

</Callout>

<Accordion type="single" collapsible>
  <AccordionItem value="hsv-features">
    <AccordionTrigger>HSVの機能と要件</AccordionTrigger>
    <AccordionContent>

**対応解像度:** 1080p、2K、4K（分析用に1080pにダウンスケール、保存は元の解像度）
**アクティビティゾーン:** Homeアプリでカメラごとに設定
**顔認識:** 「人を検出」→ 「Johnを検出」（Peopleアルバムに登録されている場合）
**録画オプション:** オフ / アクティビティを検出 / ストリーム＆録画
**ストレージ:** iCloud+（50GB/$0.99、200GB/$2.99、2TB/$9.99）— **カメラごとではない**
**Home Hub必須:** Apple TV 4K（2021+）、HomePod（mini/第2世代）、iPad（常時オン）
**ローカルストリーミング:** 同じWi-Fi上のiPhone/iPad → 直接（インターネット不要）。リモート → iCloud経由で中継（暗号化）。
**制限:** 200GBプランでは最大5台（2TB以外の場合）; 24時間連続録画不可（イベントのみ）; Web表示不可（iOS/macOSのみ）; ネイティブAndroid非対応。

    </AccordionContent>

  </AccordionItem>
  <AccordionItem value="homebridge">
    <AccordionTrigger>非HomeKitカメラのブリッジ（Homebridge / Scrypted / Hoobs）</AccordionTrigger>
    <AccordionContent>

**Homebridge**（Node.js、2000+ プラグイン）: Pi、Docker、NAS、ミニPCで動作

- `homebridge-camera-ffmpeg` → RTSP → HSV（トランスコード、動体追加）
- `homebridge-reolink` → ReolinkネイティブAPI（動体、PTZ、スナップショット）
- `homebridge-dahua` / `homebridge-hikvision` → ONVIF/ISAPI
- `homebridge-ring` → Ringカメラ/ドアベル（ポーリング、~30秒遅延）

**Scrypted**（TypeScript、より新しく、より高速）: ネイティブHomeKit、Google、Alexaプラグイン

- より優れたRTSP処理、低遅延、プラグインマーケットプレイス
- Homebridgeと同じハードウェアで動作

**ブリッジ用ハードウェア:** Raspberry Pi 4（4GB+）/ 5、Intel N100/N305 ミニPC（$120）、古いノートPC、Synology/QNAP Docker。
**遅延:** Homebridge FFmpeg ~2–5秒; Scrypted ~1–3秒; ネイティブHSV ~0.5–1秒。
**HSV via ブリッジ:** カメラはRTSPに対応している必要があります。ブリッジがトランスコード → HSV → iCloud。**HSVカメラ制限にカウントされます。**

</AccordionContent>

  </AccordionItem>
</Accordion>

### Matter（新しい標準）

**Matter 1.3（2024年）** で**カメラ**デバイスタイプが追加。**Matter 1.4（2025年）** で**ドアベル**タイプが追加。

<Accordion type="single" collapsible>
  <AccordionItem value="matter-status">
    <AccordionTrigger>現在のMatterカメラサポート状況（2026年）</AccordionTrigger>
    <AccordionContent>

**認定Matterカメラ:**

- **TP-Link Tapo C120 / C220 / C320WS**（Matter over Wi-Fi）
- **Nanoleaf Smart Cam**（Matter）
- **Eve Cam**（Thread + Matter）
- **Google Nest Cam（有線、2024年）** / **Nest Doorbell（有線、2024年）** — Google Home経由のMatterブリッジ
- **EufyCam 3 / SoloCam S340** — Matterファームウェアアップデート（2024年）

**Matterが提供するもの:**

- **マルチアドミン:** Apple Home、Google Home、Alexa、SmartThingsに同時追加可能
- **ローカル制御:** 基本的なオン/オフ/ストリームにクラウド中継不要（Thread/Wi-Fi直接）
- **標準化されたコマンド:** ライブストリーム、動体イベント、プライバシーモード

**Matterがまだできないこと:**

- 4Kストリーミング（仕様上最大1080p）
- PTZ制御（1.3/1.4では未対応）
- 顔認識 / 荷物検出（ベンダー拡張のみ）
- 連続録画（イベントクリップのみ）
- NVR統合

**結論（2026年）:** Matterカメラは**基本的なマルチプラットフォームライブビュー＋動体通知**に使用可能。録画、AI、PTZ、4Kにはネイティブ統合またはブリッジがまだ必要。

## ローカル制御 / クラウド不要: 上級者向けパス

<Callout type="tip">

**ローカル＝高速、プライベート、オフラインでも動作。** RTSP/ONVIF/WebSocket →
Frigate/Scrypted/Blue Iris → Home Assistant → ダッシュボード/自動化。ベンダーのクラウドがあなたの映像を見ることは決してありません。

</Callout>

### ローカルスタック用ハードウェア

| コンポーネント       | 予算            | 推奨                                   | ハイエンド                     |
| -------------------- | --------------- | -------------------------------------- | ------------------------------ |
| **サーバー**         | RPi 4 4GB ($80) | Intel N100/N305 ミニPC ($130)          | i5-13400 / Ryzen 5700G ($400)  |
| **AIアクセラレータ** | CPUのみ（低速） | **Google Coral USB ($60)** / M.2 ($40) | Coral PCIe x4 / Hailo-8 ($150) |
| **ストレージ**       | 1× 4TB HDD      | 2× 8TB RAID 1                          | 4× 14TB RAID 6 / ZFS           |
| **ネットワーク**     | 1 Gbps          | 2.5 Gbps スイッチ + NIC                | 10 Gbps バックボーン           |

### ローカルスタックのカメラ要件

- **RTSPサポート**（標準: `rtsp://user:pass@ip:554/stream1`）
- **ONVIF Profile S/T/G**（検出、PTZ、イベント）
- **サブストリーム**（D1/720pをAI用、4Kを録画用）
- **ストリームにクラウド依存なし**（一部のカメラはクラウド認証トークンが必要 — 避ける）

**最高のONVIF/RTSPカメラ（2026年）:**

- Reolink（全PoE、ほとんどのWi-Fi）: `rtsp://ip:554/h264Preview_01_main`
- Dahua / Hikvision / Hanwha / Axis / Uniview / Vivotek / Milesight
- UniFi G4/G5（Protect設定でRTSP）
- Amcrest / Lorex / Annke / Montavue（OEM Dahua/Hik）
- **避ける:** Ring、Blink、Nest、Arlo（2024年以前）、Wyze（制限あり）、Eufy（バッテリーカメラにRTSPなし）

## 自動化レシピ（コピペで使用可能）

### Home Assistant（YAML）

```yaml
# 人検出で玄関灯点灯（日没後）
automation:
  - alias: "玄関灯 - 正面玄関の人"
    trigger:
      - platform: mqtt
        topic: frigate/front_door/person
        payload: "on"
    condition:
      - condition: sun
        after: sunset
        after_offset: "-00:30:00"
    action:
      - service: light.turn_on
        target:
          entity_id: light.porch
        data:
          brightness_pct: 80
      - delay: "00:05:00"
      - service: light.turn_off
        target:
          entity_id: light.porch

  # 抑止: 私道で人検出時にサイレン＋ライト（夜間）
  - alias: "抑止 - 私道の人_夜間"
    trigger:
      - platform: mqtt
        topic: frigate/driveway/person
        payload: "on"
    condition:
      - condition: time
        after: "22:00:00"
        before: "06:00:00"
    action:
      - service: siren.turn_on
        target:
          entity_id: siren.outdoor
      - service: light.turn_on
        target:
          entity_id: light.floodlights
        data:
          brightness_pct: 100
      - service: notify.mobile_app
        data:
          title: "🚨 私道侵入"
          message: "私道で人を検出しました（{{ now().strftime('%H:%M') }}）"
```

## プラットフォーム比較: どれを選ぶべきか？

| 優先順位                             | 最適プラットフォーム                    | 理由                                                   |
| ------------------------------------ | --------------------------------------- | ------------------------------------------------------ |
| **プライバシー / クラウドなし**      | **HomeKit (HSV) + ローカルブリッジ**    | E2E暗号化、iCloud+含む、オンデバイスAI                 |
| **音声＋ディスプレイ**               | **Alexa**（Echo Showエコシステム）      | 最多スキル、最良のルーチンUI、安価なディスプレイ       |
| **自動化のパワー**                   | **Google Home**（スクリプトエディタ）   | 最良のトリガータイプ、複雑なロジック、Nest Hubグリッド |
| **ローカル＋マルチプラットフォーム** | **Home Assistant + Frigate + Scrypted** | 完全制御、Matterブリッジ、ベンダーロックインなし       |
| **最も簡単なセットアップ**           | **Ring / Blink / Nest**（単一ブランド） | ネイティブ、ブリッジ不要、ただしサブスク＋クラウド     |
| **ビジネス / マルチサイト**          | **UniFi Protect**                       | 集中管理、クラウド料金なし、HSV＋Alexa/Googleスキル    |
| **Matter将来性確保**                 | **TP-Link Tapo / Eve / Nanoleaf**       | Thread + Matter、どこでも動作                          |

## 移行パス: クラウドからローカルへ

1. **当面はクラウドカメラを維持**（Ringドアベル、Blink屋外用）
2. **PoEカメラを追加**（Reolink、Dahua）→ NVR（UniFi/Reolink）またはFrigateサーバー
3. **クラウドカメラをHomebridge/Scryptedでブリッジ** → HomeKit/Google/Alexa
4. **自動化をHome Assistantに移行**（またはHA＋Alexa/Googleルーチン）
5. **ブリッジしたカメラのクラウド録画を無効化**; ローカルNVR/Frigateを使用
6. **ローカル録画確認後、サブスクリプションを解約**
7. **将来のカメラ:** ONVIF/RTSP＋Matterのみ購入

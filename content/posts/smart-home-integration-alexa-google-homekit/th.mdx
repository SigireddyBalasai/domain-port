**รองรับแบบเนทีฟ:** Ring, Blink, Arlo, Echo Show (เป็นจอแสดงผล)  
**ต้องใช้ Skill:** Reolink, Eufy, UniFi, Wyze, Tapo, Amcrest, TP-Link Kasa

### Apple HomeKit (HomeKit Secure Video)

**รองรับแบบเนทีฟ:** Eufy (HomeBase 3), Arlo (Pro 3/4/5S, Ultra), Logitech Circle View, Netatmo, Eve, Aqara, TP-Link Tapo (Matter), UniFi Protect (3.0+), Onvis, Wemo  
**ต้องใช้บริดจ์:** Reolink (ผ่าน Homebridge/Scrypted), Ring (ผ่าน Homebridge), Wyze (ผ่าน Homebridge), Dahua/Hikvision (ผ่าน Homebridge/Scrypted)

<Callout type="info">

**HomeKit Secure Video (HSV) = ความเป็นส่วนตัวที่ดีที่สุด** วิดีโอถูกวิเคราะห์
**บนอุปกรณ์** (Home Hub: Apple TV 4K, HomePod, iPad) **เข้ารหัสแบบ
end-to-end** — Apple ไม่สามารถดูได้ พื้นที่จัดเก็บแบบหมุนเวียน 10 วัน **รวมกับ
iCloud+** (50GB = 1 กล้อง, 200GB = 5 กล้อง, 2TB = ไม่จำกัด)
ไม่ต้องสมัครสมาชิกแยก

</Callout>

**ความละเอียดที่รองรับ:** 1080p, 2K, 4K (ปรับลดเป็น 1080p เพื่อการวิเคราะห์, เก็บที่ความละเอียดต้นทาง)
**โซนกิจกรรม:** ต่อกล้อง, ตั้งค่าในแอป Home
**การจดจำใบหน้า:** "ตรวจพบบุคคล" → "ตรวจพบจอห์น" (ถ้าอยู่ในอัลบั้มบุคคล)
**ตัวเลือกการบันทึก:** ปิด / ตรวจจับกิจกรรม / สตรีมและบันทึก
**พื้นที่จัดเก็บ:** iCloud+ (50GB/$0.99, 200GB/$2.99, 2TB/$9.99) — **ไม่ใช่ต่อกล้อง**
**ต้องใช้ Home Hub:** Apple TV 4K (2021+), HomePod (mini/รุ่น 2), iPad (เปิดตลอดเวลา)
**สตรีมมิงในท้องถิ่น:** iPhone/iPad บน Wi-Fi เดียวกัน → โดยตรง (ไม่ต้องใช้อินเทอร์เน็ต) ระยะไกล → ถ่ายทอดผ่าน iCloud (เข้ารหัส)
**ข้อจำกัด:** สูงสุด 5 กล้องบนแผน 200GB (ยกเว้น 2TB); ไม่มีต่อเนื่อง 24/7 (เฉพาะเหตุการณ์); ไม่มีมุมมองเว็บ (เฉพาะ iOS/macOS); ไม่มี Android ดั้งเดิม

    </AccordionContent>

**Homebridge** (Node.js, 2000+ ปลั๊กอิน): ทำงานบน Pi, Docker, NAS, มินิ PC

- `homebridge-camera-ffmpeg` → RTSP → HSV (transcode, เพิ่มการเคลื่อนไหว)
- `homebridge-reolink` → API ดั้งเดิมของ Reolink (การเคลื่อนไหว, PTZ, ภาพนิ่ง)
- `homebridge-dahua` / `homebridge-hikvision` → ONVIF/ISAPI
- `homebridge-ring` → กล้อง/กริ่ง Ring (polling, ~30s หน่วงเวลา)

**Scrypted** (TypeScript, ใหม่กว่า, เร็วกว่า): ปลั๊กอินเนทีฟ HomeKit, Google, Alexa

- จัดการ RTSP ได้ดีกว่า, หน่วงเวลาน้อยกว่า, ตลาดปลั๊กอิน
- ทำงานบนฮาร์ดแวร์เดียวกับ Homebridge

**ฮาร์ดแวร์สำหรับบริดจ์:** Raspberry Pi 4 (4GB+) / 5, Intel N100/N305 มินิ PC ($120), โน้ตบุ๊กเก่า, Docker Synology/QNAP
**ความหน่วง:** Homebridge FFmpeg ~2–5s; Scrypted ~1–3s; HSV เนทีฟ ~0.5–1s
**HSV ผ่านบริดจ์:** กล้องต้องรองรับ RTSP บริดจ์ transcode → HSV → iCloud **นับรวมในขีดจำกัดกล้อง HSV**

</AccordionContent>

</Accordion>

**Matter 1.3 (2024)** เพิ่มประเภทอุปกรณ์ **กล้อง** **Matter 1.4 (2025)** เพิ่มประเภท **กริ่งประตู**

**กล้องที่ได้รับการรับรอง Matter:**

- **TP-Link Tapo C120 / C220 / C320WS** (Matter ผ่าน Wi-Fi)
- **Eve Cam** (Thread + Matter)
- **Google Nest Cam (แบบมีสาย, 2024)** / **Nest Doorbell (แบบมีสาย, 2024)** — บริดจ์ Matter ผ่าน Google Home
- **EufyCam 3 / SoloCam S340** — อัปเดตเฟิร์มแวร์ Matter (2024)

**สิ่งที่ Matter มอบให้:**

- **ผู้ดูแลหลายระบบ:** เพิ่มไปยัง Apple Home, Google Home, Alexa, SmartThings พร้อมกัน
- **ควบคุมในท้องถิ่น:** ไม่ต้องถ่ายทอดผ่านคลาวด์สำหรับเปิด/ปิด/สตรีมพื้นฐาน (Thread/Wi-Fi โดยตรง)
- **คำสั่งที่ได้มาตรฐาน:** สตรีมสด, เหตุการณ์การเคลื่อนไหว, โหมดความเป็นส่วนตัว

**สิ่งที่ Matter ยังไม่ทำ:**

- สตรีม 4K (สูงสุด 1080p ในข้อกำหนด)
- ควบคุม PTZ (ไม่อยู่ใน 1.3/1.4)
- จดจำใบหน้า / ตรวจจับพัสดุ (เฉพาะส่วนขยายจากผู้ขาย)
- บันทึกต่อเนื่อง (เฉพาะคลิปเหตุการณ์)
- การรวม NVR

**คำตัดสิน (2026):** กล้อง Matter ใช้ได้สำหรับ **ดูสดหลายแพลตฟอร์มพื้นฐาน + แจ้งเตือนการเคลื่อนไหว** สำหรับการบันทึก, AI, PTZ, 4K → ยังต้องใช้การรวมเนทีฟหรือบริดจ์

## ควบคุมในท้องถิ่น / ไม่ใช้คลาวด์: เส้นทางขั้นสูง

<Callout type="tip">

**Local = เร็ว, ส่วนตัว, ทำงานแบบออฟไลน์** RTSP/ONVIF/WebSocket →
Frigate/Scrypted/Blue Iris → Home Assistant → แดชบอร์ด/ระบบอัตโนมัติ ไม่มี
คลาวด์ของผู้ขายเห็นวิดีโอของคุณ

</Callout>

### ฮาร์ดแวร์สำหรับสแตกท้องถิ่น

| ส่วนประกอบ         | ประหยัด            | แนะนำ                                  | ระดับสูง                       |
| ------------------ | ------------------ | -------------------------------------- | ------------------------------ |
| **เซิร์ฟเวอร์**    | RPi 4 4GB ($80)    | Intel N100/N305 มินิ PC ($130)         | i5-13400 / Ryzen 5700G ($400)  |
| **ตัวเร่ง AI**     | CPU เท่านั้น (ช้า) | **Google Coral USB ($60)** / M.2 ($40) | Coral PCIe x4 / Hailo-8 ($150) |
| **พื้นที่จัดเก็บ** | 1× 4TB HDD         | 2× 8TB RAID 1                          | 4× 14TB RAID 6 / ZFS           |
| **เครือข่าย**      | 1 Gbps             | สวิตช์ 2.5 Gbps + NIC                  | แบ็คโบน 10 Gbps                |

### ข้อกำหนดกล้องสำหรับสแตกท้องถิ่น

- **รองรับ RTSP** (มาตรฐาน: `rtsp://user:pass@ip:554/stream1`)
- **ONVIF Profile S/T/G** (การค้นพบ, PTZ, เหตุการณ์)
- **สตรีมย่อย** (D1/720p สำหรับ AI, 4K สำหรับบันทึก)
- **ไม่พึ่งพาคลาวด์** สำหรับสตรีม (กล้องบางตัวต้องใช้โทเค็นยืนยันตัวตนคลาวด์ — หลีกเลี่ยง)

**กล้อง ONVIF/RTSP ที่ดีที่สุด (2026):**

## สูตรระบบอัตโนมัติ (พร้อมคัดลอก)

### Home Assistant (YAML)

```yaml
# ไฟเฉลียงเมื่อตรวจพบบุคคล (หลังพระอาทิตย์ตก)
automation:
  - alias: "ไฟเฉลียง - บุคคลที่ประตูหน้า"
    trigger:
      - platform: mqtt
        topic: frigate/front_door/person
        payload: "on"
    condition:
      - condition: sun
        after: sunset
        after_offset: "-00:30:00"
    action:
      - service: light.turn_on
        target:
          entity_id: light.porch
        data:
          brightness_pct: 80
      - delay: "00:05:00"
      - service: light.turn_off
        target:
          entity_id: light.porch

  # ขัดขวาง: ไซเรน + ไฟเมื่อตรวจพบบุคคลที่ถนน (กลางคืน)
  - alias: "ขัดขวาง - บุคคลที่ถนนกลางคืน"
    trigger:
      - platform: mqtt
        topic: frigate/driveway/person
        payload: "on"
    condition:
      - condition: time
        after: "22:00:00"
        before: "06:00:00"
    action:
      - service: siren.turn_on
        target:
          entity_id: siren.outdoor
      - service: light.turn_on
        target:
          entity_id: light.floodlights
        data:
          brightness_pct: 100
      - service: notify.mobile_app
        data:
          title: "🚨 บุกรุกถนน"
          message: "ตรวจพบบุคคลที่ถนนเวลา {{ now().strftime('%H:%M') }}"
```

## เปรียบเทียบแพลตฟอร์ม: เลือกอันไหน?

| ลำดับความสำคัญ                     | แพลตฟอร์มที่ดีที่สุด                    | เหตุผล                                               |
| ---------------------------------- | --------------------------------------- | ---------------------------------------------------- |
| **ความเป็นส่วนตัว / ไม่ใช้คลาวด์** | **HomeKit (HSV) + บริดจ์ท้องถิ่น**      | เข้ารหัส E2E, iCloud+ รวม, AI บนอุปกรณ์              |
| **เสียง + จอแสดงผล**               | **Alexa** (ระบบนิเวศ Echo Show)         | Skill มากที่สุด, UI กิจวัตรดีที่สุด, จอแสดงผลราคาถูก |
| **พลังระบบอัตโนมัติ**              | **Google Home** (ตัวแก้ไขสคริปต์)       | ตัวเริ่มที่ดีที่สุด, ตรรกะซับซ้อน, กริด Nest Hub     |
| **ท้องถิ่น + หลายแพลตฟอร์ม**       | **Home Assistant + Frigate + Scrypted** | ควบคุมทั้งหมด, บริดจ์ Matter, ไม่ถูกล็อกโดยผู้ขาย    |
| **ตั้งค่าง่ายที่สุด**              | **Ring / Blink / Nest** (แบรนด์เดียว)   | เนทีฟ, ไม่ต้องใช้บริดจ์, แต่ต้องสมัครสมาชิก + คลาวด์ |
| **ธุรกิจ / หลายสถานที่**           | **UniFi Protect**                       | รวมศูนย์, ไม่มีค่าคลาวด์, skill HSV + Alexa/Google   |
| **อนาคต Matter**                   | **TP-Link Tapo / Eve / Nanoleaf**       | Thread + Matter, ทำงานได้ทุกที่                      |

## เส้นทางการย้าย: จากคลาวด์สู่ท้องถิ่น

1. **เก็บกล้องคลาวด์ไว้ก่อน** (Ring กริ่งประตู, Blink กลางแจ้ง)
2. **เพิ่มกล้อง PoE** (Reolink, Dahua) → NVR (UniFi/Reolink) หรือเซิร์ฟเวอร์ Frigate
3. **เชื่อมกล้องคลาวด์** ผ่าน Homebridge/Scrypted → HomeKit/Google/Alexa
4. **ย้ายระบบอัตโนมัติ** ไปยัง Home Assistant (หรือ HA + กิจวัตร Alexa/Google)
5. **ปิดการบันทึกคลาวด์** บนกล้องที่เชื่อม; ใช้ NVR/Frigate ท้องถิ่น
6. **ยกเลิกการสมัครสมาชิก** เมื่อยืนยันการบันทึกในท้องถิ่นแล้ว
7. **กล้องในอนาคต:** ซื้อเฉพาะ ONVIF/RTSP + Matter เท่านั้น

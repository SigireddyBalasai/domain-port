La reconnaissance faciale dans les caméras de sécurité est passée d'un "luxe d'API cloud" à un "standard sur appareil". Mais toutes les fonctionnalités faciales ne sont pas égales : détection ≠ reconnaissance ≠ identification. Et les lois sur la vie privée (BIPA, CCPA, RGPD, lois étatiques biométriques) ont du mordant. Ce guide détaille ce qui est possible, ce qui est légal et quelles caméras le font correctement.

<Badge variant="outline">Vérification Terminologique</Badge>- **Détection
Faciale :** "Il y a un visage dans ce plan" — pas d'identité - **Reconnaissance
Faciale :** "Ce visage correspond à un visage connu dans *ma* base de données" —
1:1 ou 1:peu - **Identification Faciale :** "Ce visage = John Doe dans la base
de données *mondiale*" — 1:nombreux (échelle policière)

Les caméras grand public font **détection + reconnaissance (base de données locale uniquement)**. L'identification nécessite le cloud + mandats.

## Meilleures Caméras avec Reconnaissance Faciale sur Appareil (2026)

### 1. EufyCam 3 S330 + HomeBase 3 — Meilleur Polyvalent Grand Public

**NPU :** HomeBase 3 (NPU double cœur, 2 TOPS)  
**Base de données :** 100 visages par HomeBase  
**Reconnaissance :** 1:1 (inscrits) + 1:peu (famille)  
**Vie Privée :** **Entièrement local** — pas de cloud pour la reconnaissance ; sauvegarde cloud optionnelle _chiffrée_  
**Fonctionnalités :** Visage + personne + véhicule + animal + colis ; zones d'activité ; Apple HomeKit Secure Video  
**Compromis :** Écosystème propriétaire ; HomeBase doit rester alimenté ; 4K@15ips max  
**Prix :** ~550 $ (2 caméras + HomeBase)  
**Idéal Pour :** Propriétaires voulant une reconnaissance faciale zéro cloud + HomeKit

### 2. Reolink Argus 4 Pro / RLC-1212A (FW 3.0+) — Meilleure Intégration NVR Locale

**NPU :** Côté caméra (Ambarella CV25)  
**Base de données :** 100 visages par caméra (synchronisé via NVR)  
**Reconnaissance :** 1:1 local ; NVR agrège entre caméras  
**Vie Privée :** Local uniquement ; le NVR stocke les empreintes, pas les images (configurable)  
**Fonctionnalités :** Personne/véhicule/animal ; zones de détection personnalisées ; ONVIF Profile S/G/T  
**Compromis :** UX de l'application moins soignée ; NVR requis pour la synchronisation faciale multi-caméra ; pas de HomeKit  
**Prix :** ~130–230 $/caméra + NVR  
**Idéal Pour :** Bricoleurs avec NVR Reolink, flottes mixtes PoE/sans fil

### 3. Ubiquiti UniFi Protect (G4/G5 + AI Port) — Meilleur Prosumer/Petite Entreprise

**NPU :** Caméra (G4/G5) ou AI Port (ajoute à toute caméra ONVIF)  
**Base de données :** 200 visages par console Protect  
**Reconnaissance :** 1:1 local ; classification "Connu/Inconnu" ; recherche par vignette faciale  
**Vie Privée :** Entièrement local ; console Protect sur site ; pas de compte cloud requis pour la reconnaissance faciale  
**Fonctionnalités :** Personne/véhicule/animal ; plaque d'immatriculation (LPR) ; comptage d'occupation ; MQTT/webhooks  
**Compromis :** Verrouillage d'écosystème UniFi ; AI Port = 199 $ supplémentaire ; console matérielle nécessaire  
**Prix :** G4 Bullet 230 $, G5 Bullet 330 $, AI Port 199 $, Cloud Key Gen2+ 300 $  
**Idéal Pour :** Magasins réseau Ubiquiti, multi-site, propriétaires à l'aise en IT

### 4. Rhombus R2/R360 + Console — Meilleur Entreprise (Sans Dépendance Cloud)

**NPU :** Caméra (Ambarella CV52) + Console (Intel)  
**Base de données :** 10 000+ visages par console  
**Reconnaissance :** 1:1 local ; listes de surveillance ; horodatage "première vue" ; export API/SIEM  
**Vie Privée :** **Zéro cloud pour la reconnaissance** ; SOC 2 Type II ; conforme NDAA ; sur site uniquement  
**Fonctionnalités :** Détection de filature ; détection de masque ; occupation ; LPR ; analyse audio  
**Compromis :** Modèle d'abonnement matériel (10–20 $/cam/mois inclut garantie) ; pas d'application grand public  
**Prix :** Matériel 200–500 $/caméra + abonnement  
**Idéal Pour :** Écoles, santé, chaînes de vente au détail, gouvernement — conformité d'abord

### 5. Ring Stick Up Cam Pro / Floodlight Cam Pro — Meilleure Intégration d'Écosystème

**NPU :** Caméra (Amazon AZ2)  
**Base de données :** 100 visages par compte (synchronisé entre caméras Ring)  
**Reconnaissance :** 1:1 inférence locale ; synchronisation cloud des empreintes (chiffré)  
**Vie Privée :** **Synchro cloud par défaut** — désactivation dans le Centre de contrôle ; Ring Protect requis (4–20 $/mois) pour l'enregistrement  
**Fonctionnalités :** Détection de mouvement 3D (radar) ; vue à vol d'oiseau ; annonces Alexa ("John à la porte d'entrée")  
**Compromis :** Abonnement obligatoire pour les enregistrements ; synchro cloud faciale = surface d'attaque plus large ; pas d'ONVIF  
**Prix :** 180–250 $/caméra + forfait Protect  
**Idéal Pour :** Foyers Alexa, écosystème Ring, utilisateurs acceptant le compromis cloud

## Précision et Biais : Ce Que Les Chiffres Signifient

**NIST FRVT (Face Recognition Vendor Test) — Résultats 2024/2025 :**

- SDK commerciaux de premier plan (Cognitec, NEC, Rank One, Paravision, Idemia) : **FNIR &lt;0,5 % à FPIR 0,1 %** (vérification 1:1)
- **Les différences démographiques persistent :** Taux de non-correspondance faux 2–10× plus élevés pour les femmes à peau foncée vs hommes à peau claire sur certains algorithmes
- **Performance des masques :** Les algorithmes 2024+ récupèrent à &lt;2 % FNIR avec masques chirurgicaux ; N95 encore 5–15 %
- **Vieillissement :** Écart de 5 ans → augmentation FNIR de 3–8 % ; réinscription annuelle pour haute sécurité

**Réalité des Caméras Grand Public :**

- Eufy/Reolink/UniFi utilisent des variantes MobileFaceNet/ArcFace embarquées — **pas les meilleures du NIST**
- Attendez-vous à **5–15 % FNIR** (visages connus manqués) à des seuils raisonnables
- **Faux positifs (inconnu = connu) :** 1–3 % avec base de 100 visages ; augmente avec la taille de la base
- **Atténuation :** Réglez le seuil de manière conservatrice (alerte sur "inconnu" pas "connu") ; utilisez "visage connu" comme signal _supplémentaire_, pas seul déclencheur

## Usurpation et Vivacité : Ne Faites Pas Confiance au Seul Visage pour l'Accès

**Règle :** Reconnaissance faciale = **notification/identification**. Visage + PIN / Visage + Carte / Visage + QR = **contrôle d'accès**. Jamais un seul facteur visage pour les portes/portails.

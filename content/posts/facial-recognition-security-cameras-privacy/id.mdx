Pengenalan wajah pada kamera keamanan telah melampaui "pemicu gerakan" — sekarang dapat membedakan anggota keluarga, tetangga, dan pengirim paket dengan akurasi >99%. Namun akurasi saja tidak berarti apa-apa. Undang-undang privasi, pelatihan bias, database cloud, dan konsumsi daya membuatnya rumit.

<Badge variant="outline">Peringatan</Badge> Negara bagian AS (CA, IL, MA, MD,
WA, VT) + UE (GDPR) memiliki undang-undang pengenalan wajah yang ketat. Periksa
hukum setempat sebelum mengaktifkan pengenalan wajah. Penggunaan yang tidak sah
= tuntutan hukum, denda, bukti ditolak.

## Cara Kerja Pengenalan Wajah Kamera Keamanan

### Alur Pemrosesan

```
1. Kamera menangkap bingkai (2K/4K)
2. Deteksi: AI mendeteksi "wajah" di bingkai
3. Ekstraksi: 128 titik landmark wajah (jarak mata, bentuk rahang, kontur bibir)
4. Enkode: Titik landmark → vektor numerik (sidik jari wajah)
5. Pencocokan: Vektor dibandingkan dengan database sidik jari wajah yang dikenal
6. Tindakan: Jika cocok → "John terdeteksi". Jika tidak → "Orang asing terdeteksi."
```

### Lokal vs Cloud

| Metode Pemrosesan  | Kecepatan     | Privasi | Diperlukan Koneksi Internet | Akurasi | Contoh Perangkat Keras                                                   |
| ------------------ | ------------- | ------- | --------------------------- | ------- | ------------------------------------------------------------------------ |
| **Cloud**          | 1–3 detik     | ❌      | ✅                          | 99,9%   | Ring, Nest, Arlo, Wyze                                                   |
| **Lokal (NVR)**    | 0,5–1 detik   | ✅      | ❌                          | 99,5%   | Dahua WizSense, Hikvision AcuSense, UniFi Protect AI, Reolink (terbatas) |
| **Lokal (Kamera)** | 0,1–0,5 detik | ✅      | ❌                          | 98%     | Kamera AI dengan NPU (Ambarella, HiSilicon, Rockchip)                    |

<Callout type="info">

**Pemrosesan lokal = privasi maksimal.** Wajah tidak pernah meninggalkan rumah Anda. Tidak ada database cloud. Wajah tidak digunakan untuk melatih sistem orang lain.

</Callout>

## Privasi & Hukum

### Hukum AS (per 2026)

| Negara Bagian     | Status Hukum Pengenalan Wajah                                                                                                                                                                                         |
| ----------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Illinois**      | **BIPA** — Undang-undang privasi biometrik paling ketat. Persetujuan tertulis diperlukan untuk mengumpulkan/menyimpan data wajah. Denda $1.000–$5.000 per pelanggaran. **Kamera pengenalan wajah = berisiko tinggi.** |
| **California**    | CCPA + CCR — Data biometrik = informasi pribadi. Wajib memberi tahu.                                                                                                                                                  |
| **Massachusetts** | Undang-undang pengenalan wajah — larangan penggunaan oleh lembaga pemerintah (tidak termasuk kamera pribadi). Pedoman kepemilikan.                                                                                    |
| **Maryland**      | **Peringatan** — Undang-undang penggunaan pengenalan wajah oleh penegak hukum — kamera pribadi di area bersama? Abu-abu.                                                                                              |
| **Vermont**       | Hukum paling ketat — secara efektif melarang penggunaan komersial pengenalan wajah. Kamera pribadi? Risiko.                                                                                                           |
| **Washington**    | Undang-undang privasi biometrik — persetujuan diperlukan, hak untuk menuntut secara pribadi.                                                                                                                          |

### Hukum UE (GDPR)

**GDPR: Data biometrik = data sensitif.** Diperlukan persetujuan eksplisit. Konsesi penggunaan rumah tangga ("pengecualian rumah tangga") — pengenalan wajah pribadi di rumah = tidak terkena GDPR. Namun jika kamera merekam ke area publik (trotoar, properti tetangga) → terkena GDPR.

## Batasan & Pertimbangan

| Faktor                  | Dampak pada Pengenalan Wajah                                                                       |
| ----------------------- | -------------------------------------------------------------------------------------------------- |
| **Pencahayaan**         | Cahaya redup = akurasi turun 10–20%. Gunakan IR atau lampu sorot.                                  |
| **Sudut**               | >30° dari depan = akurasi turun 10–30%. Posisikan kamera untuk tampilan depan.                     |
| **Jarak**               | >25 kaki = wajah terlalu kecil untuk dikenali. Gunakan kamera varifokal / zoom.                    |
| **Masker**              | Masker menutupi mulut/hidung = akurasi turun 50%+. Beberapa sistem memiliki mode "setengah wajah". |
| **Kacamata hitam**      | Menutupi mata = turun 20–40%. Silau lensa = tidak ada deteksi.                                     |
| **Topi / tudung**       | Menutupi dahi = turun 30–50%. Pengenalan wajah tidak dapat diandalkan dengan tudung.               |
| **Pertumbuhan jenggot** | Perubahan penampilan = basa data perlu diperbarui secara teratur.                                  |

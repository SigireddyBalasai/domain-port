ワイヤレスカメラは「配線不要、簡単設置」を謳っています。現実は：デッドゾーン、バッテリー不安、2.4 GHzの混雑、冬に充電しないソーラーパネル。このガイドはワイヤレスをイライラの種から信頼できるものに変えます。

<Badge variant="outline">ワイヤレス ≠ 配線不要</Badge>
**プラグインワイヤレス**（Reolink E1、Wyze v4、TP-Link
Tapo）は120Vコンセントが必要。**バッテリー/ソーラー**（EufyCam 3、Reolink Argus
4 Pro、Ring Stick
Up）は真の配線不要。どちらもWi-Fiを使用し、どちらもしっかりしたRF計画が必要です。

---

## Wi-Fiの物理: カメラが切断される理由

### 周波数帯

| 帯域                   | チャンネル         | 最大速度      | 浸透性 | 範囲    | 最適                   |
| ---------------------- | ------------------ | ------------- | ------ | ------- | ---------------------- |
| **2.4 GHz**            | 1, 6, 11（非重複） | 600 Mbps (ax) | ★★★★★  | 150+ ft | **すべての屋外カメラ** |
| **5 GHz**              | 36–165             | 2.4 Gbps (ax) | ★★☆☆☆  | 50 ft   | 屋内、短距離           |
| **6 GHz (Wi-Fi 6E/7)** | 1–233              | 5+ Gbps       | ★☆☆☆☆  | 30 ft   | 将来性確保、屋内のみ   |

**ルール:** カメラは**2.4 GHzのみ**に強制。5/6 GHzは壁/サイディングを透過しません。ほとんどのカメラは6 GHzに対応していません。

### チャンネル計画（2.4 GHz）

```
チャンネル:  1       6       11
周波数:   2412    2437    2462  MHz
幅:  20 MHz each (non-overlapping)
```

**まずスキャン:** **WiFi Analyzer（Android）** または **AirPort Utility（iOS）** を使用 → スキャン → 近隣のチャンネルを確認。

**割り当て:**

- ルーター/AP: チャンネル1、6、または11（最も混雑していないもの）
- カメラAP（別の場合）: 異なる非重複チャンネル
- **幅: 20 MHzのみ**（40 MHzは隣接チャンネルと重複し、再送が発生）

## ネットワークアーキテクチャ: メッシュ vs エクステンダー vs 専用AP

| アーキテクチャ                                                | 長所                                        | 短所                                             | 最適                                |
| ------------------------------------------------------------- | ------------------------------------------- | ------------------------------------------------ | ----------------------------------- |
| **メッシュ（Eero Pro 6E、UniFi Dream Router、Asus ZenWiFi）** | シームレスローミング、単一SSID、アプリ管理  | コスト（$300–600）、全トラフィックがメッシュ経由 | 家中全体、5台以上のカメラ           |
| **有線AP（UniFi U6 Lite、U6+、U7 Pro）**                      | 最高のパフォーマンス、PoE給電、VLANサポート | Ethernet配線が必要                               | **カメラに最適** — 有線バックホール |
| **Powerline + Wi-Fi（TP-Link TL-WPA8630）**                   | 新しい配線不要、電気配線を使用              | 実効50–200 Mbps、AFCIでノイズが発生              | ケーブル配線が不可能なリフォーム    |
| **レンジエクステンダー（TP-Link RE605X）**                    | 安価（$50）                                 | 帯域幅が半減、遅延追加、別SSID                   | **カメラには避ける**                |
| **屋外用AP（UniFi U6 Mesh、UAP-AC-M）**                       | 耐候性、PoE、メッシュアップリンク           | コスト（$180–300）                               | 離れたガレージ、ゲート、納屋        |

<Callout type="tip">

**プロ向け構成:** PoEスイッチ → UniFi U6+ AP（天井、中央）→ 2.4 GHz専用SSID「CAM-2G」（VLAN 20）→ 全カメラ。IoTを分離、ローミング問題なし、有線バックホール。

</Callout>

## カメラのWi-Fi要件

| 仕様                   | 最小               | 推奨             | 理由                         |
| ---------------------- | ------------------ | ---------------- | ---------------------------- |
| **信号 (RSSI)**        | -70 dBm            | **-55 dBm 以上** | -70 = 頻繁な切断; -55 = 堅牢 |
| **SNR**                | 15 dB              | **25+ dB**       | 信号対ノイズマージン         |
| **アップロード帯域幅** | 1 Mbps/台（1080p） | 5 Mbps/台（4K）  | 持続的、バーストではない     |
| **遅延**               | &lt;100 ms         | &lt;30 ms        | ライブビューの応答性         |
| **パケットロス**       | &lt;2%             | &lt;0.5%         | 録画ギャップ、フレーム凍結   |

**カメラ設置場所でテスト:** 同じSSIDのスマホ → Speedtest → アップロードを記録。カメラ設置予定場所に移動 → 再テスト。アップロード&lt;5 Mbpsの場合は、先にWi-Fiを修正。

## メッシュ/APの配置（カメラ用）

<Accordion type="single" collapsible>
  <AccordionItem value="placement">
    <AccordionTrigger>AP配置ルール</AccordionTrigger>
    <AccordionContent>

- **中央、高所** — 廊下の天井取付 &gt; 棚の隅
- **見通し線** — 避ける: 煙突、レンガ壁、金属サイディング、HVAC、鏡、水槽
- **壁1枚まで** — カメラ ↔ AP の間は内壁1枚以内
- **隅を避ける** — RFが反射し、マルチパスが発生
- **高さ** — 地上7–9 ft AGL が最適なパターン
- **アンテナ方向** — 垂直で水平カバレッジ; 複数階の場合は45°傾ける

**UniFi U6+例:** 廊下中央の天井取付 → 2.4 GHzで3,000 sq ftの家＋50 ftの庭をカバー。カメラのRSSIは-50〜-60 dBm。

## バッテリーカメラの最適化

### ソーラー＋Wi-Fiのための配置

| 要素               | 理想的                                                     | 避ける                                             |
| ------------------ | ---------------------------------------------------------- | -------------------------------------------------- |
| **ソーラーパネル** | 真南（北半球）、30–45°傾斜、**12月21日9時–15時に日陰なし** | 北面、軒下、木の影                                 |
| **カメラ**         | パネルから10–15 ft（ケーブル届く範囲）、雨よけに軒下       | レンズに直射日光（過熱）、スプリンクラーの水しぶき |
| **Wi-Fi**          | APから50 ft以内、壁1枚まで                                 | 間の金属サイディング、レンガ煙突                   |

### バッテリー寿命延長

| 設定               | デフォルト   | 最適化                           | 節約効果 |
| ------------------ | ------------ | -------------------------------- | -------- |
| **動体感度**       | 高           | 中 / カスタムゾーン              | 30–50%   |
| **クリップ長**     | 30秒         | 10–15秒                          | 40–60%   |
| **再トリガー間隔** | 10秒         | 30–60秒                          | 20–30%   |
| **暗視**           | オートIR     | スケジュール: 街灯があればIRオフ | 15–25%   |
| **ライブビュー**   | オンデマンド | 自動プレビューを無効化           | 10–20%   |
| **ファームウェア** | 自動         | 手動（月1回確認）                | バグ防止 |

**実績:** EufyCam 3（13,000 mAh）@ 中感度、15秒クリップ、30秒再トリガー、2.4 GHz -55 dBm → **4–6ヶ月**（公称1年に対し）。

### 冬のバッテリーケア（ゾーン5以上）

- **リチウムイオンは&lt;32°F（0°C）で充電停止。** カメラはバッテリーが切れるまで動作。
- **修正方法:**
  - 月に1回屋内に持ち込んで温かいうちに充電（面倒だが効果的）
  - **LiFePO₄ アップグレード**（Reolink Go PT Plus、カスタム）— -4°Fまで充電可能
  - **断熱バッテリーボックス** + 5Wヒーターパッド（サーモスタット40°F）— DIY
  - **大型パネル**（6W→20W）— 冬の収穫量増加

## プラグインワイヤレスカメラの設定

### 電源の考慮事項

| カメラ                  | 消費電力       | コンセントタイプ     | ケーブル管理               |
| ----------------------- | -------------- | -------------------- | -------------------------- |
| Reolink E1 Outdoor      | 5W（12V/0.5A） | 120V → 12Vアダプター | 耐候性ボックス＋グロメット |
| Wyze Cam v4             | 3W（5V/0.6A）  | 120V → 5V USB        | USB延長＋シリコンシール    |
| TP-Link Tapo C120       | 4W（5V/0.8A）  | 120V → 5V USB        | 短いUSB、ドリップループ    |
| Google Nest Cam（有線） | 7W（5V/1.4A）  | 120V → 5V専用        | 専用ケーブルのみ           |

**ドリップループ:** ケーブルは**コンセントの下でU字**を形成 → 水はケーブルを伝い、ループから滴下し、コンセントに入らない。

### GFCIとコード

- **屋外コンセントはGFCI必須**（NEC 210.8(A)(3)）
- **耐候性カバー**（使用中/バブルカバー）— 湿気のある場所で必要
- **延長コード** — **絶対に**恒久的に使用しない。適切な屋外用ケーブル＋ボックスを使用。

## ワイヤレスカメラの問題トラブルシューティング

| 症状                          | 診断                            | 修正                                                           |
| ----------------------------- | ------------------------------- | -------------------------------------------------------------- |
| **毎日「カメラオフライン」**  | RSSI < -75 dBm                  | APを近くに移動; 有線APを追加; チャンネル確認                   |
| **ライブビューが回る/失敗**   | アップロード < 1 Mbps           | Speedtestをカメラ位置で実行; APをアップグレード                |
| **夜間にクリップ欠落**        | IR反射 / 白飛び                 | 15°下向き; 壁から離す; IRを無効＋外部照明器                    |
| **バッテリーが2週間で切れる** | 高トラフィックゾーン / RSSI不良 | 感度を下げる; ゾーンを狭く; ソーラー追加; -60 dBm確認          |
| **ライブビューの遅延 > 5秒**  | 高遅延 / バッファブロート       | ルーターでQoS（カメラVLANを優先）; 有線APバックホール          |
| **停電後に「Wi-Fiなし」**     | カメラがルーターより先に起動    | ルーター＋APにUPS; カメラの自動再接続（ファームウェア確認）    |
| **2.4 GHzに接続できない**     | ルーターのバンドステアリング    | バンドステアリングを無効; SSIDを分離（「HOME-2G」「HOME-5G」） |

## セキュリティ: カメラを隔離する

<Callout type="danger">

**カメラはIoT — 最も侵害リスクが高い。** **インターネットアクセスなし**のVLANに隔離（NTP/ファームウェア以外）。カメラ→LANをブロック。LAN→カメラ（表示のみ）を許可。

</Callout>

**UniFi / VLAN例:**

```
VLAN 10 (LAN)     192.168.10.x  → フルアクセス
VLAN 20 (CAMERAS) 192.168.20.x  → インターネットブロック、LANアクセスブロック
                    許可: NTP (udp/123)、DNS (udp/53) からルーターへ
                    許可: NVR (192.168.10.50) → カメラ (tcp/554、8000)
```

**ルーターファイアウォールルール（Ubiquiti / OPNsense / pfSense）:**

- **許可** CAMERAS → ルーター（NTP、DNS）
- **許可** NVR → CAMERAS（RTSP、HTTP、ONVIF）
- **ブロック** CAMERAS → LAN
- **ブロック** CAMERAS → WAN（インターネット）
- **許可** LAN → CAMERAS（管理のみ、制限付きIP）

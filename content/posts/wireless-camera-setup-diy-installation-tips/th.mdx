กล้องไร้สายสัญญาว่า "ไร้สาย ติดตั้งง่าย" ความเป็นจริง: จุดอับสัญญาณ ความกังวลเรื่องแบตเตอรี่ คลื่น 2.4 GHz แออัด และแผงโซลาร์เซลล์ที่ชาร์จไม่เต็มในฤดูหนาว คู่มือนี้เปลี่ยนกล้องไร้สายจากที่น่าหงุดหงิดเป็นเชื่อถือได้

<Badge variant="outline">ไร้สาย ≠ ไม่ใช้สาย</Badge> **ไร้สายแบบเสียบปลั๊ก**
(Reolink E1, Wyze v4, TP-Link Tapo) ต้องใช้เต้าเสียบ 120V **แบตเตอรี่/โซลาร์**
(EufyCam 3, Reolink Argus 4 Pro, Ring Stick Up) ไร้สายอย่างแท้จริง ทั้งคู่ใช้
Wi-Fi ทั้งคู่ต้องการการวางแผน RF ที่ดี

## ฟิสิกส์ Wi-Fi: ทำไมกล้องของคุณตัดการเชื่อมต่อ

### ย่านความถี่

| แบนด์                  | ช่องสัญญาณ            | ความเร็วสูงสุด | การทะลุทะลวง | ระยะ   | เหมาะที่สุด                 |
| ---------------------- | --------------------- | -------------- | ------------ | ------ | --------------------------- |
| **2.4 GHz**            | 1, 6, 11 (ไม่ทับซ้อน) | 600 Mbps (ax)  | ★★★★★        | 45+ ม. | **กล้องกลางแจ้งทั้งหมด**    |
| **5 GHz**              | 36–165                | 2.4 Gbps (ax)  | ★★☆☆☆        | 15 ม.  | ในร่ม, ระยะสั้น             |
| **6 GHz (Wi-Fi 6E/7)** | 1–233                 | 5+ Gbps        | ★☆☆☆☆        | 9 ม.   | ป้องกันอนาคต, ในร่มเท่านั้น |

**กฎ:** บังคับกล้องเป็น **2.4 GHz เท่านั้น** 5/6 GHz ไม่ทะลุผนัง/ฝาผนัง กล้องส่วนใหญ่ไม่รองรับ 6 GHz อยู่แล้ว

### การวางแผนช่องสัญญาณ (2.4 GHz)

```
ช่องสัญญาณ:  1       6       11
ความถี่:   2412    2437    2462  MHz
ความกว้าง: 20 MHz ต่อช่อง (ไม่ทับซ้อน)
```

**สแกนก่อน:** ใช้ **WiFi Analyzer (Android)** หรือ **AirPort Utility (iOS)** → สแกน → จดช่องสัญญาณเพื่อนบ้าน

**กำหนด:**

- เราเตอร์/AP ของคุณ: ช่องสัญญาณ 1, 6 หรือ 11 (ช่องที่แออัดน้อยที่สุด)
- AP ของกล้อง (ถ้าแยก): ช่องสัญญาณไม่ทับซ้อนอื่น
- **ความกว้าง: 20 MHz เท่านั้น** (40 MHz ทับซ้อนเพื่อนบ้าน, ทำให้ต้องส่งซ้ำ)

## สถาปัตยกรรมเครือข่าย: Mesh vs Extender vs AP เฉพาะ

| สถาปัตยกรรม                                              | ข้อดี                                       | ข้อเสีย                                  | เหมาะที่สุด                            |
| -------------------------------------------------------- | ------------------------------------------- | ---------------------------------------- | -------------------------------------- |
| **Mesh (Eero Pro 6E, UniFi Dream Router, Asus ZenWiFi)** | โรมมิ่งไร้รอยต่อ, SSID เดียว, จัดการผ่านแอป | ราคา ($300–600), ทราฟฟิกทั้งหมดผ่าน mesh | ทั้งบ้าน, 5+ กล้อง                     |
| **AP แบบมีสาย (UniFi U6 Lite, U6+, U7 Pro)**             | ประสิทธิภาพดีที่สุด, ไฟ PoE, รองรับ VLAN    | ต้องใช้สาย Ethernet                      | **ดีที่สุดสำหรับกล้อง** — backhaul สาย |
| **Powerline + Wi-Fi (TP-Link TL-WPA8630)**               | ไม่ต้องเดินสายใหม่, ใช้สายไฟ                | 50–200 Mbps จริง, มีสัญญาณรบกวนใน AFCI   | ปรับปรุงใหม่ที่เดินสายไม่ได้           |
| **ตัวขยายสัญญาณ (TP-Link RE605X)**                       | ถูก ($50)                                   | ลดแบนด์วิดท์ครึ่ง, เพิ่มหน่วง, SSID แยก  | **หลีกเลี่ยงสำหรับกล้อง**              |
| **AP กลางแจ้ง (UniFi U6 Mesh, UAP-AC-M)**                | กันน้ำ, PoE, mesh uplink                    | ราคา ($180–300)                          | โรงรถแยก, ประตูรั้ว, ยุ้งฉาง           |

<Callout type="tip">

**สถาปัตยกรรมมือโปร:** PoE Switch → AP UniFi U6+ (เพดาน, กลาง) → 2.4 GHz SSID
เฉพาะ "CAM-2G" (VLAN 20) → กล้องทั้งหมด แยก IoT, ไม่มีปัญหา โรมมิ่ง, backhaul
แบบมีสาย

</Callout>

## ข้อกำหนด Wi-Fi ของกล้อง

| สเปก                   | ขั้นต่ำ              | แนะนำ                  | เหตุผล                          |
| ---------------------- | -------------------- | ---------------------- | ------------------------------- |
| **สัญญาณ (RSSI)**      | -70 dBm              | **-55 dBm หรือดีกว่า** | -70 = หลุดบ่อย; -55 = เสถียรมาก |
| **SNR**                | 15 dB                | **25+ dB**             | อัตราส่วนสัญญาณต่อสัญญาณรบกวน   |
| **แบนด์วิดท์อัปโหลด**  | 1 Mbps/กล้อง (1080p) | 5 Mbps/กล้อง (4K)      | ต่อเนื่อง ไม่ใช่เป็นช่วง        |
| **ความหน่วง**          | &lt;100 ms           | &lt;30 ms              | การตอบสนองการดูสด               |
| **การสูญเสียแพ็กเก็ต** | &lt;2%               | &lt;0.5%               | ช่องว่างการบันทึก, เฟรมค้าง     |

**ทดสอบที่ตำแหน่งกล้อง:** โทรศัพท์บน SSID เดียวกัน → Speedtest → จดอัปโหลด เดินไปที่จุดกล้อง → ทดสอบอีกครั้ง ถ้าอัปโหลด &lt;5 Mbps, แก้ไข Wi-Fi ก่อน

- **กลาง, สูง** — ติดตั้งบนเพดานในโถงทางเดิน > ชั้นวางมุม

## การเพิ่มประสิทธิภาพกล้องแบตเตอรี่

### การวางตำแหน่งสำหรับโซลาร์ + Wi-Fi

| ปัจจัย             | เหมาะสม                                                              | หลีกเลี่ยง                                |
| ------------------ | -------------------------------------------------------------------- | ----------------------------------------- |
| **แผงโซลาร์เซลล์** | ทิศใต้จริง (NH), เอียง 30–45°, **ไม่มีร่มเงา 9:00–15:00 น. 21 ธ.ค.** | ด้านเหนือ, ใต้ชายคา, เงาต้นไม้            |
| **กล้อง**          | ห่างจากแผง 3–4.5 ม. (ระยะสาย), ใต้ชายคากันฝน                         | แดดตรงเลนส์ (ร้อนเกิน), น้ำจากสปริงเกลอร์ |
| **Wi-Fi**          | ≤15 ม. จาก AP, ผนังสูงสุด 1 ชั้น                                     | ฝาผนังโลหะ, ปล่องไฟอิฐระหว่าง             |

### การยืดอายุแบตเตอรี่

| การตั้งค่า               | ค่าเริ่มต้น  | ปรับปรุง                    | ประหยัด     |
| ------------------------ | ------------ | --------------------------- | ----------- |
| **ความไวการเคลื่อนไหว**  | สูง          | ปานกลาง / โซนกำหนดเอง       | 30–50%      |
| **ความยาวคลิป**          | 30s          | 10–15s                      | 40–60%      |
| **ระยะเวลากระตุ้นซ้ำ**   | 10s          | 30–60s                      | 20–30%      |
| **การมองเห็นตอนกลางคืน** | IR อัตโนมัติ | กำหนดการ: ปิด IR ถ้ามีไฟถนน | 15–25%      |
| **ดูสด**                 | ตามต้องการ   | ปิดตัวอย่างอัตโนมัติ        | 10–20%      |
| **เฟิร์มแวร์**           | อัตโนมัติ    | ด้วยตนเอง (ตรวจสอบรายเดือน) | ป้องกันบั๊ก |

**โลกจริง:** EufyCam 3 (13,000 mAh) @ ความไวปานกลาง, คลิป 15s, กระตุ้นซ้ำ 30s, 2.4 GHz -55 dBm → **4–6 เดือน** (vs อ้าง 1 ปี)

### การดูแลแบตเตอรี่ในฤดูหนาว (โซน 5 และเย็นกว่า)

- **Li-ion หยุดชาร์จ &lt;0°C** กล้องทำงานด้วยแบตเตอรี่จนหมด
- **วิธีแก้:**
  - นำเข้าในบ้านทุกเดือนเพื่อชาร์จอุ่น (ลำบากแต่ได้ผล)
  - **อัปเกรด LiFePO₄** (Reolink Go PT Plus, แบบกำหนดเอง) — ชาร์จได้ถึง -20°C
  - **กล่องแบตเตอรี่หุ้มฉนวน** + แผ่นทำความร้อน 5W (เทอร์โมสตัท 4°C) — DIY
  - **แผงใหญ่ขึ้น** (20W vs 6W) — เก็บเกี่ยวในฤดูหนาวมากขึ้น

## การตั้งค่ากล้องไร้สายแบบเสียบปลั๊ก

### ข้อควรพิจารณาด้านไฟ

| กล้อง                      | การใช้ไฟ      | ประเภทเต้าเสียบ       | การจัดการสาย                 |
| -------------------------- | ------------- | --------------------- | ---------------------------- |
| Reolink E1 Outdoor         | 5W (12V/0.5A) | 120V → อะแดปเตอร์ 12V | กล่องกันน้ำ + grommet        |
| Wyze Cam v4                | 3W (5V/0.6A)  | 120V → 5V USB         | สายต่อ USB + น้ำยาซีลซิลิโคน |

**ห่วงหยดน้ำ:** สายเป็น **รูปตัว U ใต้เต้าเสียบ** → น้ำหยดจากห่วง, ไม่เข้าปลั๊ก

### GFCI และรหัส

- **เต้าเสียบกลางแจ้งต้องเป็น GFCI** (NEC 210.8(A)(3))
- **ฝาครอบกันน้ำ** (in-use / bubble cover) — จำเป็นสำหรับพื้นที่ชื้น
- **สายต่อพ่วง** — **ห้าม** ถาวร ใช้สายเกรดกลางแจ้ง + กล่องที่เหมาะสม

## การแก้ปัญหากล้องไร้สาย

| อาการ                        | การวินิจฉัย                | การแก้ไข                                                            |
| ---------------------------- | -------------------------- | ------------------------------------------------------------------- |
| **"กล้องออฟไลน์" ทุกวัน**    | RSSI < -75 dBm             | ย้าย AP ใกล้ขึ้น; เพิ่ม AP แบบมีสาย; ตรวจสอบช่องสัญญาณ              |
| **ดูสดหมุน/ล้มเหลว**         | อัปโหลด < 1 Mbps           | ทดสอบ Speedtest ที่กล้อง; อัปเกรด AP                                |
| **คลิปหายตอนกลางคืน**        | การสะท้อน IR / ขาวโพลน     | เอียงลง 15°; ขยับออกจากผนัง; ปิด IR + อิลลูมิเนเตอร์ภายนอก          |
| **แบตเตอรี่หมดใน 2 สัปดาห์** | โซนทราฟฟิกสูง / RSSI ไม่ดี | ลดความไว; แคบโซน; เพิ่มโซลาร์; ตรวจสอบ -60 dBm                      |
| **หน่วงดูสด > 5 วินาที**     | หน่วงสูง / bufferbloat     | QoS บนเราเตอร์ (จัดลำดับความสำคัญ VLAN กล้อง); backhaul AP แบบมีสาย |
| **"ไม่มี Wi-Fi" หลังไฟดับ**  | กล้องบูตก่อนเราเตอร์       | UPS บนเราเตอร์ + AP; การเชื่อมต่ออัตโนมัติกล้อง (ตรวจสอบเฟิร์มแวร์) |
| **เชื่อมต่อ 2.4 GHz ไม่ได้** | Band-steering ของเราเตอร์  | ปิด band-steering; SSID แยก ("HOME-2G", "HOME-5G")                  |

## ความปลอดภัย: แยกกล้องของคุณ

<Callout type="danger">

**กล้องคือ IoT — ความเสี่ยง breach สูงสุด** แยกบน VLAN โดย **ไม่มีการเข้าถึง
อินเทอร์เน็ต** (ยกเว้น NTP/เฟิร์มแวร์) บล็อกกล้อง → LAN อนุญาต LAN → กล้อง (ดู
เท่านั้น)

</Callout>

**ตัวอย่าง UniFi / VLAN:**

```
VLAN 10 (LAN)     192.168.10.x  → เข้าถึงทั้งหมด
VLAN 20 (กล้อง) 192.168.20.x  → บล็อกอินเทอร์เน็ต, บล็อกการเข้าถึง LAN
                    อนุญาต: NTP (udp/123), DNS (udp/53) ไปยังเราเตอร์
                    อนุญาต: NVR (192.168.10.50) → กล้อง (tcp/554, 8000)
```

**กฎไฟร์วอลล์เราเตอร์ (Ubiquiti / OPNsense / pfSense):**

- **อนุญาต** กล้อง → เราเตอร์ (NTP, DNS)
- **อนุญาต** NVR → กล้อง (RTSP, HTTP, ONVIF)
- **บล็อก** กล้อง → LAN
- **บล็อก** กล้อง → WAN (อินเทอร์เน็ต)
- **อนุญาต** LAN → กล้อง (จัดการเท่านั้น, IP ที่จำกัด)

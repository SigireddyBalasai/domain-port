Kamera nirkabel menjanjikan "tanpa kabel, mudah dipasang." Kenyataan: zona mati, kecemasan baterai, kemacetan 2,4 GHz, dan panel surya yang tidak mengisi daya di musim dingin. Panduan ini mengubah nirkabel dari frustasi menjadi andal.

<Badge variant="outline">Nirkabel ≠ Tanpa Kabel</Badge> **Nirkabel colokan**
(Reolink E1, Wyze v4, TP-Link Tapo) memerlukan stopkontak 120V.
**Baterai/Surya** (EufyCam 3, Reolink Argus 4 Pro, Ring Stick Up) benar-benar
tanpa kabel. Keduanya menggunakan Wi-Fi. Keduanya memerlukan perencanaan RF yang
matang.

## Fisika Wi-Fi: Mengapa Kamera Anda Terputus

### Pita Frekuensi

| Pita                   | Saluran                         | Kecepatan Maks | Penetrasi | Jangkauan | Terbaik Untuk                    |
| ---------------------- | ------------------------------- | -------------- | --------- | --------- | -------------------------------- |
| **2,4 GHz**            | 1, 6, 11 (tidak tumpang tindih) | 600 Mbps (ax)  | ★★★★★     | 150+ kaki | **Semua kamera luar ruangan**    |
| **5 GHz**              | 36–165                          | 2,4 Gbps (ax)  | ★★☆☆☆     | 50 kaki   | Dalam ruangan, jarak pendek      |
| **6 GHz (Wi-Fi 6E/7)** | 1–233                           | 5+ Gbps        | ★☆☆☆☆     | 30 kaki   | Future-proof, dalam ruangan saja |

**Aturan:** Paksa kamera ke **2,4 GHz saja**. 5/6 GHz tidak menembus dinding/kelongsong. Kebanyakan kamera tidak mendukung 6 GHz.

### Perencanaan Saluran (2,4 GHz)

```
Saluran:  1       6       11
Frekuensi:   2412    2437    2462  MHz
Lebar:  20 MHz masing-masing (tidak tumpang tindih)
```

**Pindai dulu:** Gunakan **WiFi Analyzer (Android)** atau **AirPort Utility (iOS)** → Pindai → Catat saluran tetangga.

**Tetapkan:**

- Router/AP Anda: Saluran 1, 6, atau 11 (yang paling tidak padat)
- AP kamera (jika terpisah): Saluran tidak tumpang tindih yang berbeda
- **Lebar: hanya 20 MHz** (40 MHz tumpang tindih dengan tetangga, menyebabkan pengiriman ulang)

## Arsitektur Jaringan: Mesh vs Extender vs AP Khusus

| Arsitektur                                               | Pro                                             | Kontra                                            | Terbaik Untuk                                |
| -------------------------------------------------------- | ----------------------------------------------- | ------------------------------------------------- | -------------------------------------------- |
| **Mesh (Eero Pro 6E, UniFi Dream Router, Asus ZenWiFi)** | Roaming mulus, SSID tunggal, manajemen aplikasi | Biaya ($300–600), semua lalu lintas melalui mesh  | Seluruh rumah, 5+ kamera                     |
| **AP Berkabel (UniFi U6 Lite, U6+, U7 Pro)**             | Kinerja terbaik, ditenagai PoE, dukungan VLAN   | Memerlukan kabel Ethernet                         | **Terbaik untuk kamera** — backhaul berkabel |
| **Powerline + Wi-Fi (TP-Link TL-WPA8630)**               | Tidak ada kabel baru, menggunakan kabel listrik | 50–200 Mbps nyata, berisik pada AFCI              | Retrofit di mana kabel tidak memungkinkan    |
| **Range Extender (TP-Link RE605X)**                      | Murah ($50)                                     | Setengah bandwidth, tambah latensi, SSID terpisah | **Hindari untuk kamera**                     |
| **AP Luar Ruangan (UniFi U6 Mesh, UAP-AC-M)**            | Tahan cuaca, PoE, uplink mesh                   | Biaya ($180–300)                                  | Garasi terpisah, gerbang, lumbung            |

<Callout type="tip">

**Arsitektur Pro:** Switch PoE → AP UniFi U6+ (langit-langit, sentral) → 2,4 GHz
SSID khusus "CAM-2G" (VLAN 20) → Semua kamera. Mengisolasi IoT, tanpa masalah
roaming, backhaul berkabel.

</Callout>

## Persyaratan Wi-Fi Kamera

| Spesifikasi          | Minimum            | Direkomendasikan            | Mengapa                                 |
| -------------------- | ------------------ | --------------------------- | --------------------------------------- |
| **Sinyal (RSSI)**    | -70 dBm            | **-55 dBm atau lebih baik** | -70 = sering putus; -55 = sangat stabil |
| **SNR**              | 15 dB              | **25+ dB**                  | Margin sinyal vs kebisingan             |
| **Bandwidth Unggah** | 1 Mbps/kam (1080p) | 5 Mbps/kam (4K)             | Berkelanjutan, bukan semburan           |
| **Latensi**          | &lt;100 ms         | &lt;30 ms                   | Responsivitas tampilan langsung         |
| **Kehilangan Paket** | &lt;2%             | &lt;0,5%                    | Kesenjangan rekaman, bingkai beku       |

**Uji di lokasi kamera:** Ponsel di SSID yang sama → Speedtest → Catat unggah. Berjalan ke titik kamera → uji ulang. Jika unggah &lt;5 Mbps, perbaiki Wi-Fi dulu.

## Penempatan Mesh / AP untuk Kamera

<Accordion type="single" collapsible>
  <AccordionItem value="placement">
    <AccordionTrigger>Aturan Penempatan AP</AccordionTrigger>
    <AccordionContent>

1. **Tengah, tinggi** — Pasang di langit-langit lorong > rak sudut
2. **Garis pandang** — Hindari: cerobong, dinding bata, kelongsong logam, HVAC, cermin, akuarium
3. **Maks satu dinding** — Kamera ↔ AP melalui ≤1 dinding interior
4. **Hindari sudut** — RF memantul, menciptakan multipath
5. **Tinggi** — 7–9 kaki AGL (di atas permukaan tanah) untuk pola terbaik
6. **Orientasi antena** — Vertikal untuk cakupan horizontal; miring 45° untuk multi-lantai

**Contoh UniFi U6+:** Pasang langit-langit di tengah lorong → 2,4 GHz mencakup rumah 3.000 kaki persegi + halaman 50 kaki. Kamera pada -50 hingga -60 dBm.

## Optimasi Kamera Baterai

### Penempatan untuk Surya + Wi-Fi

| Faktor          | Ideal                                                                           | Hindari                                                                  |
| --------------- | ------------------------------------------------------------------------------- | ------------------------------------------------------------------------ |
| **Panel Surya** | Benar selatan (NH), kemiringan 30–45°, **tanpa naungan 9:00–15:00 21 Desember** | Muka utara, di bawah atap, naungan pohon                                 |
| **Kamera**      | 10–15 kaki dari panel (jangkauan kabel), di bawah atap untuk hujan              | Sinar matahari langsung pada lensa (panas berlebih), semprotan sprinkler |
| **Wi-Fi**       | ≤50 kaki dari AP, maks 1 dinding                                                | Kelongsong logam, cerobong bata di antara                                |

### Perpanjangan Masa Pakai Baterai

| Pengaturan                | Default           | Dioptimalkan                            | Penghematan  |
| ------------------------- | ----------------- | --------------------------------------- | ------------ |
| **Sensitivitas Gerakan**  | Tinggi            | Sedang / Zona Kustom                    | 30–50%       |
| **Panjang Klip**          | 30s               | 10–15s                                  | 40–60%       |
| **Interval Pemicu Ulang** | 10s               | 30–60s                                  | 20–30%       |
| **Penglihatan Malam**     | IR Otomatis       | Jadwalkan: IR mati jika ada lampu jalan | 15–25%       |
| **Tampilan Langsung**     | Sesuai permintaan | Nonaktifkan pratinjau otomatis          | 10–20%       |
| **Firmware**              | Otomatis          | Manual (periksa bulanan)                | Mencegah bug |

**Dunia nyata:** EufyCam 3 (13.000 mAh) @ Sensitivitas Sedang, klip 15s, pemicu ulang 30s, 2,4 GHz -55 dBm → **4–6 bulan** (vs klaim 1 tahun).

### Perawatan Baterai Musim Dingin (Zona 5 dan lebih dingin)

- **Li-ion berhenti mengisi daya &lt;32°F (0°C).** Kamera berjalan dengan baterai hingga habis.
- **Perbaikan:**
  - Bawa masuk bulanan untuk isi daya hangat (merepotkan tapi berhasil)
  - **Upgrade LiFePO₄** (Reolink Go PT Plus, kustom) — mengisi daya hingga -4°F
  - **Kotak baterai terisolasi** + bantalan pemanas 5W (termostat 40°F) — DIY
  - **Panel lebih besar** (20W vs 6W) — panen musim dingin lebih banyak

## Pengaturan Kamera Nirkabel Colokan

### Pertimbangan Daya

| Kamera                     | Konsumsi Daya | Jenis Stopkontak      | Manajemen Kabel              |
| -------------------------- | ------------- | --------------------- | ---------------------------- |
| Reolink E1 Outdoor         | 5W (12V/0,5A) | 120V → adaptor 12V    | Kotak tahan air + grommet    |
| Wyze Cam v4                | 3W (5V/0,6A)  | 120V → 5V USB         | Ekstensi USB + segel silikon |
| TP-Link Tapo C120          | 4W (5V/0,8A)  | 120V → 5V USB         | USB pendek, loop tetes       |
| Google Nest Cam (berkabel) | 7W (5V/1,4A)  | 120V → 5V proprietary | Hanya kabel proprietary      |

**Loop Tetes:** Kabel membentuk **bentuk U di bawah stopkontak** → air menetes dari loop, bukan ke stopkontak.

### GFCI & Kode

- **Stopkontak luar ruangan harus GFCI**
- **Penutup tahan cuaca** (penutup penggunaan / penutup gelembung) — diperlukan untuk lokasi lembab
- **Kabel ekstensi** — **Jangan pernah** permanen. Gunakan kabel + kotak peringkat luar ruangan yang tepat.

## Pemecahan Masalah Kamera Nirkabel

| Gejala                                     | Diagnosis                            | Perbaikan                                                                            |
| ------------------------------------------ | ------------------------------------ | ------------------------------------------------------------------------------------ |
| **"Kamera Offline" setiap hari**           | RSSI < -75 dBm                       | Pindahkan AP lebih dekat; tambah AP berkabel; periksa saluran                        |
| **Tampilan langsung berputar / gagal**     | Unggah < 1 Mbps                      | Uji Speedtest di kamera; upgrade AP                                                  |
| **Klip hilang di malam hari**              | Refleksi IR / whiteout               | Miringkan ke bawah 15°; jauhkan dari dinding; nonaktifkan IR + illuminator eksternal |
| **Baterai habis dalam 2 minggu**           | Zona lalu lintas tinggi / RSSI buruk | Kurangi sensitivitas; persempit zona; tambah surya; periksa -60 dBm                  |
| **Latensi tampilan langsung > 5 detik**    | Latensi tinggi / bufferbloat         | QoS di router (prioritaskan VLAN kamera); backhaul AP berkabel                       |
| **"Tidak Ada Wi-Fi" setelah mati listrik** | Kamera boot sebelum router           | UPS di router + AP; sambungkan kembali kamera secara otomatis (periksa firmware)     |
| **Tidak dapat terhubung ke 2,4 GHz**       | Band-steering router                 | Nonaktifkan band-steering; SSID terpisah ("RUMAH-2G", "RUMAH-5G")                    |

## Keamanan: Isolasi Kamera Anda

<Callout type="danger">

**Kamera adalah IoT — risiko pelanggaran tertinggi.** Isolasi pada VLAN dengan **tanpa akses internet** (kecuali NTP/firmware). Blokir kamera → LAN. Izinkan LAN → kamera (hanya tampilan).

</Callout>

**Contoh UniFi / VLAN:**

```
VLAN 10 (LAN)     192.168.10.x  → Akses penuh
VLAN 20 (KAMERA) 192.168.20.x  → Blokir internet, Blokir akses LAN
                    Izinkan: NTP (udp/123), DNS (udp/53) ke router
                    Izinkan: NVR (192.168.10.50) → Kamera (tcp/554, 8000)
```

**Aturan Firewall Router (Ubiquiti / OPNsense / pfSense):**

1. **Izinkan** KAMERA → Router (NTP, DNS)
2. **Izinkan** NVR → KAMERA (RTSP, HTTP, ONVIF)
3. **Blokir** KAMERA → LAN
4. **Blokir** KAMERA → WAN (internet)
5. **Izinkan** LAN → KAMERA (hanya manajemen, IP terbatas)

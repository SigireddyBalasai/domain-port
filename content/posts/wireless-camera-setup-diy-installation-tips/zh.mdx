无线摄像头承诺"无线缆、易安装"。现实: 盲区、电池焦虑、2.4 GHz 拥塞、冬天不充电的太阳能板。本指南将无线从令人沮丧变为可靠。

<Badge variant="outline">无线 ≠ 无线缆</Badge> **插电式无线** (Reolink E1, Wyze
v4, TP-Link Tapo) 需 120V 插座。**电池/太阳能** (EufyCam 3, Reolink Argus 4 Pro,
Ring Stick Up) 真正无线。两者都用 Wi-Fi。都需要扎实的射频规划。

## Wi-Fi 物理: 为何您的摄像头掉线

### 频段

| 频段                   | 信道              | 最大速度      | 穿墙  | 距离      | 最适合             |
| ---------------------- | ----------------- | ------------- | ----- | --------- | ------------------ |
| **2.4 GHz**            | 1, 6, 11 (不重叠) | 600 Mbps (ax) | ★★★★★ | 150+ 英尺 | **所有户外摄像头** |
| **5 GHz**              | 36–165            | 2.4 Gbps (ax) | ★★☆☆☆ | 50 英尺   | 室内、短距离       |
| **6 GHz (Wi-Fi 6E/7)** | 1–233             | 5+ Gbps       | ★☆☆☆☆ | 30 英尺   | 面向未来、仅室内   |

**原则:** 强制摄像头用 **仅 2.4 GHz**。5/6 GHz 穿墙/侧板差。多数摄像头也不支持 6 GHz。

### 信道规划 (2.4 GHz)

```
Channels:  1       6       11
Freq:   2412    2437    2462  MHz
Width:  20 MHz each (non-overlapping)
```

**先扫描:** 用 **WiFi Analyzer (Android)** 或 **AirPort Utility (iOS)** → 扫描 → 记录邻居信道。

**分配:**

- 您的路由器/AP: 信道 1, 6, 或 11 (最不拥挤者)
- 摄像头 AP (若单独): 不同的不重叠信道
- **宽度: 仅 20 MHz** (40 MHz 重叠邻居, 导致重传)

## 网络架构: Mesh vs 扩展器 vs 专用 AP

| 架构                                                     | 优点                          | 缺点                             | 最适合                    |
| -------------------------------------------------------- | ----------------------------- | -------------------------------- | ------------------------- |
| **Mesh (Eero Pro 6E, UniFi Dream Router, Asus ZenWiFi)** | 无缝漫游, 单 SSID, App 管理   | 成本 ($300–600), 所有流量过 Mesh | 全屋, 5+ 摄像头           |
| **有线 AP (UniFi U6 Lite, U6+, U7 Pro)**                 | 最佳性能, PoE 供电, VLAN 支持 | 需布以太网                       | **摄像头最佳** — 有线回程 |
| **Powerline + Wi-Fi (TP-Link TL-WPA8630)**               | 无新布线, 用电力线            | 50–200 Mbps 实测, AFCI 上噪大    | 无法布线的改造            |
| **范围扩展器 (TP-Link RE605X)**                          | 便宜 ($50)                    | 带宽减半, 延迟增加, 单独 SSID    | **摄像头避用**            |
| **户外 AP (UniFi U6 Mesh, UAP-AC-M)**                    | 防水, PoE, Mesh 上行          | 成本 ($180–300)                  | 独立车库、大门、谷仓      |

<Callout type="tip">

**专业架构:** PoE 交换机 → UniFi U6+ AP (吊顶, 居中) → 2.4 GHz 专用 SSID
"CAM-2G" (VLAN 20) → 所有摄像头. 隔离 IoT, 无漫游问题, 有线回程.

</Callout>

## 摄像头 Wi-Fi 需求

| 指标            | 最低              | 推荐               | 理由                           |
| --------------- | ----------------- | ------------------ | ------------------------------ |
| **信号 (RSSI)** | -70 dBm           | **-55 dBm 或更好** | -70 = 频繁掉线; -55 = 稳如磐石 |
| **SNR**         | 15 dB             | **25+ dB**         | 信噪余量                       |
| **上传带宽**    | 1 Mbps/机 (1080p) | 5 Mbps/机 (4K)     | 持续, 非突发                   |
| **延迟**        | &lt;100 ms        | &lt;30 ms          | 实时查看响应                   |
| **丢包**        | &lt;2%            | &lt;0.5%           | 录像间隙、冻帧                 |

**在摄像头位置测试:** 手机连同一 SSID → Speedtest → 记录上传。走到摄像头点 → 再测。若上传 &lt;5 Mbps, 先修 Wi-Fi。

- **中心、高处** — 走廊吊顶 > 角落架子
- **视线** — 避开: 烟囱、砖墙、金属侧板、HVAC、镜子、鱼缸
- **最多穿 1 墙** — 摄像头 ↔ AP 穿 ≤1 内墙
- **避开角落** — RF 反射, 产生多径
- **高度** — 7–9 英尺 AGL (离地高度) 覆盖最佳
- **天线朝向** — 垂直覆盖水平; 倾 45° 覆盖多层

**UniFi U6+ 例:** 走廊中心吊顶 → 2.4 GHz 覆盖 3,000 平方英尺住宅 + 50 英尺院子。摄像头 -50 到 -60 dBm。

## 电池摄像头优化

### 太阳能 + Wi-Fi 定位

| 因素         | 理想                                                  | 避免                      |
| ------------ | ----------------------------------------------------- | ------------------------- |
| **太阳能板** | 真南 (北半球), 30–45° 倾斜, **9am–3pm Dec 21 无阴影** | 北面, 檐下, 树影          |
| **摄像头**   | 面板 10–15 英尺 (线长), 檐下避雨                      | 镜头直射阳光 (过热), 喷淋 |
| **Wi-Fi**    | 距 AP ≤50 英尺, 穿 ≤1 墙                              | 金属侧板、砖烟囱在中间    |

### 电池寿命延长

| 设置           | 默认    | 优化              | 节省   |
| -------------- | ------- | ----------------- | ------ |
| **运动灵敏度** | 高      | 中 / 自定义区域   | 30–50% |
| **片段长度**   | 30s     | 10–15s            | 40–60% |
| **重触发间隔** | 10s     | 30–60s            | 20–30% |
| **夜视**       | Auto IR | 计划: 有路灯关 IR | 15–25% |
| **实时预览**   | 按需    | 禁用自动预览      | 10–20% |
| **固件**       | Auto    | 手动 (月检)       | 防 Bug |

**实测:** EufyCam 3 (13,000 mAh) @ 中灵敏度, 15s 片段, 30s 重触发, 2.4 GHz -55 dBm → **4–6 个月** (宣称 1 年)。

### 冬季电池护理 (5 区及更冷)

- **Li-ion &lt;32°F (0°C) 停止充电。** 摄像头靠电池撑到死。
- **对策:**
  - 每月拿进屋暖充 (痛苦但管用)
  - **LiFePO₄ 升级** (Reolink Go PT Plus, 自建) — -4°F 仍能充
  - **绝缘电池盒** + 5W 加热垫 (恒温 40°F) — DIY
  - **更大面板** (20W 对 6W) — 更多冬季收获

## 插电式无线摄像头设置

### 供电考量

| 摄像头                 | 功耗          | 插座类型          | 线缆管理              |
| ---------------------- | ------------- | ----------------- | --------------------- |
| Reolink E1 Outdoor     | 5W (12V/0.5A) | 120V → 12V 适配器 | 防水盒 + 胶圈         |
| Wyze Cam v4            | 3W (5V/0.6A)  | 120V → 5V USB     | USB 延长线 + 硅胶密封 |
| TP-Link Tapo C120      | 4W (5V/0.8A)  | 120V → 5V USB     | 短 USB, 下垂环        |
| Google Nest Cam (有线) | 7W (5V/1.4A)  | 120V → 5V 专用    | 仅专用线缆            |

**下垂环:** 线缆在插座下方成 **U 型** → 水滴落在环上, 不进插座。

### GFCI & 规范

- **户外插座必须 GFCI** (NEC 210.8(A)(3))
- **防水盖** (使用中/气泡盖) — 潮湿场所必需
- **延长线** — **永不** 永久使用. 用正规户外线缆 + 盒子。

## 无线摄像头故障排查

| 症状                  | 诊断                 | 修复                                               |
| --------------------- | -------------------- | -------------------------------------------------- |
| **"摄像头离线" 每天** | RSSI < -75 dBm       | 移 AP 近; 加有线 AP; 查信道                        |
| **实时画面转圈/失败** | 上传 < 1 Mbps        | 摄像头位 Speedtest; 升级 AP                        |
| **夜间片段丢失**      | IR 反射/白化         | 向下倾 15°; 离墙; 关 IR + 外置补光                 |
| **电池 2 周死**       | 高流量区 / 差 RSSI   | 降灵敏度; 窄区域; 加太阳能; 查 -60 dBm             |
| **实时延迟 >5 秒**    | 高延迟 / bufferbloat | 路由器 QoS (优先摄像头 VLAN); 有线 AP 回程         |
| **断电后 "无 Wi-Fi"** | 摄像头启动快于路由器 | 路由器+AP 上 UPS; 摄像头自动重连 (查固件)          |
| **连不上 2.4 GHz**    | 路由器 band-steering | 关 band-steering; 分离 SSID ("HOME-2G", "HOME-5G") |

## 安全: 隔离您的摄像头

<Callout type="danger">

**摄像头是 IoT — 最高违规风险.** 隔离到 VLAN, **无互联网访问** (除 NTP/固件).
封禁 camera → LAN. 允许 LAN → camera (仅查看).

</Callout>

**UniFi / VLAN 例:**

```
VLAN 10 (LAN)     192.168.10.x  → 完全访问
VLAN 20 (CAMERAS) 192.168.20.x  → 封禁互联网, 封禁 LAN 访问
                    允许: NTP (udp/123), DNS (udp/53) 到路由器
                    允许: NVR (192.168.10.50) → 摄像头 (tcp/554, 8000)
```

**路由器防火墙规则 (Ubiquiti / OPNsense / pfSense):**

- **允许** CAMERAS → 路由器 (NTP, DNS)
- **允许** NVR → CAMERAS (RTSP, HTTP, ONVIF)
- **封禁** CAMERAS → LAN
- **封禁** CAMERAS → WAN (互联网)
- **允许** LAN → CAMERAS (仅管理, 限制 IP)

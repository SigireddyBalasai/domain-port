무선 카메라는 "선 없음, 쉬운 설치"를 약속합니다. 현실: 사각지대, 배터리 불안, 2.4 GHz 혼잡, 겨울에 충전되지 않는 태양광 패널. 이 가이드는 무선을 실망스러운 경험에서 신뢰할 수 있는 경험으로 바꿔줍니다.

<Badge variant="outline">무선 ≠ 무선(전원)</Badge> **플러그인 무선** (Reolink
E1, Wyze v4, TP-Link Tapo)은 120V 콘센트 필요. **배터리/태양광** (EufyCam 3,
Reolink Argus 4 Pro, Ring Stick Up)은 진정한 무선. 둘 다 Wi-Fi 사용. 둘 다
견고한 RF 계획 필요.

---

## Wi-Fi 물리학: 카메라가 연결 끊기는 이유

### 주파수 대역

| 대역                   | 채널              | 최대 속도     | 투과력 | 범위    | 최적 용도            |
| ---------------------- | ----------------- | ------------- | ------ | ------- | -------------------- |
| **2.4 GHz**            | 1, 6, 11 (비중첩) | 600 Mbps (ax) | ★★★★★  | 150+ ft | **모든 야외 카메라** |
| **5 GHz**              | 36–165            | 2.4 Gbps (ax) | ★★☆☆☆  | 50 ft   | 실내, 단거리         |
| **6 GHz (Wi-Fi 6E/7)** | 1–233             | 5+ Gbps       | ★☆☆☆☆  | 30 ft   | 미래 대비, 실내 전용 |

**규칙:** 카메라를 **2.4 GHz 전용**으로 강제. 5/6 GHz는 벽/사이딩을 투과하지 못함. 대부분의 카메라는 어차피 6 GHz를 지원하지 않음.

### 채널 계획 (2.4 GHz)

```
채널:  1       6       11
주파수: 2412    2437    2462  MHz
폭:    20 MHz each (비중첩)
```

**먼저 스캔:** **WiFi Analyzer (Android)** 또는 **AirPort Utility (iOS)** 사용 → 스캔 → 이웃 채널 확인.

**할당:**

- 라우터/AP: 채널 1, 6, 또는 11 (가장 덜 혼잡한 채널)
- 카메라 AP (별도일 경우): 다른 비중첩 채널
- **폭: 20 MHz 전용** (40 MHz는 이웃과 중첩되어 재시도 발생)

## 네트워크 아키텍처: Mesh vs Extender vs 전용 AP

| 아키텍처                                                 | 장점                            | 단점                                     | 최적 용도                     |
| -------------------------------------------------------- | ------------------------------- | ---------------------------------------- | ----------------------------- |
| **Mesh (Eero Pro 6E, UniFi Dream Router, Asus ZenWiFi)** | 원활한 로밍, 단일 SSID, 앱 관리 | 비용 ($300–600), 모든 트래픽이 mesh 통해 | 전체 집, 5대 이상 카메라      |
| **유선 AP (UniFi U6 Lite, U6+, U7 Pro)**                 | 최고 성능, PoE 전원, VLAN 지원  | 이더넷 배선 필요                         | **카메라에 최적** — 유선 백홀 |
| **Powerline + Wi-Fi (TP-Link TL-WPA8630)**               | 새 배선 불필요, 전기 배선 사용  | 실속 50–200 Mbps, AFCI에서 잡음          | 케이블이 불가능한 리트로핏    |
| **범위 확장기 (TP-Link RE605X)**                         | 저렴 ($50)                      | 대역폭 절반, 지연 증가, 별도 SSID        | **카메라에 사용 금지**        |
| **야외 AP (UniFi U6 Mesh, UAP-AC-M)**                    | 방수, PoE, Mesh 업링크          | 비용 ($180–300)                          | 별도 차고, 게이트, 헛간       |

<Callout type="tip">

**프로 아키텍처:** PoE 스위치 → UniFi U6+ AP (천장, 중앙) → 2.4 GHz 전용 SSID "CAM-2G" (VLAN 20) → 모든 카메라. IoT 격리, 로밍 문제 없음, 유선 백홀.

</Callout>

## 카메라 Wi-Fi 요구사항

| 사양              | 최소                | 권장             | 이유                            |
| ----------------- | ------------------- | ---------------- | ------------------------------- |
| **신호 (RSSI)**   | -70 dBm             | **-55 dBm 이상** | -70 = 빈번한 끊김; -55 = 견고함 |
| **SNR**           | 15 dB               | **25+ dB**       | 신호 대 잡음 마진               |
| **업로드 대역폭** | 캠당 1 Mbps (1080p) | 캠당 5 Mbps (4K) | 지속적, 버스트 아님             |
| **지연 시간**     | <100 ms             | <30 ms           | 라이브 뷰 응답성                |
| **패킷 손실**     | <2%                 | <0.5%            | 녹화 공백, 프레임 정지          |

**카메라 위치에서 테스트:** 동일 SSID의 휴대폰 → Speedtest → 업로드 확인. 카메라 위치로 이동 → 재테스트. 업로드 <5 Mbps면 먼저 Wi-Fi 수정.

## 카메라용 Mesh / AP 배치

<Accordion type="single" collapsible>
  <AccordionItem value="placement">
    <AccordionTrigger>AP 배치 규칙</AccordionTrigger>
    <AccordionContent>

- **중앙, 높은 위치** — 복도 천장 장착 > 모서리 선반
- **가시선** — 피해야 할 것: 굴뚝, 벽돌 벽, 금속 사이딩, HVAC, 거울, 수족관
- **최대 벽 1개** — 카메라 ↔ AP 사이 내벽 ≤1개
- **모서리 피하기** — RF 반사, 다중 경로 생성
- **높이** — 최적 패턴을 위해 지상 7–9 ft
- **안테나 방향** — 수평 커버리지는 수직; 다층은 45° 기울임

**UniFi U6+ 예:** 복도 중앙 천장 장착 → 2.4 GHz가 3,000 sq ft 주택 + 50 ft 마당 커버. 카메라 -50 to -60 dBm.

## 배터리 카메라 최적화

### 태양광 + Wi-Fi 배치

| 요소            | 이상적                                                                    | 피해야 할 것                            |
| --------------- | ------------------------------------------------------------------------- | --------------------------------------- |
| **태양광 패널** | 정남향 (북반구), 30–45° 기울임, **12월 21일 오전 9시–오후 3시 그늘 없음** | 북향, 처마 아래, 나무 그늘              |
| **카메라**      | 패널에서 10–15 ft (케이블 도달), 비 보호 처마 아래                        | 렌즈에 직사광선 (과열), 스프링클러 분사 |
| **Wi-Fi**       | AP에서 50 ft 이내, 벽 최대 1개                                            | 사이의 금속 사이딩, 벽돌 굴뚝           |

### 배터리 수명 연장

| 설정              | 기본값  | 최적화                    | 절감      |
| ----------------- | ------- | ------------------------- | --------- |
| **모션 감도**     | 높음    | 중간 / 맞춤 영역          | 30–50%    |
| **클립 길이**     | 30초    | 10–15초                   | 40–60%    |
| **재트리거 간격** | 10초    | 30–60초                   | 20–30%    |
| **야간 투시**     | 자동 IR | 일정: 가로등 있으면 IR 끔 | 15–25%    |
| **라이브 뷰**     | 요청 시 | 자동 미리보기 비활성화    | 10–20%    |
| **펌웨어**        | 자동    | 수동 (매월 확인)          | 버그 방지 |

**실제:** EufyCam 3 (13,000 mAh) @ 중간 감도, 15초 클립, 30초 재트리거, 2.4 GHz -55 dBm → **4–6개월** (광고된 1년 대비).

### 겨울 배터리 관리 (Zone 5 및 더 추운 지역)

- **Li-ion은 32°F(0°C) 미만에서 충전 중단.** 카메라는 배터리가 방전될 때까지 실행.
- **해결책:**
  - 매월 실내로 가져와 온충전 (번거롭지만 효과적)
  - **LiFePO₄ 업그레이드** (Reolink Go PT Plus, 맞춤형) — -4°F까지 충전
  - **단열 배터리 박스** + 5W 히터 패드 (서모스탯 40°F) — DIY
  - **더 큰 패널** (6W 대신 20W) — 더 많은 겨울 수확

| ---------------------- | ------------- | ----------------- | ---------------------- |

| Wyze Cam v4            | 3W (5V/0.6A)  | 120V → 5V USB     | USB 연장 + 실리콘 실링 |

| Google Nest Cam (유선) | 7W (5V/1.4A)  | 120V → 5V 전용    | 전용 케이블만          |

**드립 루프:** 케이블이 콘센트 **아래에 U자 형태** 형성 → 물이 루프에서 떨어지고 콘센트로 들어가지 않음.

### GFCI 및 코드

- **야외 콘센트는 반드시 GFCI** (NEC 210.8(A)(3))
- **방수 커버** (인유스 / 버블 커버) — 습기 장소에 필요
- **연장 코드** — **절대** 영구 사용 금지. 적절한 야외용 케이블 + 박스 사용.

## 무선 카메라 문제 해결

| 증상                       | 진단                          | 수정                                                        |
| -------------------------- | ----------------------------- | ----------------------------------------------------------- |
| **매일 "카메라 오프라인"** | RSSI < -75 dBm                | AP를 더 가까이; 유선 AP 추가; 채널 확인                     |
| **라이브 뷰 회전 / 실패**  | 업로드 < 1 Mbps               | 카메라 위치에서 Speedtest; AP 업그레이드                    |
| **야간 클립 누락**         | IR 반사 / 화이트아웃          | 아래로 15° 각도 조정; 벽에서 멀리; IR 끄고 외부 조명기 사용 |
| **2주 만에 배터리 방전**   | 높은 트래픽 영역 / 나쁜 RSSI  | 감도 감소; 영역 좁힘; 태양광 추가; -60 dBm 확인             |
| **라이브 뷰 지연 > 5초**   | 높은 지연 / 버퍼블로트        | 라우터 QoS (카메라 VLAN 우선 순위); 유선 AP 백홀            |
| **정전 후 "Wi-Fi 없음"**   | 카메라가 라우터보다 먼저 부팅 | 라우터 + AP에 UPS; 카메라 자동 재연결 (펌웨어 확인)         |
| **2.4 GHz에 연결 불가**    | 라우터 대역 스티어링          | 대역 스티어링 비활성화; SSID 분리 ("HOME-2G", "HOME-5G")    |

## 보안: 카메라 격리

<Callout type="danger">

**카메라는 IoT — 가장 높은 침해 위험.** **인터넷 접근 없이** VLAN에 격리 (NTP/펌웨어 제외). 카메라 → LAN 차단. LAN → 카메라 허용 (보기 전용).

</Callout>

**UniFi / VLAN 예:**

```
VLAN 10 (LAN)     192.168.10.x  → 전체 접근
VLAN 20 (CAMERAS) 192.168.20.x  → 인터넷 차단, LAN 접근 차단
                    허용: NTP (udp/123), DNS (udp/53) to router
                    허용: NVR (192.168.10.50) → Cameras (tcp/554, 8000)
```

**라우터 방화벽 규칙 (Ubiquiti / OPNsense / pfSense):**

- **허용** CAMERAS → 라우터 (NTP, DNS)
- **허용** NVR → CAMERAS (RTSP, HTTP, ONVIF)
- **차단** CAMERAS → LAN
- **차단** CAMERAS → WAN (인터넷)
- **허용** LAN → CAMERAS (관리 전용, 제한된 IP)

Jedes Kamerasystem entwickelt Probleme. Dieser Leitfaden behandelt die 25 häufigsten Probleme mit Schritt-für-Schritt-Diagnose und Lösungen — geordnet nach Symptom, damit Sie schnell die Lösung finden.

<Badge variant="outline">Diagnosemethode</Badge> **Fangen Sie einfach an:**
Strom → Kabel → Netzwerk → Einstellungen → Hardware. 80 % der Probleme sind
Strom oder Kabel.

## Schnelldiagnose-Flussdiagramm

```
Kamera funktioniert nicht?
    │
    ├─ Keine Power-LED? → Netzteil / PoE / Akku (Abschnitt 1)
    ├─ Strom OK, kein Bild? → Kabel / Link / IP-Konflikt (Abschnitt 2)
    ├─ Bild OK, keine Aufzeichnung? → NVR / Speicher / Zeitplan (Abschnitt 3)
    ├─ Bild OK, Fehlalarme? → Bewegungszonen / Empfindlichkeit / KI (Abschnitt 4)
    ├─ Nachtsicht schlecht? → IR / Reflexion / Verschluss (Abschnitt 5)
    └─ Fernzugriff fehlgeschlagen? → Port-Forward / VPN / Hersteller-Cloud (Abschnitt 6)
```

## 1. Strom- & Startprobleme

### 1.1 Kamera tot — Keine LED, Keine Reaktion

| Schritt | Prüfen                  | Lösung                                                                                   |
| ------- | ----------------------- | ---------------------------------------------------------------------------------------- |
| 1       | **PoE-Switch-Port-LED** | Aus = Switch-Port defekt / Budget überschritten → Auf bekanntermaßen guten Port wechseln |
| 2       | **PoE-Injektor-LED**    | Aus = Injektor defekt / kein Strom → Injektor ersetzen (25 €)                            |
| 3       | **Spannung an Kamera**  | Multimeter: 44–57V (PoE), 12V±10 % (DC) → Niedrig = Spannungsabfall / schlechtes Kabel   |
| 4       | **Kabeldurchgang**      | Tester: alle 8 Adern → Offenes Paar = beide Enden neu anschließen                        |
| 5       | **Kamera-Reset**        | Reset 10–30s gedrückt halten (Handbuch prüfen) → Neu in App/NVR einbinden                |

**Akku-Kameras:** 24 Std. über USB-C laden, bevor Sie sie für tot erklären. Li-Ion-BMS sperrt nach Tiefentladung.

### 1.2 Kamera startet nachts neu (IR an)

**Ursache:** Spannungsabfall, wenn IR-LEDs aktiviert werden (zusätzlich 1–2 W).

| Diagnose                                 | Lösung                                                |
| ---------------------------------------- | ----------------------------------------------------- |
| PoE-Switch am Budgetlimit                | Kamera auf Injektor umstecken; Switch upgraden        |
| Kabellänge >76 m bei Cat6                | Kabel kürzen; Cat6a; PoE++-Injektor an der Kamera     |
| CCA-Kabel (kupferkaschiertes Aluminium)  | **Durch massives Kupfer-Cat6 ersetzen**               |
| Akku-Kamera: Li-Ion-Kältespannungsabfall | LiFePO₄-Upgrade; isolierte Box; zum Laden hereinholen |

### 1.3 PoE-Switch überhitzt / Ports flackern

- **PoE-Budget prüfen:** Σ Kamera max. Verbrauch ≤ Switch-Budget × 0,8
- **Thermisch:** Switch in geschlossenem Schrank → Lüfter hinzufügen oder in belüfteten Rack umziehen
- **Firmware:** Switch-Firmware aktualisieren (PoE-Aushandlungsfehler häufig)

## 2. Netzwerk & Konnektivität

### 2.1 Kamera "Offline" in App/NVR

| Schritt | Test                                      | Lösung                                                     |
| ------- | ----------------------------------------- | ---------------------------------------------------------- |
| 1       | Ping Kamera-IP vom PC                     | Zeitüberschreitung = Layer-1/2-Problem (Kabel, Switch, IP) |
| 2       | Ping Gateway von Kamera (NVR-Shell)       | Zeitüberschreitung = VLAN / Firewall / Routing             |
| 3       | `arp -a` → MAC stimmt mit Kamera überein? | IP-Konflikt → Statische IP / DHCP-Reservierung             |
| 4       | DHCP-Lease abgelaufen                     | Statische IP oder DHCP-Reservierung (Router)               |
| 5       | IP-Konflikt                               | Zwei Geräte gleiche IP → Eindeutige IPs reservieren        |

### 2.2 Wi-Fi-Kamera fällt täglich aus

| Diagnose            | Lösung                                                        |
| ------------------- | ------------------------------------------------------------- |
| RSSI < -75 dBm      | AP näher bringen; kabelgebundenen AP hinzufügen; Kanal prüfen |
| 2,4-GHz-Überlastung | Nur Kanal 1/6/11; 20 MHz Breite; Band-Steering deaktivieren   |
| DFS-Kanal (5 GHz)   | 2,4 GHz erzwingen; 5 GHz Radar = Trennung                     |
| Energiesparmodus    | Wi-Fi-Energiesparen in Kamera-App deaktivieren                |
| Router-Neustartplan | Auto-Neustart deaktivieren; UPS am Router                     |

### 2.3 RTSP-Stream öffnet nicht (VLC / Frigate / NVR)

```
Test-URL: rtsp://benutzer:passwort@192.168.1.100:554/stream1
```

| Fehler               | Ursache                    | Lösung                                                                                                                             |
| -------------------- | -------------------------- | ---------------------------------------------------------------------------------------------------------------------------------- |
| 401 Unauthorized     | Falscher Benutzer/Passwort | In NVR/App prüfen: oft `admin` / `123456` oder leer                                                                                |
| 404 Not Found        | Falscher Pfad              | Herstellerpfade: Hikvision `/Streaming/Channels/101`, Dahua `/cam/realmonitor?channel=1&subtype=0`, Reolink `/h264Preview_01_main` |
| Verbindung abgelehnt | RTSP deaktiviert           | In Kamera-Web-UI aktivieren → Netzwerk → RTSP                                                                                      |
| Zeitüberschreitung   | Firewall / VLAN            | TCP 554 von NVR-VLAN zu Kamera-VLAN erlauben                                                                                       |

**Hersteller-RTSP-Pfade (Üblich):**

- **Reolink:** `/h264Preview_01_main` (Main), `/h264Preview_01_sub` (Sub)
- **Dahua/Amcrest:** `/cam/realmonitor?channel=1&subtype=0`
- **Hikvision:** `/Streaming/Channels/101` (Main), `/Streaming/Channels/102` (Sub)
- **UniFi:** `rtsp://benutzer:passwort@ip:7447/...` (Protect-Einstellungen → RTSP)
- **Amcrest:** Gleich wie Dahua (OEM)

## 3. Aufzeichnung & Speicher

### 3.1 NVR zeichnet nicht auf / Lücken im Zeitstrahl

| Prüfung                      | Lösung                                                             |
| ---------------------------- | ------------------------------------------------------------------ |
| HDD-Zustand (S.M.A.R.T.)     | Neu zugeordnete Sektoren >0 → **Sofort ersetzen**                  |
| HDD voll / Überschreiben aus | "Älteste überschreiben" in Speichereinstellungen aktivieren        |
| Zeitplan falsch konfiguriert | Prüfen: Dauerhaft 24/7 auf Main-Stream; Bewegung auf Sub           |
| Stream-Konflikt              | NVR erwartet Main; Kamera sendet Sub → Encodierung anpassen        |
| Zeitstempel-Drift            | NTP auf NVR + Kameras aktivieren (gleicher Server: `pool.ntp.org`) |

### 3.2 microSD zeichnet nicht auf / defekt

| Symptom                             | Ursache                             | Lösung                                                                               |
| ----------------------------------- | ----------------------------------- | ------------------------------------------------------------------------------------ |
| "Formatieren erforderlich"-Schleife | Karte defekt / falsches Dateisystem | Durch **U3/V30 A2** ersetzen (SanDisk Extreme, Samsung Pro)                          |
| Dateien 0 Bytes / nicht abspielbar  | Stromausfall während Schreibens     | "Sicheres Entfernen" in App aktivieren; UPS bei Steckdosenkameras                    |
| Stoppt bei 4 GB                     | FAT32-Grenze                        | Kamera sollte automatisch exFAT verwenden; falls nicht, auf PC mit exFAT formatieren |
| Schleife funktioniert nicht         | Karte voll / gesperrt               | Schreibschutz prüfen; alte Dateien löschen; in Kamera formatieren                    |

### 3.3 Wiedergabe eingefroren / verpixelt / grüner Bildschirm

| Ursache                      | Lösung                                                              |
| ---------------------------- | ------------------------------------------------------------------- |
| H.265/HEVC nicht unterstützt | Player benötigt HEVC-Codec (Windows: 0,99 € Store; VLC: integriert) |
| Sub-Stream ausgewählt        | In Wiedergabe auf Main-Stream umschalten                            |
| Keyframe-Intervall zu hoch   | Keyframe-Intervall = fps setzen (z. B. 15 fps → 15)                 |
| VBR-Bitrate-Spitzen          | CBR oder VBR Max Cap (4–6 Mbps für 4K) einstellen                   |

## 4. Bewegungserkennung & Benachrichtigungen

### 4.1 Zu viele Fehlalarme (Spam)

| Einstellung                | Empfohlen                          | Warum                                        |
| -------------------------- | ---------------------------------- | -------------------------------------------- |
| **Erkennungstyp**          | Nur Person + Fahrzeug              | Schließt Tiere, Bäume, Schatten, Regen aus   |
| **Bewegungszonen**         | NUR Zielbereich zeichnen           | Straße, Bäume, Nachbarn, Himmel ausschließen |
| **Empfindlichkeit**        | Mittel (Start) → anpassen          | Hoch = alles löst aus                        |
| **Wiederholungsintervall** | 30–60 Sekunden                     | Verhindert 50 Clips desselben Autos          |
| **Objektgrößenfilter**     | Min 5 % / Max 50 % des Bildes      | Ignoriert Insekten, Vögel, entfernte Autos   |
| **KI/Deep Learning**       | Aktivieren (Person/Fahrzeug/Paket) | 90 %+ Reduzierung von Fehlalarmen            |

### 4.2 Verpasste Ereignisse (Keine Benachrichtigung bei Vorfällen)

| Prüfung                        | Lösung                                                                      |
| ------------------------------ | --------------------------------------------------------------------------- |
| Zone deckt Ereignisbereich ab? | Zone weiter zeichnen; Testlauf machen                                       |
| Empfindlichkeit zu niedrig?    | Eine Stufe erhöhen; testen                                                  |
| "Person" nachts nicht erkannt  | IR-Reflexion verwirrt KI → Kamera nach unten neigen; externes IR hinzufügen |
| Kamera im "Schlaf"-Modus       | Akku-Kameras: Schlafmodus deaktivieren / PIR-Empfindlichkeit erhöhen        |
| NVR-Bewegungserkennung AUS     | Auf NVR-Seite aktivieren (kameraseitig kann aus sein)                       |

### 4.3 Benachrichtigungsverzögerung >10 Sekunden

| Ursache                         | Lösung                                                   |
| ------------------------------- | -------------------------------------------------------- |
| Cloud-Relay (Ring, Arlo, Blink) | Systembedingt — lokalen NVR/Frigate für &lt;1s verwenden |
| Schwaches Wi-Fi (RSSI < -75)    | Signal verbessern (Abschnitt 2.2)                        |
| NVR-CPU ausgelastet             | Analytik-Kanäle reduzieren; NVR upgraden                 |
| MQTT/Webhook-Latenz             | Zuerst direkten App-Push testen                          |

## 5. Bildqualität & Nachtsicht

### 5.1 Nachtsicht überstrahlt / ausgewaschen

**Ursache:** IR-LEDs reflektieren von naher Oberfläche (Wand, Dachvorsprung, Glas, Spinnennetz).

| Lösung                           | Wie                                                                  |
| -------------------------------- | -------------------------------------------------------------------- |
| **15–30° nach unten neigen**     | Neigehalterung; Kamera blickt nach unten, nicht auf die Wand         |
| **Kamera abrücken**              | 30–45 cm von Wand/Dachvorsprung (Verlängerungsarm)                   |
| **Eingebautes IR deaktivieren**  | **Externen IR-Strahler** hinzufügen (separat, auf Ziel ausgerichtet) |
| **Kuppel/Linse reinigen**        | Spinnennetze = IR-Streuung → Mikrofaser + Linsenreiniger             |
| **Smart IR (falls unterstützt)** | Passt IR-Intensität automatisch an die Entfernung an                 |

### 5.2 Kennzeichen nachts nicht lesbar

| Anforderung        | Technische Daten                                                            |
| ------------------ | --------------------------------------------------------------------------- |
| **Verschlusszeit** | Mindestens 1/100s (1/250s ideal) — einstellen unter Belichtung → Verschluss |
| **Auflösung**      | 4K/8MP + optischer Zoom ODER 2MP-LPR-Kamera                                 |
| **IR-Reichweite**  | Kennzeichen-Entfernung ≤ 50 % der IR-Nennreichweite der Kamera              |
| **WDR**            | 120 dB+ (gleicht Scheinwerfer aus)                                          |
| **Winkel**         | ≤30° von der Senkrechten (Kennzeichen frontal)                              |

**Lösung:** Verschluss erhöhen → Bild wird dunkler, aber Kennzeichen scharf. Bei zu dunklem Bild externes IR hinzufügen.

### 5.3 Tagesbild: diesig / kontrastarm / Farbstich

| Ursache                        | Lösung                                                                  |
| ------------------------------ | ----------------------------------------------------------------------- |
| Kuppel verschmutzt / verkratzt | Mikrofaser reinigen; Kuppel ersetzen (15 €)                             |
| IR-Sperrfilter klemmt          | Neustart (12V/PoE 30s aus). Kamera leicht antippen. Firmware-Update.    |
| Weißabgleich falsch            | "Außenbereich" / "Auto" / Benutzerdefiniertes Kelvin (5500K) einstellen |
| Gegenlicht (Sonne im Bild)     | Neu positionieren; WDR aktivieren; Sonnenschutz anbringen               |

### 5.4 Flackern / Rollende Linien (50/60 Hz)

| Ursache                       | Lösung                                                                |
| ----------------------------- | --------------------------------------------------------------------- |
| **Netzfrequenz-Konflikt**     | Kamera auf 60 Hz (US) / 50 Hz (EU) eingestellt — an Ihr Netz anpassen |
| **Masseschleife**             | Alle Geräte gleiche Masse; PoE-Isolator (Transtector)                 |
| **Leuchtstoff-/LED-Flackern** | Verschluss auf 1/100 (50 Hz) oder 1/120 (60 Hz) einstellen            |

## 6. Fernzugriff & Mobile App

### 6.1 App funktioniert im WLAN, nicht über LTE/5G

| Architektur                                       | Lösung                                                                           |
| ------------------------------------------------- | -------------------------------------------------------------------------------- |
| **Hersteller-Cloud** (Reolink, UniFi, Eufy, Ring) | Internet auf Handy prüfen; Hersteller-Statusseite                                |
| **Port-Forwarding**                               | **Nicht verwenden.** CGNAT, Double NAT, ISP-Sperren. Auf VPN/Cloud umstellen.    |
| **VPN (Tailscale, WireGuard)**                    | Sicherstellen, dass Handy-VPN aktiv ist; Split-Tunneling könnte App ausschließen |
| **Cloudflare Tunnel**                             | Prüfen, ob `cloudflared`-Dienst läuft                                            |

### 6.2 "Verbindung fehlgeschlagen" / Zeitüberschreitung bei Fernzugriff

| Test                               | Erwartet  | Lösung                         |
| ---------------------------------- | --------- | ------------------------------ |
| Ping NVR-LAN-IP vom Handy (VPN an) | &lt;50 ms | VPN-Routing-Problem            |
| `curl http://nvr:80` vom Handy     | HTML      | Port blockiert / Firewall      |
| Hersteller-Cloud-Statusseite       | Grün      | Warten / lokales VPN verwenden |

### 6.3 Live-Ansicht funktioniert, Wiedergabe scheitert remote

- Wiedergabe verwendet anderen Port/Stream (oft HTTPS/443 vs. RTSP/554)
- **Prüfen:** NVR → Netzwerk → HTTPS-Port (Standard 443) über VPN erreichbar
- **Bandbreite:** Wiedergabe benötigt dauerhaft 4–8 Mbps; LTE kann drosseln

## 7. PTZ & Mechanik

### 7.1 PTZ bewegt sich nicht / klemmt / schleift

| Ursache                           | Lösung                                                     |
| --------------------------------- | ---------------------------------------------------------- |
| Mechanischer Endanschlag erreicht | Preset-Limits prüfen; auf Ausgangsposition zurücksetzen    |
| Schleifringdefekt (360° Endlos)   | Kabelbaum abgenutzt → Kamera ersetzen (nicht reparierbar)  |
| Eis / Schmutz                     | Heizung defekt; manuell reinigen; Heizkreislauf prüfen     |
| Protokollkonflikt                 | Pelco-D/P vs. ONVIF PTZ → NVR-Protokoll an Kamera anpassen |

### 7.2 Preset-Tour läuft nicht

- Zeitplan aktiviert? (NVR → PTZ → Tour-Zeitplan)
- Verweildauer >0? (10–30s pro Preset)
- Ausgangspreset definiert? (Tour kehrt zwischen Durchläufen zur Ausgangsposition zurück)

## 8. Audio (Gegensprechen)

### 8.1 Kein Audio / Nur eine Richtung

| Prüfung                   | Lösung                                                                      |
| ------------------------- | --------------------------------------------------------------------------- |
| App-Berechtigung          | Handy-Einstellungen → App → Mikrofon: EIN                                   |
| Kamera-Audio aktiviert    | Kamera-Web-UI → Audio → Eingang/Ausgang: EIN                                |
| Codec-Konflikt            | G.711 (PCM) am kompatibelsten; G.726 / AAC versuchen                        |
| Halbduplex vs. Vollduplex | Die meisten Kameras = PTT (Halbduplex); nicht sprechen während des Zuhörens |

### 8.2 Echo / Rückkopplung / Rauschen

| Ursache                                            | Lösung                                                                      |
| -------------------------------------------------- | --------------------------------------------------------------------------- |
| Lautsprecher zu laut / Mikrofonverstärkung zu hoch | Lautstärke in App reduzieren; Mikrofonverstärkung verringern                |
| Keine AEC (Akustische Echokompensation)            | Vollduplex-Kameras benötigen AEC; Halbduplex = Mikrofon stumm beim Sprechen |
| Wind am Mikrofon                                   | Windfilter aktivieren; Schaumstoff-Windschutz; neu positionieren            |

## 9. Firmware & Updates

### 9.1 Sicheres Firmware-Update-Verfahren

1. **Changelog lesen** — Überspringen bei "nur Sicherheit" und stabilem System
2. **Konfiguration sichern** — NVR: Konfiguration exportieren; Kamera: Einstellungen screenshotten
3. **NVR zuerst aktualisieren** — Dann Kameras (NVR kann kompatible Firmware übertragen)
4. **Eine Kamera nach der anderen** — Jede vor der nächsten prüfen
5. **Nicht ausschalten** — 5–10 Min. pro Kamera
6. **Prüfen** — Live-Ansicht, Aufzeichnung, Bewegung, PTZ, Audio

### 9.2 Fehlgeschlagene Firmware zurücksetzen

| Marke               | Methode                                                                           |
| ------------------- | --------------------------------------------------------------------------------- |
| **Reolink**         | Firmware-Seite → "Vorherige Version"-Schaltfläche                                 |
| **Dahua/Hikvision** | TFTP-Wiederherstellung (Boot + Reset gedrückt halten + TFTP-Server)               |
| **UniFi**           | Protect → Geräte → Kamera → "Firmware zurücksetzen"                               |
| **Generisch**       | SD-Karte mit `firmware.bin` im Stammverzeichnis; mit gedrücktem Reset einschalten |

## 10. Umwelt & Physisch

### 10.1 Wassereintritt / Kondensation

| Symptom                | Lösung                                                                        |
| ---------------------- | ----------------------------------------------------------------------------- |
| Beschlag in der Kuppel | Trockenmittelbeutel (jährlich ersetzen); Entlüftungsstopfen prüfen (Gore-Tex) |
| Wasser am Kabeleingang | Silikon + Tülle; Tropfschlaufe; Leerrohr                                      |
| Korrosion an RJ45      | Dielektrisches Fett auf Kontakten; Stecker ersetzen; geschirmtes Kabel        |

### 10.2 Insekten / Spinnen / Vögel

| Abwehrmittel                            | Wirksamkeit                        |
| --------------------------------------- | ---------------------------------- |
| Pfefferminzöl (Wattebausch nahe Kamera) | Hoch (Spinnen hassen es)           |
| Ultraschall-Schädlingsvertreiber        | Mittel                             |
| Vogelspikes auf Halterung               | Hoch (Vögel)                       |
| Fleece-Abdeckung über IR-Ring           | Hoch (Spinnen bauen auf warmem IR) |
| Regelmäßige Reinigung (monatlich)       | Unverzichtbar                      |

### 10.3 Vandalismus / Diebstahl

- **Torx-/Sicherheitsschrauben** (bei den meisten Profi-Kameras inklusive)
- **Leerrohr** zur Kamera (Kabel kann nicht leicht durchtrennt werden)
- **Höhe** >3 m (erfordert Leiter)
- **Backup-Kamera**, die die Kamera beobachtet (Überlappung)
- **Manipulationsalarm** (NVR: Kamera → Manipulationserkennung → Alarm)

## Wann Sie einen Profi rufen sollten

| Situation                            | Grund                                      |
| ------------------------------------ | ------------------------------------------ |
| Mehrstöckige Dachverlegung           | Sturzrisiko, Spezialhebebühne              |
| Beton-/Ziegelbohrungen >6 Löcher     | Bohrmaschine + Bohrer verschleißen schnell |
| Genehmigung erforderlich             | Gewerblich, Mehrfamilienhäuser, >500 €     |
| Glasfaserverlegung                   | Fusion-Spleißgerät, OTDR-Messung           |
| Blitzschutz / Erdung                 | DIN VDE 0185, geprüfte Komponenten         |
| Versicherungspflichtige Installation | Konformitätsbescheinigung                  |

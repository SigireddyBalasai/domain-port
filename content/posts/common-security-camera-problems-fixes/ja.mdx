Every camera system develops issues. This guide covers the 25 most common problems with step-by-step diagnosis and fixes — organized by symptom so you find the solution fast.

<Badge variant="outline">Diagnosis Method</Badge> **Start simple:** Power →
Cable → Network → Settings → Hardware. 80% of issues are power or cable.

## Quick Diagnostic Flowchart

```
Camera not working?
    │
    ├─ No power LED? → Power Supply / PoE / Battery (Section 1)
    ├─ Power OK, no image? → Cable / Link / IP Conflict (Section 2)
    ├─ Image OK, no recording? → NVR / Storage / Schedule (Section 3)
    ├─ Image OK, false alerts? → Motion Zones / Sensitivity / AI (Section 4)
    ├─ Night vision bad? → IR / Reflection / Shutter (Section 5)
    └─ Remote access fails? → Port Forward / VPN / Vendor Cloud (Section 6)
```

## 1. Power & Startup Issues

### 1.1 Camera Dead — No LED, No Response

| Step | Check                   | Fix                                                                     |
| ---- | ----------------------- | ----------------------------------------------------------------------- |
| 1    | **PoE Switch Port LED** | Off = switch port dead / budget exceeded → Move to known-good port      |
| 2    | **PoE Injector LED**    | Off = injector failed / no AC → Replace injector ($25)                  |
| 3    | **Voltage at Camera**   | Multimeter: 44–57V (PoE), 12V±10% (DC) → Low = voltage drop / bad cable |
| 4    | **Cable Continuity**    | Tester: all 8 wires → Open pair = re-terminate both ends                |
| 5    | **Camera Reset**        | Hold reset 10–30s (check manual) → Re-adopt in app/NVR                  |

**Battery Cameras:** Charge 24 hrs via USB-C before declaring dead. Li-ion BMS locks out after deep discharge.

### 1.2 Camera Reboots at Night (IR On)

**Cause:** Voltage drop when IR LEDs kick in (adds 1–2W).

| Diagnosis                        | Fix                                                    |
| -------------------------------- | ------------------------------------------------------ |
| PoE switch at budget limit       | Move camera to injector; upgrade switch                |
| Cable run >250 ft on Cat6        | Shorten run; Cat6a; PoE++ injector at camera           |
| CCA (copper-clad aluminum) cable | **Replace with solid copper Cat6**                     |
| Battery cam: Li-ion cold sag     | LiFePO₄ upgrade; insulated box; bring inside to charge |

### 1.3 PoE Switch Overheating / Ports Flapping

- **Check PoE Budget:** Σ camera max draw ≤ switch budget × 0.8
- **Thermal:** Switch in enclosed cabinet → add fan or move to ventilated rack
- **Firmware:** Update switch firmware (PoE negotiation bugs common)

## 2. Network & Connectivity

### 2.1 Camera "Offline" in App/NVR

| Step | Test                                 | Fix                                           |
| ---- | ------------------------------------ | --------------------------------------------- |
| 1    | Ping camera IP from PC               | Timeout = Layer 1/2 issue (cable, switch, IP) |
| 2    | Ping gateway from camera (NVR shell) | Timeout = VLAN / firewall / routing           |
| 3    | `arp -a` → MAC matches camera?       | IP conflict → Static IP / DHCP reservation    |
| 4    | DHCP lease expired                   | Static IP or DHCP reservation (router)        |
| 5    | IP conflict                          | Two devices same IP → Reserve unique IPs      |

### 2.2 Wi-Fi Camera Drops Daily

| Diagnosis              | Fix                                                      |
| ---------------------- | -------------------------------------------------------- |
| RSSI < -75 dBm         | Move AP closer; add wired AP; check channel              |
| 2.4 GHz congestion     | Channel 1/6/11 only; 20 MHz width; disable band-steering |
| DFS Channel (5 GHz)    | Force 2.4 GHz; 5 GHz radar = disconnect                  |
| Power Save Mode        | Disable Wi-Fi power save in camera app                   |
| Router reboot schedule | Disable auto-reboot; UPS on router                       |

### 2.3 RTSP Stream Won't Open (VLC / Frigate / NVR)

```
Test URL: rtsp://user:pass@192.168.1.100:554/stream1
```

| Error              | Cause           | Fix                                                                                                                            |
| ------------------ | --------------- | ------------------------------------------------------------------------------------------------------------------------------ |
| 401 Unauthorized   | Wrong user/pass | Check NVR/app: often `admin` / `123456` or blank                                                                               |
| 404 Not Found      | Wrong path      | Brand paths: Hikvision `/Streaming/Channels/101`, Dahua `/cam/realmonitor?channel=1&subtype=0`, Reolink `/h264Preview_01_main` |
| Connection Refused | RTSP disabled   | Enable in camera web UI → Network → RTSP                                                                                       |
| Timeout            | Firewall / VLAN | Allow TCP 554 from NVR VLAN to Camera VLAN                                                                                     |

**Brand RTSP Paths (Common):**

- **Reolink:** `/h264Preview_01_main` (main), `/h264Preview_01_sub` (sub)
- **Dahua/Amcrest:** `/cam/realmonitor?channel=1&subtype=0`
- **Hikvision:** `/Streaming/Channels/101` (main), `/Streaming/Channels/102` (sub)
- **UniFi:** `rtsp://user:pass@ip:7447/...` (Protect settings → RTSP)
- **Amcrest:** Same as Dahua (OEM)

## 3. Recording & Storage

### 3.1 NVR Not Recording / Gaps in Timeline

| Check                    | Fix                                                          |
| ------------------------ | ------------------------------------------------------------ |
| HDD Health (S.M.A.R.T.)  | Reallocated sectors >0 → **Replace immediately**             |
| HDD Full / Overwrite Off | Enable "Overwrite Oldest" in storage settings                |
| Schedule Misconfigured   | Verify: Continuous 24/7 on Main Stream; Motion on Sub        |
| Stream Mismatch          | NVR expects Main; camera sending Sub → Match encode settings |
| Timestamp Drift          | Enable NTP on NVR + Cameras (same server: `pool.ntp.org`)    |

### 3.2 microSD Not Recording / Corrupt

| Symptom                    | Cause                   | Fix                                                       |
| -------------------------- | ----------------------- | --------------------------------------------------------- |
| "Format Required" loop     | Card failed / wrong FS  | Replace with **U3/V30 A2** (SanDisk Extreme, Samsung Pro) |
| Files 0 bytes / unplayable | Power loss during write | Enable "Safe Eject" in app; UPS on plug-in cams           |
| Stops at 4 GB              | FAT32 limit             | Camera should auto-exFAT; if not, format exFAT on PC      |
| Loop not working           | Card full / locked      | Check lock switch; delete old files; format in camera     |

### 3.3 Playback Frozen / Pixelated / Green Screen

| Cause                      | Fix                                                           |
| -------------------------- | ------------------------------------------------------------- |
| H.265/HEVC not supported   | Player needs HEVC codec (Windows: $0.99 Store; VLC: built-in) |
| Sub-stream selected        | Switch to Main Stream in playback                             |
| Keyframe interval too high | Set Keyframe Interval = FPS (e.g., 15 fps → 15)               |
| VBR bitrate spikes         | Set CBR or VBR Max Cap (4–6 Mbps for 4K)                      |

## 4. Motion Detection & Alerts

### 4.1 Too Many False Alerts (Spam)

| Setting                | Recommended                     | Why                                    |
| ---------------------- | ------------------------------- | -------------------------------------- |
| **Detection Type**     | Person + Vehicle only           | Excludes animals, trees, shadows, rain |
| **Motion Zones**       | Draw ONLY target area           | Exclude street, trees, neighbor, sky   |
| **Sensitivity**        | Medium (start) → adjust         | High = everything triggers             |
| **Retrigger Interval** | 30–60 seconds                   | Prevents 50 clips of same car          |
| **Object Size Filter** | Min 5% / Max 50% of frame       | Ignores bugs, birds, distant cars      |
| **AI/Deep Learning**   | Enable (Person/Vehicle/Package) | 90%+ false alert reduction             |

### 4.2 Missed Events (No Alert When Something Happened)

| Check                          | Fix                                                         |
| ------------------------------ | ----------------------------------------------------------- |
| Zone covers event area?        | Redraw zone wider; test walk                                |
| Sensitivity too low?           | Increase one notch; test                                    |
| "Person" not detected at night | IR reflection confuses AI → angle cam down; add external IR |
| Camera in "Sleep" mode         | Battery cams: disable sleep / increase PIR sensitivity      |
| NVR motion detection OFF       | Enable on NVR side (camera-side may be off)                 |

### 4.3 Alert Delay >10 Seconds

| Cause                           | Fix                                         |
| ------------------------------- | ------------------------------------------- |
| Cloud relay (Ring, Arlo, Blink) | Inherent — use local NVR/Frigate for &lt;1s |
| Weak Wi-Fi (RSSI < -75)         | Improve signal (Section 2.2)                |
| NVR CPU maxed                   | Reduce analytics channels; upgrade NVR      |
| MQTT/Webhook latency            | Test direct app push first                  |

## 5. Image Quality & Night Vision

### 5.1 Night Vision Whiteout / Washed Out

**Cause:** IR LEDs reflecting off nearby surface (wall, eave, glass, spider web).

| Fix                         | How                                                         |
| --------------------------- | ----------------------------------------------------------- |
| **Angle down 15–30°**       | Tilt bracket; camera looks down, not at wall                |
| **Move camera out**         | 12–18" from wall/eave (extension arm)                       |
| **Disable built-in IR**     | Add **external IR illuminator** (separate, aimed at target) |
| **Clean dome/lens**         | Spider webs = IR scatter → microfiber + lens cleaner        |
| **Smart IR (if supported)** | Auto-adjusts IR intensity by distance                       |

### 5.2 License Plates Unreadable at Night

| Requirement       | Spec                                                      |
| ----------------- | --------------------------------------------------------- |
| **Shutter Speed** | 1/100s minimum (1/250s ideal) — set in Exposure → Shutter |
| **Resolution**    | 4K/8MP + optical zoom OR 2MP LPR camera                   |
| **IR Range**      | Plate distance ≤ 50% of camera IR rating                  |
| **WDR**           | 120 dB+ (handles headlights)                              |
| **Angle**         | ≤30° off perpendicular (plate face-on)                    |

**Fix:** Increase shutter → image darker but plate sharp. Add external IR if too dark.

### 5.3 Daytime Image: Hazy / Low Contrast / Color Cast

| Cause                    | Fix                                                   |
| ------------------------ | ----------------------------------------------------- |
| Dome dirty / scratched   | Clean microfiber; replace dome ($15)                  |
| IR Cut Filter Stuck      | Power cycle; firmware update; tap gently (mechanical) |
| White Balance Wrong      | Set "Outdoor" / "Auto" / Custom Kelvin (5500K)        |
| Backlight (sun in frame) | Reposition; enable WDR; add sun shield                |

### 5.4 Flickering / Rolling Lines (50/60 Hz)

| Cause                             | Fix                                                     |
| --------------------------------- | ------------------------------------------------------- |
| **Power Line Frequency Mismatch** | Camera set to 60 Hz (US) / 50 Hz (EU) — match your grid |
| Ground Loop                       | All equipment same ground; PoE isolator (Transtector)   |
| Fluorescent/LED Flicker           | Set shutter to 1/100 (50 Hz) or 1/120 (60 Hz)           |

## 6. Remote Access & Mobile App

### 6.1 App Works on Wi-Fi, Fails on LTE/5G

| Architecture                                  | Fix                                                                |
| --------------------------------------------- | ------------------------------------------------------------------ |
| **Vendor Cloud** (Reolink, UniFi, Eufy, Ring) | Check internet on phone; vendor status page                        |
| **Port Forward**                              | **Don't use.** CGNAT, double NAT, ISP blocks. Switch to VPN/Cloud. |
| **VPN (Tailscale, WireGuard)**                | Ensure phone VPN active; split-tunnel may exclude app              |
| **Cloudflare Tunnel**                         | Verify `cloudflared` service healthy                               |

### 6.2 "Connection Failed" / Timeout on Remote

| Test                                | Expected  | Fix                     |
| ----------------------------------- | --------- | ----------------------- |
| Ping NVR LAN IP from phone (VPN on) | &lt;50 ms | VPN routing issue       |
| `curl http://nvr:80` from phone     | HTML      | Port blocked / firewall |
| Vendor cloud status page            | Green     | Wait / use local VPN    |

### 6.3 Live View Works, Playback Fails Remotely

- Playback uses different port/stream (often HTTPS/443 vs RTSP/554)
- **Check:** NVR → Network → HTTPS Port (default 443) accessible via VPN
- **Bandwidth:** Playback needs sustained 4–8 Mbps; LTE may throttle

## 7. PTZ & Mechanical

### 7.1 PTZ Won't Move / Stuck / Grinding

| Cause                            | Fix                                                   |
| -------------------------------- | ----------------------------------------------------- |
| Mechanical limit reached         | Check preset limits; reset to home                    |
| Slip ring failure (360° endless) | Cable harness worn → replace camera (not repairable)  |
| Ice / Debris                     | Heater failed; clear manually; verify heater circuit  |
| Protocol Mismatch                | Pelco-D/P vs ONVIF PTZ → Match NVR protocol to camera |

### 7.2 Preset Tour Not Running

- Schedule enabled? (NVR → PTZ → Tour Schedule)
- Dwell time >0? (10–30s per preset)
- Home preset defined? (Tour returns to home between loops)

## 8. Audio (Two-Way)

### 8.1 No Audio / One-Way Only

| Check                | Fix                                                |
| -------------------- | -------------------------------------------------- |
| App permission       | Phone Settings → App → Microphone: ON              |
| Camera audio enabled | Camera web UI → Audio → Input/Output: ON           |
| Codec mismatch       | G.711 (PCM) most compatible; try G.726 / AAC       |
| Half-duplex vs Full  | Most cams = PTT (half); don't talk while listening |

### 8.2 Echo / Feedback / Static

| Cause                               | Fix                                                              |
| ----------------------------------- | ---------------------------------------------------------------- |
| Speaker too loud / mic gain high    | Lower speaker volume in app; reduce mic gain                     |
| No AEC (Acoustic Echo Cancellation) | Full-duplex cams need AEC; half-duplex = mute mic while speaking |
| Wind on mic                         | Enable wind filter; foam windscreen; reposition                  |

## 9. Firmware & Updates

### 9.1 Safe Firmware Update Procedure

1. **Read changelog** — Skip if "security only" and you're stable
2. **Backup config** — NVR: Export config; Camera: Screenshot settings
3. **Update NVR first** — Then cameras (NVR may push compatible firmware)
4. **One camera at a time** — Verify each before next
5. **Don't power off** — 5–10 min per camera
6. **Verify** — Live view, recording, motion, PTZ, audio

### 9.2 Rollback Failed Firmware

| Brand               | Method                                                        |
| ------------------- | ------------------------------------------------------------- |
| **Reolink**         | Firmware page → "Previous Version" button                     |
| **Dahua/Hikvision** | TFTP recovery (boot + hold reset + TFTP server)               |
| **UniFi**           | Protect → Devices → Camera → "Rollback Firmware"              |
| **Generic**         | SD card with `firmware.bin` in root; power on with reset held |

## 10. Environmental & Physical

### 10.1 Water Ingress / Condensation

| Symptom              | Fix                                                                 |
| -------------------- | ------------------------------------------------------------------- |
| Fog inside dome      | Desiccant pack (replace annually); verify vent plug open (Gore-Tex) |
| Water at cable entry | Silicone + grommet; drip loop; conduit body                         |
| Corrosion on RJ45    | Dielectric grease on contacts; replace connector; shielded cable    |

### 10.2 Insects / Spiders / Birds

| Deterrent                             | Effectiveness                   |
| ------------------------------------- | ------------------------------- |
| Peppermint oil (cotton ball near cam) | High (spiders hate)             |
| Ultrasonic pest repeller              | Medium                          |
| Bird spikes on bracket                | High (birds)                    |
| Fleece cover over IR ring             | High (spiders build on warm IR) |
| Regular cleaning (monthly)            | Essential                       |

### 10.3 Vandalism / Theft

- **Torx / Security screws** (included with most pro cams)
- **Conduit** to camera (can't cut cable easily)
- **Height** >10 ft (requires ladder)
- **Backup camera** watching the camera (overlap)
- **Tamper Alert** (NVR: Camera → Tamper Detection → Alarm)

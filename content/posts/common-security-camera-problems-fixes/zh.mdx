每个摄像头系统都会出问题。本指南涵盖 25 个最常见问题的分步诊断和修复 — 按症状分类, 让您快速找到解决方案。

## 快速诊断流程图

```
摄像头不工作?
    │
    ├─ 无电源指示灯? → 供电 / PoE / 电池 (第 1 节)
    ├─ 有电无画面? → 线缆 / 连接 / IP 冲突 (第 2 节)
    ├─ 有画面不录制? → NVR / 存储 / 计划 (第 3 节)
    ├─ 有画面误报? → 动作区域 / 灵敏度 / AI (第 4 节)
    ├─ 夜视差? → IR / 反射 / 快门 (第 5 节)
    └─ 远程访问失败? → 端口映射 / VPN / 厂商云 (第 6 节)
```

## 1. 供电与启动问题

### 1.1 摄像头死机 — 无指示灯, 无响应

| 步骤 | 检查 | 修复 |
|------|-------|-----|
| 1 | **PoE 交换机端口灯** | 灭 = 端口死 / 预算超限 → 换已知好端口 |
| 2 | **PoE 注入器指示灯** | 灭 = 注入器坏 / 无交流电 → 换注入器 ($25) |
| 3 | **摄像头端电压** | 万用表: 44–57V (PoE), 12V±10% (DC) → 低 = 压降 / 坏线 |
| 4 | **线缆通断** | 测试仪: 8 根全通 → 断对 = 两头重压 |
| 5 | **摄像头复位** | 按住 reset 10–30 秒 (查手册) → App/NVR 重新领养 |

**电池摄像头:** 先用 USB-C 充 24 小时再下定论。Li-ion BMS 深放电后锁死。

### 1.2 夜间重启 (IR 开启时)

**原因:** IR 灯启动增加 1–2W 功耗导致电压跌落。

| 诊断 | 修复 |
|-----------|-----|
| PoE 交换机预算满 | 换注入器; 升级交换机 |
| Cat6 布线 >250 英尺 | 缩短布线; Cat6a; 摄像头端 PoE++ 注入器 |
| CCA (铜包铝) 线缆 | **换纯铜 Cat6** |
| 电池机型: Li-ion 低温跌压 | LiFePO₄ 升级; 保温箱; 拿进屋充电 |

### 1.3 PoE 交换机过热 / 端口跳动

- **查 PoE 预算:** Σ 摄像头最大功耗 ≤ 交换机预算 × 0.8
- **散热:** 交换机在封闭机柜 → 加风扇或移通风机架
- **固件:** 升级交换机固件 (PoE 协商 Bug 常见)

## 2. 网络与连通性

### 2.1 App/NVR 显示摄像头 "离线"

| 步骤 | 测试 | 修复 |
|------|------|-----|
| 1 | PC ping 摄像头 IP | 超时 = 第 1/2 层问题 (线缆、交换机、IP) |
| 2 | 从摄像头 ping 网关 (NVR shell) | 超时 = VLAN / 防火墙 / 路由 |
| 3 | `arp -a` → MAC 匹配摄像头? | IP 冲突 → 静态 IP / DHCP 保留 |
| 4 | DHCP 租约过期 | 静态 IP 或 DHCP 保留 (路由器) |
| 5 | IP 冲突 | 两设备同 IP → 保留唯一 IP |

### 2.2 Wi-Fi 摄像头天天掉线

| 诊断 | 修复 |
|-----------|-----|
| RSSI < -75 dBm | 移 AP 近; 加有线 AP; 查信道 |
| 2.4 GHz 拥塞 | 仅信道 1/6/11; 20 MHz 宽度; 关 band-steering |
| DFS 信道 (5 GHz) | 强制 2.4 GHz; 5 GHz 雷达 = 断连 |
| 省电模式 | App 里关摄像头 Wi-Fi 省电模式 |
| 路由器定时重启 | 关自动重启; 路由器上 UPS |

### 2.3 RTSP 流打不开 (VLC / Frigate / NVR)

```
测试 URL: rtsp://user:pass@192.168.1.100:554/stream1
```

| 错误 | 原因 | 修复 |
|-------|-------|-----|
| 401 Unauthorized | 用户名/密码错 | 查 NVR/App: 常为 `admin` / `123456` 或空 |
| 404 Not Found | 路径错 | 厂商路径: Hikvision `/Streaming/Channels/101`, Dahua `/cam/realmonitor?channel=1&subtype=0`, Reolink `/h264Preview_01_main` |
| Connection Refused | RTSP 未启用 | 摄像头 Web UI → Network → RTSP 开启 |
| Timeout | 防火墙 / VLAN | 放行 TCP 554 从 NVR VLAN 到 Camera VLAN |

**品牌 RTSP 路径 (常用):**
- **Reolink:** `/h264Preview_01_main` (主), `/h264Preview_01_sub` (子)
- **Dahua/Amcrest:** `/cam/realmonitor?channel=1&subtype=0`
- **Hikvision:** `/Streaming/Channels/101` (主), `/Streaming/Channels/102` (子)
- **UniFi:** `rtsp://user:pass@ip:7447/...` (Protect 设置 → RTSP)
- **Amcrest:** 同 Dahua (OEM)

## 3. 录像与存储

### 3.1 NVR 不录制 / 时间轴有空洞

| 检查 | 修复 |
|-------|-----|
| HDD 健康 (S.M.A.R.T.) | 重映射扇区 >0 → **立即换盘** |
| HDD 满 / 覆盖关 | 存储设置开 "覆盖最旧" |
| 计划配置错 | 验证: 主流 24/7 连续; 子流动检 (省空间) |
| 流不匹配 | NVR 期望 Main; 摄像头发 Sub → 对齐编码设置 |
| 时间戳漂移 | NVR + 摄像头同开 NTP (同一服务器: `pool.ntp.org`) |

### 3.2 microSD 不录制 / 损坏

| 症状 | 原因 | 修复 |
|---------|-------|-----|
| "需格式化" 循环 | 卡坏 / FS 错 | 换 **U3/V30 A2** (SanDisk Extreme, Samsung Pro) |
| 文件 0 字节 / 无法播放 | 断电中写入 | App 开 "安全弹出"; 插电机型加 UPS |
| 卡在 4 GB 停 | FAT32 限制 | 摄像头应自动 exFAT; 不行则 PC 格 exFAT |
| 循环不工作 | 满 / 锁住 | 查锁开关; 删旧文件; 摄像头内格式化 |

### 3.3 回放卡顿 / 像素化 / 绿屏

| 原因 | 修复 |
|-------|-----|
| H.265/HEVC 不支持 | 播放器需 HEVC 解码 (Win: $0.99 Store; VLC: 自带) |
| 选了子流 | 回放切主流 |
| 关键帧间隔太大 | Keyframe Interval = FPS (如 15 fps → 15) |
| VBR 码率尖峰 | 设 CBR 或 VBR Max Cap (4K 4–6 Mbps) |

## 4. 动作检测与提醒

### 4.1 假警报刷屏 (垃圾信息)

| 设置 | 推荐 | 原因 |
|---------|-------------|-----|
| **检测类型** | 仅人 + 车 | 排除动物、树、影子、雨 |
| **运动区域** | 仅画目标区 | 排除街道、树、邻居、天空 |
| **灵敏度** | 中 (起步) → 调 | 高 = 万物触发 |
| **重触发间隔** | 30–60 秒 | 防同一车产 50 个片段 |
| **物体大小过滤** | 最小 5% / 最大 50% 帧 | 忽略虫、鸟、远处车 |
| **AI/深度学习** | 开启 (人/车/包裹) | 90%+ 误报降低 |

### 4.2 漏报 (该报警时没报)

| 检查 | 修复 |
|-------|-----|
| 区域覆盖事件区? | 重画区域更宽; 实地走测 |
| 灵敏度太低? | 调高一档; 测试 |
| 夜间 "人" 未检出 | IR 反射干扰 AI → 向下倾摄像头; 加外置 IR |
| 摄像头处于 "睡眠" | 电池款: 关睡眠模式 / 增加 PIR 灵敏度 |
| NVR 端动检关 | NVR 端开启 (摄像头侧可能关) |

### 4.3 提醒延迟 >10 秒

| 原因 | 修复 |
|-------|-----|
| 云中转 (Ring, Arlo, Blink) | 天生 — 用本地 NVR/Frigate <1s |
| 弱 Wi-Fi (RSSI < -75) | 改善信号 (第 2.2 节) |
| NVR CPU 满 | 减少分析通道; 升级 NVR |
| MQTT/Webhook 延迟 | 先测 App 直接推送 |

## 5. 画质与夜视

### 5.1 夜视发白 / 灰蒙

**原因:** IR 灯反射自附近表面 (墙、檐、玻璃、蜘蛛网)。

| 修复 | 做法 |
|-----|-----|
| **向下倾 15–30°** | 调托架; 摄像头向下看, 不对墙 |
| **摄像头外移** | 离墙/檐 12–18" (延伸臂) |
| **关内置 IR** | 加 **外置 IR 补光灯** (分离, 对准目标) |
| **清洁球罩/镜头** | 蜘蛛网 = IR 散射 → 超细纤维 + 镜头清洁剂 |
| **Smart IR (如支持)** | 按距离自动调 IR 强度 |

### 5.2 车牌夜间不可读

| 要求 | 规格 |
|-------------|------|
| **快门速度** | 1/100s 起 (1/250s 理想) — Exposure → Shutter 设 |
| **分辨率** | 4K/8MP + 光学变焦 OR 2MP LPR 专用摄像头 |
| **IR 范围** | 车牌距离 ≤ 摄像头 IR 额定值 50% |
| **WDR** | 120 dB+ (抗车灯) |
| **角度** | ≤30° 偏垂直 (车牌正对) |

**修复:** 加快快门 → 画面变暗但车牌锐利. 太暗加外置 IR.

### 5.3 白天画面: 雾蒙 / 低对比度 / 色偏

| 原因 | 修复 |
|-------|-----|
| 球罩脏/划伤 | 超细纤维擦; 换球罩 ($15) |
| IR Cut 卡住 | 断电重启; 固件升级; 轻敲 (机械卡滞) |
| 白平衡错 | "Outdoor" / "Auto" / 自定义 Kelvin (5500K) |
| 逆光 (阳光入框) | 重新定位; 开 WDR; 加遮阳罩 |

### 5.4 闪烁 / 滚动条纹 (50/60 Hz)

| 原因 | 修复 |
|-------|-----|
| **电网频率不匹配** | 摄像头设 60 Hz (美) / 50 Hz (欧) — 匹配当地电网 |
| 接地环路 | 所有设备共地; PoE 隔离器 (Transtector) |
| 荧光/LED 灯闪烁 | 快门设 1/100 (50 Hz) 或 1/120 (60 Hz) |

## 6. 远程访问与手机 App

### 6.1 Wi-Fi 下 App 正常, LTE/5G 失败

| 架构 | 修复 |
|--------------|-----|
| **厂商云** (Reolink, UniFi, Eufy, Ring) | 检查手机网; 厂商状态页 |
| **端口映射** | **别用.** CGNAT, 双层 NAT, ISP 拦截. 改 VPN/云. |
| **VPN (Tailscale, WireGuard)** | 确认手机 VPN 开; split-tunnel 可能排除 App |
| **Cloudflare Tunnel** | 确认 `cloudflared` 服务健康 |

### 6.2 远程 "连接失败" / 超时

| 测试 | 预期 | 修复 |
|------|----------|-----|
| 手机 VPN 开 ping NVR 内网 IP | <50 ms | VPN 路由问题 |
| 手机 `curl http://nvr:80` | HTML | 端口被挡 / 防火墙 |
| 厂商云状态页 | 绿 | 等 / 用本地 VPN |

### 6.3 实时看正常, 回放远程失败

- 回放用不同端口/流 (常 HTTPS/443 vs RTSP/554)
- **查:** NVR → Network → HTTPS Port (默认 443) VPN 下可达
- **带宽:** 回放需持续 4–8 Mbps; LTE 可能限速

## 7. PTZ 与机械

### 7.1 PTZ 不动 / 卡住 / 响声

| 原因 | 修复 |
|-------|-----|
| 机械限位到达 | 查预设限位; 复位 home |
| 集电环故障 (360° 无限) | 线束磨损 → 换摄像头 (不可修) |
| 冰 / 杂物 | 加热器坏; 手动清; 验证加热电路 |
| 协议不匹配 | Pelco-D/P vs ONVIF PTZ → NVR 协议匹配摄像头 |

### 7.2 预设巡航不跑

- Schedule 开了? (NVR → PTZ → Tour Schedule)
- Dwell time >0? (每预设 10–30s)
- Home preset 定义了? (巡航间回 home)

## 8. 音频 (双向)

### 8.1 无声音 / 单向

| 检查 | 修复 |
|-------|-----|
| App 权限 | 手机 Settings → App → Microphone: ON |
| 摄像头音频开 | 摄像头 Web UI → Audio → Input/Output: ON |
| 编解码不匹配 | G.711 (PCM) 最兼容; 试 G.726 / AAC |
| 半双工 vs 全双工 | 大多 = PTT (半双工); 听时别说话 |

### 8.2 回声 / 啸叫 / 杂音

| 原因 | 修复 |
|-------|-----|
| 扬声器太大 / 麦克风增益高 | App 降扬声器音量; 降麦克风增益 |
| 无 AEC (声学回声消除) | 全双工需 AEC; 半双工 = 说时静麦 |
| 风吹麦克风 | 开风噪滤波; 泡沫防风罩; 换位置 |

## 9. 固件与更新

### 9.1 安全固件升级流程

1. **读 changelog** — "仅安全" 且稳定则跳过
2. **备份配置** — NVR: 导出配置; 摄像头: 截图设置
3. **先升 NVR** — 再升摄像头 (NVR 可能推兼容固件)
4. **逐台升** — 每台验证后再下一台
5. **别断电** — 单台 5–10 分钟
6. **验证** — Live view, 录制, 动检, PTZ, 音频

### 9.2 失败固件回滚

| 品牌 | 方法 |
|-------|--------|
| **Reolink** | 固件页 → "Previous Version" 按钮 |
| **Dahua/Hikvision** | TFTP 恢复 (开机 + 按住 reset + TFTP 服务器) |
| **UniFi** | Protect → Devices → Camera → "Rollback Firmware" |
| **通用** | SD 卡根目录放 `firmware.bin`; 按住 reset 上电 |

## 10. 环境与物理

### 10.1 进水 / 冷凝

| 症状 | 修复 |
|---------|-----|
| 球罩内起雾 | 干燥剂包 (年换); 验证透气塞开 (Gore-Tex) |
| 线缆入口进水 | 硅胶 + 胶圈; 下垂环; 管道盒 |
| RJ45 锈蚀 | 触点涂介电脂; 换连接器; 屏蔽线 |

### 10.2 昆虫 / 蜘蛛 / 鸟

| 驱赶 | 效果 |
|-----------|---------------|
| 薄荷油 (棉球贴摄像头旁) | 高 (蜘蛛怕) |
| 超声波驱虫器 | 中 |
| 支架上防鸟刺 | 高 (鸟) |
| IR 环上抓绒套 | 高 (蜘蛛爱暖 IR 筑网) |
| 定期清洁 (月度) | 必不可少 |

### 10.3 破坏 / 偷盗

- **Torx / 安全螺丝** (多数专业机附带)
- **管道** 到摄像头 (不易剪线)
- **高度** >10 ft (需梯子)
- **备用摄像头** 监视摄像头 (重叠)
- **防拆报警** (NVR: Camera → Tamper Detection → Alarm)

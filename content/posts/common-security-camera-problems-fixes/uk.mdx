Кожна система камер з часом має проблеми. Цей посібник охоплює 25 найпоширеніших проблем з покроковою діагностикою та виправленнями — організовано за симптомами, щоб ви швидко знайшли рішення.

живлення або кабель.

## Швидка діагностична блок-схема

```
Камера не працює?
    │
    ├─ Немає LED живлення? → Блок живлення / PoE / Батарея (Розділ 1)
    ├─ Живлення OK, немає зображення? → Кабель / З'єднання / IP конфлікт (Розділ 2)
    ├─ Зображення OK, немає запису? → NVR / Сховище / Розклад (Розділ 3)
    ├─ Зображення OK, хибні сповіщення? → Зони руху / Чутливість / AI (Розділ 4)
    ├─ Нічне бачення погане? → ІЧ / Відбиття / Витримка (Розділ 5)
    └─ Віддалений доступ не працює? → Порт форвардинг / VPN / Хмара (Розділ 6)
```

## 1. Проблеми живлення та запуску

### 1.1 Камера мертва — немає LED, немає реакції

| Крок | Перевірка                    | Виправлення                                                                        |
| ---- | ---------------------------- | ---------------------------------------------------------------------------------- |
| 1    | **LED порту PoE комутатора** | Вимкнено = порт мертвий / бюджет перевищено → Перемкніть на справний порт          |
| 2    | **LED PoE інжектора**        | Вимкнено = інжектор несправний / немає AC → Замініть інжектор ($25)                |
| 3    | **Напруга на камері**        | Мультиметр: 44–57V (PoE), 12V±10% (DC) → Низька = падіння напруги / поганий кабель |
| 4    | **Цілісність кабелю**        | Тестер: всі 8 дротів → Відкрита пара = перетермінуйте обидва кінці                 |
| 5    | **Скидання камери**          | Тримайте скидання 10–30с → Повторно додайте в додатку/NVR                          |

**Акумуляторні камери:** Заряджайте 24 год через USB-C перед оголошенням мертвою. BMS Li-ion блокується після глибокого розряду.

### 1.2 Камера перезавантажується вночі (ІЧ увімкнено)

**Причина:** Падіння напруги, коли ІЧ-світлодіоди вмикаються (додають 1–2W).

| Діагностика                      | Виправлення                                          |
| -------------------------------- | ---------------------------------------------------- |
| PoE комутатор на межі бюджету    | Перемкніть камеру на інжектор; оновіть комутатор     |
| Кабель >75 м на Cat6             | Скоротіть кабель; Cat6a; PoE++ інжектор біля камери  |
| CCA кабель (алюміній)            | **Замініть на суцільний мідний Cat6**                |
| Акумулятор: Li-ion холодний спад | LiFePO₄ оновлення; ізольований бокс; занесіть додому |

### 1.3 PoE комутатор перегрівається / порти flapping

- **Перевірте бюджет PoE:** Σ макс. споживання камер ≤ бюджет комутатора × 0.8
- **Температура:** Комутатор у закритій шафі → додайте вентилятор або перемістіть

## 2. Мережа та з'єднання

### 2.1 Камера "Офлайн" у додатку/NVR

| Крок | Тест                                 | Виправлення                                       |
| ---- | ------------------------------------ | ------------------------------------------------- |
| 1    | Ping IP камери з ПК                  | Тайм-аут = проблема рівня 1/2 (кабель, комутатор) |
| 2    | Ping шлюзу з камери (оболонка NVR)   | Тайм-аут = VLAN / брандмауер / маршрутизація      |
| 3    | `arp -a` → MAC збігається з камерою? | IP конфлікт → Статичний IP / DHCP резервація      |
| 4    | Термін дії DHCP минув                | Статичний IP або DHCP резервація (роутер)         |
| 5    | IP конфлікт                          | Два пристрої з одним IP → Призначте унікальні IP  |

### 2.2 Wi-Fi камера відключається щодня

| Діагностика            | Виправлення                                                 |
| ---------------------- | ----------------------------------------------------------- |
| RSSI < -75 dBm         | Перемістіть AP ближче; додайте дротовий AP; перевірте канал |
| Завантаженість 2.4 ГГц | Канали 1/6/11; ширина 20 МГц; вимкніть band-steering        |
| DFS канал (5 ГГц)      | Примусово 2.4 ГГц; 5 ГГц радар = відключення                |
| Режим енергозбереження | Вимкніть Wi-Fi енергозбереження в додатку камери            |

### 2.3 RTSP потік не відкривається (VLC / Frigate / NVR)

```
Тест URL: rtsp://user:pass@192.168.1.100:554/stream1
```

| Помилка            | Причина         | Виправлення                                                                                                       |
| ------------------ | --------------- | ----------------------------------------------------------------------------------------------------------------- |
| 401 Unauthorized   | Невірний пароль | Перевірте NVR/додаток: часто `admin` / `123456`                                                                   |
| 404 Not Found      | Невірний шлях   | Hikvision `/Streaming/Channels/101`, Dahua `/cam/realmonitor?channel=1&subtype=0`, Reolink `/h264Preview_01_main` |
| Connection Refused | RTSP вимкнено   | Увімкніть у веб-інтерфейсі камери → Мережа → RTSP                                                                 |
| Timeout            | Брандмауер/VLAN | Дозвольте TCP 554 з NVR VLAN до камери VLAN                                                                       |

**Типові RTSP шляхи:**

- **Reolink:** `/h264Preview_01_main` (основний), `/h264Preview_01_sub` (додатковий)
- **Dahua/Amcrest:** `/cam/realmonitor?channel=1&subtype=0`
- **Hikvision:** `/Streaming/Channels/101` (основний), `/Streaming/Channels/102` (додатковий)
- **UniFi:** `rtsp://user:pass@ip:7447/...`

## 3. Запис та сховище

### 3.1 NVR не записує / прогалини в таймлайні

| Перевірка                      | Виправлення                                                          |
| ------------------------------ | -------------------------------------------------------------------- |
| HDD Health (S.M.A.R.T.)        | Перерозподілені сектори >0 → **Замініть негайно**                    |
| HDD заповнений / оверрайт вимк | Увімкніть "Перезапис найстарішого"                                   |
| Неправильний розклад           | Перевірте: Безперервний 24/7 на основному потоці; Рух на додатковому |
| Розбіжність потоків            | NVR очікує основний; камера надсилає додатковий                      |
| Дрейф часової мітки            | Увімкніть NTP на NVR + камери (той самий сервер)                     |

### 3.2 microSD не записує / пошкоджена

| Симптом                         | Причина                        | Виправлення                                              |
| ------------------------------- | ------------------------------ | -------------------------------------------------------- |
| Цикл "Format Required"          | Карта несправна / FS           | Замініть на **U3/V30 A2** (SanDisk Extreme, Samsung Pro) |
| Файли 0 байт / не відтворюються | Втрата живлення під час запису | Увімкніть "Safe Eject" в додатку; UPS для камер          |
| Зупиняється на 4 ГБ             | Обмеження FAT32                | Камера має автоматично exFAT                             |

### 3.3 Відтворення заморожене / піксельне / зелений екран

| Причина                            | Виправлення                                          |
| ---------------------------------- | ---------------------------------------------------- |
| H.265/HEVC не підтримується        | Плеєру потрібен кодек HEVC (VLC: вбудований)         |
| Вибрано додатковий потік           | Перемкніться на основний потік у відтворенні         |
| Інтервал ключових кадрів завеликий | Встановіть інтервал = FPS (напр., 15 fps → 15)       |
| Сплески VBR бітрейту               | Встановіть CBR або обмеження VBR (4–6 Мбіт/с для 4K) |

## 4. Виявлення руху та сповіщення

### 4.1 Забагато хибних сповіщень (спам)

| Налаштування         | Рекомендовано                | Чому                                    |
| -------------------- | ---------------------------- | --------------------------------------- |
| **Тип виявлення**    | Людина + Транспорт           | Виключає тварин, дерева, тіні, дощ      |
| **Зони руху**        | Малюйте ТІЛЬКИ цільову зону  | Виключіть вулицю, дерева, сусідів, небо |
| **Чутливість**       | Середня (старт) → налаштуйте | Висока = все спрацьовує                 |
| **Інтервал повтору** | 30–60 секунд                 | Запобігає 50 кліпам однієї машини       |
| **AI/Deep Learning** | Увімкніть (Людина/Транспорт) | Скорочення хибних на 90%+               |

### 4.2 Пропущені події (немає сповіщення, коли щось сталося)

| Перевірка                   | Виправлення                                                    |
| --------------------------- | -------------------------------------------------------------- |
| Зона покриває місце події?  | Перемалюйте зону ширше; пройдіться для тесту                   |
| Чутливість занадто низька?  | Збільшіть на один крок; тестуйте                               |
| "Людину" не виявлено вночі  | Відбиття ІЧ плутає AI → нахиліть камеру вниз; додайте зовн. ІЧ |
| Камера в режимі "Сон"       | Акумуляторні камери: вимкніть сон / збільшіть чутливість PIR   |
| Виявлення руху NVR вимкнено | Увімкніть на стороні NVR                                       |

### 4.3 Затримка сповіщення >10 секунд

| Причина                          | Виправлення                                     |
| -------------------------------- | ----------------------------------------------- |
| Хмарна ретрансляція (Ring, Arlo) | Властиво — використовуйте локальний NVR/Frigate |
| Слабкий Wi-Fi (RSSI < -75)       | Покращте сигнал                                 |
| NVR CPU на максимумі             | Зменшіть канали аналітики; оновіть NVR          |

## 5. Якість зображення та нічне бачення

### 5.1 Нічне засвічення / розмиття

**Причина:** ІЧ-світлодіоди відбиваються від близької поверхні.

| Виправлення                       | Як                                                               |
| --------------------------------- | ---------------------------------------------------------------- |
| **Нахиліть вниз 15–30°**          | Кронштейн з нахилом; камера дивиться вниз, не на стіну           |
| **Відсуньте камеру**              | 12–18" від стіни/карниза (подовжувач)                            |
| **Вимкніть вбудований ІЧ**        | Додайте **зовнішній ІЧ-прожектор** (окремо, спрямований на ціль) |
| **Протріть купол/лінзу**          | Павутина = розсіювання ІЧ → мікрофібра + засіб для лінз          |
| **Smart IR (якщо підтримується)** | Автоматично регулює інтенсивність ІЧ за відстанню                |

### 5.2 Номерні знаки нечитабельні вночі

| Вимога              | Специфікація                                            |
| ------------------- | ------------------------------------------------------- |
| **Витримка**        | 1/100с мінімум (1/250с ідеал) — встановіть в Експозиція |
| **Розд. здатність** | 4K/8MP + оптичний зум АБО 2MP LPR камера                |
| **Дальність ІЧ**    | Відстань до номера ≤ 50% від рейтингу ІЧ камери         |
| **WDR**             | 120 дБ+ (справляється з фарами)                         |
| **Кут**             | ≤30° від перпендикуляра                                 |

### 5.3 Денне зображення: серпанок / низький контраст / відтінок

| Причина                         | Виправлення                                                   |
| ------------------------------- | ------------------------------------------------------------- |
| Купол брудний / подряпаний      | Протріть мікрофіброю; замініть купол ($15)                    |
| IR Cut Filter застряг           | Перезавантажте; оновіть прошивку; обережно постукайте         |
| Неправильний баланс білого      | Встановіть "Outdoor" / "Auto" / Користувацький 5500K          |
| Контросвітлення (сонце в кадрі) | Змініть позицію; увімкніть WDR; додайте сонцезахисний козирок |

### 5.4 Мерехтіння / лінії (50/60 Гц)

| Причина                            | Виправлення                                                     |
| ---------------------------------- | --------------------------------------------------------------- |
| **Невідповідність частоти мережі** | Камера встановлена на 60 Гц (США) / 50 Гц (ЄС) — виберіть вашу  |
| Контур заземлення                  | Все обладнання на одному заземленні; PoE ізолятор (Transtector) |
| Мерехтіння флуоресцентних/LED      | Встановіть витримку 1/100 (50 Гц) або 1/120 (60 Гц)             |

## 6. Віддалений доступ та мобільний додаток

### 6.1 Додаток працює по Wi-Fi, не працює по LTE/5G

| Архітектура                                    | Виправлення                                                          |
| ---------------------------------------------- | -------------------------------------------------------------------- |
| **Хмара вендора** (Reolink, UniFi, Eufy, Ring) | Перевірте інтернет на телефоні; сторінка статусу вендора             |
| **Порт форвардинг**                            | **Не використовуйте.** CGNAT, подвійний NAT. Перейдіть на VPN/хмару. |
| **VPN (Tailscale, WireGuard)**                 | Переконайтеся, що VPN активний на телефоні                           |

### 6.2 "Помилка з'єднання" / тайм-аут віддалено

| Тест                                   | Очікується | Виправлення                   |
| -------------------------------------- | ---------- | ----------------------------- |
| Ping NVR LAN IP з телефону (VPN увімк) | <50 мс     | Проблема VPN маршрутизації    |
| `curl http://nvr:80` з телефону        | HTML       | Порт заблоковано / брандмауер |

### 6.3 Живий перегляд працює, відтворення не працює віддалено

- Відтворення використовує інший порт/потік (часто HTTPS/443 vs RTSP/554)
- Пропускна здатність: відтворення потребує 4–8 Мбіт/с; LTE може обмежувати

## 7. PTZ та механіка

### 7.1 PTZ не рухається / застряг / скрегоче

| Причина                            | Виправлення                                                |
| ---------------------------------- | ---------------------------------------------------------- |
| Досягнуто механічного обмеження    | Перевірте ліміти presetів; скиньте на домашню позицію      |
| Відмова slip ring (360° безмежний) | Зношений кабельний вузол → замініть камеру                 |
| Лід / сміття                       | Обігрівач несправний; очистіть вручну; перевірте обігрівач |
| Невідповідність протоколу          | Pelco-D/P vs ONVIF PTZ → збігайте протокол NVR з камерою   |

### 7.2 Preset-тур не працює

- Розклад увімкнено? (NVR → PTZ → Tour Schedule)
- Час очікування >0? (10–30с на preset)
- Домашній preset визначено?

## 8. Аудіо (двостороннє)

### 8.1 Немає аудіо / тільки один напрямок

| Перевірка              | Виправлення                                                 |
| ---------------------- | ----------------------------------------------------------- |
| Дозвіл додатку         | Налаштування телефону → Додаток → Мікрофон: Увімк           |
| Аудіо камери увімкнено | Веб-інтерфейс → Аудіо → Вхід/Вихід: Увімк                   |
| Напівдуплекс vs Повний | Більшість камер = PTT (напів); не говоріть під час слухання |

### 8.2 Відлуння / зворотній зв'язок / шум

| Причина                               | Виправлення                                                                                  |
| ------------------------------------- | -------------------------------------------------------------------------------------------- |
| Динамік занадто гучний / високий gain | Зменшіть гучність динаміка в додатку; зменшіть gain мікрофона                                |
| Немає AEC                             | Камери з повним дуплексом потребують AEC; напівдуплекс = вимкніть мікрофон під час говоріння |
| Вітер на мікрофоні                    | Увімкніть фільтр вітру; поролоновий вітрозахист; змініть позицію                             |

## 9. Прошивка та оновлення

### 9.1 Безпечна процедура оновлення прошивки

1. **Прочитайте список змін** — Пропустіть, якщо "тільки безпека" і ви стабільні
2. **Збережіть резервну копію конфігурації** — NVR: Експорт конфігурації; Камера: Скріншот налаштувань
3. **Оновіть NVR першим** — Потім камери (NVR може надсилати сумісну прошивку)
4. **Одну камеру за раз** — Перевірте кожну перед наступною
5. **Не вимикайте живлення** — 5–10 хв на камеру
6. **Перевірте** — Живий перегляд, запис, рух, PTZ, аудіо

### 9.2 Відкат невдалої прошивки

| Бренд               | Метод                                                       |
| ------------------- | ----------------------------------------------------------- |
| **Reolink**         | Сторінка прошивки → кнопка "Previous Version"               |
| **Dahua/Hikvision** | TFTP відновлення (завантаження + утримання скидання + TFTP) |
| **UniFi**           | Protect → Devices → Camera → "Rollback Firmware"            |

## 10. Навколишнє середовище та фізичне

### 10.1 Потрапляння води / конденсат

| Симптом                | Виправлення                                                              |
| ---------------------- | ------------------------------------------------------------------------ |
| Туман усередині купола | Пакет осушувача; перевірте вентиляційний отвір (Gore-Tex)                |
| Вода біля входу кабелю | Силікон + ущільнювач; drip петля; канальний корпус                       |
| Корозія на RJ45        | Діелектричне мастило на контактах; замініть конектор; екранований кабель |

### 10.2 Комахи / Павуки / Птахи

| Засіб                           | Ефективність                          |
| ------------------------------- | ------------------------------------- |
| М'ятна олія (ватка біля камери) | Висока (павуки не люблять)            |
| Ультразвуковий відлякувач       | Середня                               |
| Шипи для птахів на кронштейні   | Висока (птахи)                        |
| Флісова накладка на ІЧ-кільце   | Висока (павуки будують на теплому ІЧ) |
| Регулярне чищення (щомісяця)    | Обов'язково                           |

### 10.3 Вандалізм / Крадіжка

- **Гвинти Torx / Security** (включено з більшістю про-камер)
- **Канал** до камери (не можна легко перерізати кабель)
- **Висота** >3 м (потрібна драбина)
- **Резервна камера** спрямована на основну
- **Сповіщення про втручання** (NVR: Камера → Tamper Detection → Alarm)

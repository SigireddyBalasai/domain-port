Каждая система камер развивает проблемы. Это руководство покрывает 25 самых частых проблем с пошаговой диагностикой и исправлениями — организовано по симптомам, чтобы вы быстро находили решение.

## Быстрая диагностическая блок-схема

```
Камера не работает?
    │
    ├─ Нет индикатора питания? → Блок питания / PoE / Батарея (Раздел 1)
    ├─ Питание ок, нет изображения? → Кабель / Линк / IP конфликт (Раздел 2)
    ├─ Изображение ок, нет записи? → NVR / Хранилище / Расписание (Раздел 3)
    ├─ Изображение ок, ложные алерты? → Зоны движения / Чувствительность / AI (Раздел 4)
    ├─ Ночная визия плохая? → IR / Отражение / Выдержка (Раздел 5)
    └─ Удаленный доступ не работает? → Проброс портов / VPN / Вендорское облако (Раздел 6)
```

## 1. Проблемы с питанием и запуском

### 1.1 Камера мертва — нет светодиода, никакой реакции

| Шаг | Проверка | Исправление |
|------|-------|-----|
| 1 | **LED порта PoE свитча** | Не горит = порт мертв / бюджет превышен → Переставьте на известный хороший порт |
| 2 | **LED PoE инжектора** | Не горит = инжектор умер / нет AC → Замените инжектор ($25) |
| 3 | **Напряжение на камере** | Мультиметр: 44–57V (PoE), 12V±10% (DC) → Низкое = падение напряжения / плохой кабель |
| 4 | **Прерывность кабеля** | Тестер: все 8 проводов → Обрыв пары = переобжать оба конца |
| 5 | **Сброс камеры** | Зажмите reset 10–30с (смотрите мануал) → Перепривяжите в app/NVR |

**Батарейные камеры:** Зарядите 24 ч по USB-C перед приговором. Li-ion BMS может заблокироваться после глубокого разряда.

### 1.2 Камера перезагружается ночью (IR вкл)

**Причина:** IR светодиоды добавляют 1–2W потребления → просадка напряжения.

| Диагноз | Исправление |
|-----------|-----|
| PoE свитч на пределе бюджета | Перенесите на инжектор; обновите свитч |
| Кабель >250 футов на Cat6 | Сократите пролёт; Cat6a; PoE++ инжектор у камеры |
| CCA (медная оболочка алюминий) кабель | **Замените на цельную медь Cat6** |
| Батарейная: Li-ion холодное падение напряжения | LiFePO₄ апгрейд; изолированная коробка; заряжайте внутри |

### 1.3 PoE свитч перегревается / порты дергаются

- **Проверьте PoE Бюджет:** Σ макс. потребление камер ≤ бюджет свитча × 0.8
- **Тепло:** Свитч в закрытом шкафу → добавьте вентилятор или перенесите в вентилируемую стойку
- **Прошивка:** Обновите прошивку свитча (баги PoE переговоров часты)

## 2. Сеть и связь

### 2.1 Камера "Offline" в App/NVR

| Шаг | Тест | Исправление |
|------|------|-----|
| 1 | Ping IP камеры с ПК | Таймаут = L1/2 проблема (кабель, свитч, IP) |
| 2 | Ping шлюза из камеры (NVR shell) | Таймаут = VLAN / фаервол / маршрутизация |
| 3 | `arp -a` → MAC совпадает с камерой? | IP конфликт → Статический IP / DHCP резерв |
| 4 | DHCP лицензия истекла | Статический IP или DHCP резерв (роутер) |
| 5 | IP конфликт | Два устройства с одним IP → Зарезервируйте уникальные IP |

### 2.2 Wi-Fi камера отваливается каждый день

| Диагноз | Исправление |
|-----------|-----|
| RSSI < -75 dBm | Сдвиньте AP ближе; добавьте проводной AP; проверьте канал |
| 2.4 GHz загрузка | Только каналы 1/6/11; 20 MHz ширина; выключите band-steering |
| DFS канал (5 GHz) | Принудительно 2.4 GHz; 5 GHz radar = отключение |
| Режим экономии энергии | Отключите Wi-Fi power save в приложении камеры |
| Расписание перезагрузки роутера | Отключите авто-перезагрузку; UPS на роутере |

### 2.3 RTSP поток не открывается (VLC / Frigate / NVR)

```
Тестовый URL: rtsp://user:pass@192.168.1.100:554/stream1
```

| Ошибка | Причина | Исправление |
|-------|-------|-----|
| 401 Unauthorized | Неверный user/pass | Проверьте NVR/app: часто `admin` / `123456` или пусто |
| 404 Not Found | Неправильный путь | Брендовые пути: Hikvision `/Streaming/Channels/101`, Dahua `/cam/realmonitor?channel=1&subtype=0`, Reolink `/h264Preview_01_main` |
| Connection Refused | RTSP отключен | Включите в веб UI камеры → Network → RTSP |
| Timeout | Фаервол / VLAN | Разрешите TCP 554 из NVR VLAN в Camera VLAN |

**Брендовые RTSP пути (частые):**
- **Reolink:** `/h264Preview_01_main` (main), `/h264Preview_01_sub` (sub)
- **Dahua/Amcrest:** `/cam/realmonitor?channel=1&subtype=0`
- **Hikvision:** `/Streaming/Channels/101` (main), `/Streaming/Channels/102` (sub)
- **UniFi:** `rtsp://user:pass@ip:7447/...` (Protect settings → RTSP)
- **Amcrest:** Тот же Dahua (OEM)

## 3. Запись и хранение

### 3.1 NVR не пишет / пробелы в таймлайне

| Проверка | Исправление |
|-------|-----|
| HDD Health (S.M.A.R.T.) | Переназначенные сектора >0 → **Меняйте немедленно** |
| HDD полен / Overwrite Off | Включите "Overwrite Oldest" в настройках хранилища |
| Расписание неверно | Проверьте: Continuous 24/7 на Main; Motion на Sub (экономия места) |
| Несоответствие потока | NVR ждет Main; камера шлет Sub → Согласуйте настройки кодирования |
| Дрейф таймстемпа | Включите NTP на NVR + Камерах (один сервер: `pool.ntp.org`) |

### 3.2 microSD не пишет / битый

| Симптом | Причина | Исправление |
|---------|-------|-----|
| "Требуется форматирование" в цикле | Карта умерла / не та ФС | Замените на **U3/V30 A2** (SanDisk Extreme, Samsung Pro) |
| Файлы 0 байт / не играют | Потеря питания при записи | Включите "Safe Eject" в app; UPS на plug-in камерах |
| Останавливается на 4 GB | FAT32 лимит | Камера должна сама exFAT; если нет — форматируйте exFAT на ПК |
| Loop не работает | Полна / залочена | Проверьте защелку; удалите старые файлы; форматируйте в камере |

### 3.3 Воспроизведение застыло / пикселизировано / зеленый экран

| Причина | Исправление |
|-------|-----|
| H.265/HEVC не поддерживается | Плееру нужен HEVC кодек (Win: $0.99 Store; VLC: встроен) |
| Выбран Sub-stream | Переключите на Main Stream в воспроизведении |
| Keyframe interval слишком велик | Set Keyframe Interval = FPS (напр., 15 fps → 15) |
| VBR скачки битрейта | CBR или VBR Max Cap (4–6 Mbps для 4K) |

## 4. Детекция движения и алерты

### 4.1 Слишком много ложных алертов (Спам)

| Настройка | Рекомендуется | Почему |
|---------|-------------|-----|
| **Detection Type** | Person + Vehicle only | Исключает животных, деревья, тени, дождь |
| **Motion Zones** | Рисуйте ТОЛЬКО целевую зону | Исключите улицу, деревья, соседа, небо |
| **Sensitivity** | Medium (старт) → adjust | High = все триггерит |
| **Retrigger Interval** | 30–60 seconds | Предотвращает 50 клипов от одной машины |
| **Object Size Filter** | Min 5% / Max 50% of frame | Игнорирует жуков, птиц, далекие машины |
| **AI/Deep Learning** | Включите (Person/Vehicle/Package) | 90%+ сокращение ложных алертов |

### 4.2 Пропущенные события (Нет алерта когда что-то случилось)

| Проверка | Исправление |
|-------|-----|
| Зона покрывает зону события? | Перерисуйте шире; пройдитесь тестом |
| Чувствительность слишком низка? | Поднимите на одну ноту; тест |
| "Person" не детектируется ночью | IR отражение путает AI → наклоните камеру вниз; добавьте внешний IR |
| Камера в режиме "Sleep" | Батарейные: выкл sleep / увеличьте PIR чувствительность |
| NVR motion detection OFF | Включите на стороне NVR (камерная сторона может быть выкл) |

### 4.3 Задержка алерта >10 секунд

| Причина | Исправление |
|-------|-----|
| Cloud relay (Ring, Arlo, Blink) | Природное — используйте локальный NVR/Frigate для <1с |
| Слабый Wi-Fi (RSSI < -75) | Улучшите сигнал (Раздел 2.2) |
| NVR CPU забит | Уменьшите аналитические каналы; апгрейд NVR |
| MQTT/Webhook задержка | Тестируйте прямой app push первым |

## 5. Качество изображения и ночная визия

### 5.1 Ночная визия белая / вымытая

**Причина:** IR светодиоды отражаются от ближайшей поверхности (стена, навес, стекло, паутина).

| Исправление | Как |
|-----|-----|
| **Наклоните вниз 15–30°** | Наклоните кронштейн; камера смотрит вниз, не в стену |
| **Отодвиньте камеру** | 12–18" от стены/навеса (удлинительная рука) |
| **Отключите встроенный IR** | Добавьте **внешний IR iluminator** (отдельный, нацелен на цель) |
| **Очистите купол/объектив** | Паутина = IR рассеяние → микрофибра + чистильщик для линз |
| **Smart IR (если есть)** | Авто-регулирует IR интенсивность по дистанции |

### 5.2 Номера машин нечитаемы ночью

| Требование | Спец |
|-------------|------|
| **Shutter Speed** | 1/100s минимум (1/250s идеально) — выставьте в Exposure → Shutter |
| **Разрешение** | 4K/8MP + оптический зум ИЛИ 2MP LPR камера |
| **IR Диапазон** | Дистанция номера ≤ 50% от IR рейтинга камеры |
| **WDR** | 120 dB+ (гасит фары) |
| **Угол** | ≤30° от перпендикуляра (номер лицом) |

**Фикс:** Увеличьте выдержку → картинка темнее но номер резкий. Добавьте внешний IR если слишком темно.

### 5.3 Дневное изображение: Туманное / Низкий контраст / Цветной налет

| Причина | Исправление |
|-------|-----|
| Купол грязный / поцарапан | Микрофибра; замена купола ($15) |
| IR Cut Filter Заедал | Power cycle; firmware update; легкое постукивание (механика) |
| White Balance Не тот | "Outdoor" / "Auto" / Custom Kelvin (5500K) |
| Контрсвет (солнце в кадре) | Перепозиционируйте; включите WDR; добавьте солнцезащитный козырек |

### 5.4 Мерцание / Роллинг линии (50/60 Hz)

| Причина | Исправление |
|-------|-----|
| **Несоответствие частоты сети** | Камера на 60 Hz (US) / 50 Hz (EU) — подстройте под свою сеть |
| Заземляющая петля | Все оборудование общее заземление; PoE изолятор (Transtector) |
| Мерцание люминесцентных/LED | Выдержка 1/100 (50 Hz) или 1/120 (60 Hz) |

## 6. Удаленный доступ и мобильное приложение

### 6.1 App работает на Wi-Fi, падает на LTE/5G

| Архитектура | Исправление |
|--------------|-----|
| **Vendor Cloud** (Reolink, UniFi, Eufy, Ring) | Проверьте интернет на телефоне; статус-страница вендора |
| **Port Forward** | **Не используйте.** CGNAT, double NAT, ISP блокирует. Переключитесь на VPN/Cloud. |
| **VPN (Tailscale, WireGuard)** | Убедитесь телефон VPN активен; split-tunnel может исключать app |
| **Cloudflare Tunnel** | Проверьте `cloudflared` сервис здоров |

### 6.2 "Connection Failed" / Timeout на удаленке

| Тест | Ожидаемо | Исправление |
|------|----------|-----|
| Ping NVR LAN IP с телефона (VPN on) | <50 ms | VPN routing issue |
| `curl http://nvr:80` с телефона | HTML | Порт заблокирован / фаервол |
| Vendor cloud status page | Green | Подождите / используйте локальный VPN |

### 6.3 Live View работает, Playback падает удаленно

- Playback использует другой порт/поток (часто HTTPS/443 vs RTSP/554)
- **Проверьте:** NVR → Network → HTTPS Port (default 443) доступен через VPN
- **Bandwidth:** Playback нужен sustained 4–8 Mbps; LTE может троттлить

## 7. PTZ и механика

### 7.1 PTZ не движется / застрял / скрипит

| Причина | Исправление |
|-------|-----|
| Механический предел достигнут | Проверьте пресетные лимиты; сброс в home |
| Slip ring failure (360° endless) | Кабельный жгут износился → меняйте камеру (не ремонтируется) |
| Лед / Мусор | Обогреватель сбоил; чистите вручную; проверьте цепь обогревателя |
| Protocol Mismatch | Pelco-D/P vs ONVIF PTZ → Согласуйте протокол NVR с камерой |

### 7.2 Preset Tour не запускается

- Schedule включен? (NVR → PTZ → Tour Schedule)
- Dwell time >0? (10–30s на пресет)
- Home preset задан? (Tour возвращается в home между циклами)

## 8. Аудио (Двустороннее)

### 8.1 Нет звука / Одностороннее

| Проверка | Исправление |
|-------|-----|
| App permission | Phone Settings → App → Microphone: ON |
| Camera audio enabled | Camera web UI → Audio → Input/Output: ON |
| Codec mismatch | G.711 (PCM) самый совместимый; попробуйте G.726 / AAC |
| Half-duplex vs Full | Большинство камер = PTT (half); не говорите пока слушаете |

### 8.2 Эхо / Обратная связь / Шум

| Причина | Исправление |
|-------|-----|
| Динамик слишком громко / mic gain высокий | Уменьшите громкость динамика в app; снизьте mic gain |
| Нет AEC (Acoustic Echo Cancellation) | Full-duplex камерам нужен AEC; half-duplex = mute mic while speaking |
| Ветер на микрофоне | Включите wind filter; пенная насадка; перепозиционируйте |

## 9. Прошивка и обновления

### 9.1 Безопасная процедура обновления прошивки

1. **Прочитайте changelog** — Пропустите если "только безопасность" и вы стабильны
2. **Бэкап конфига** — NVR: Export config; Камера: Скриншот настроек
3. **Обновите NVR сначала** — Потом камеры (NVR может запушить совместимую прошивку)
4. **По одной камере за раз** — Проверьте каждую перед следующей
5. **Не отключайте питание** — 5–10 мин на камеру
6. **Верифицируйте** — Live view, запись, motion, PTZ, аудио

### 9.2 Откат неудачной прошивки

| Бренд | Метод |
|-------|--------|
| **Reolink** | Страница прошивки → кнопка "Previous Version" |
| **Dahua/Hikvision** | TFTP recovery (boot + hold reset + TFTP server) |
| **UniFi** | Protect → Devices → Camera → "Rollback Firmware" |
| **Generic** | SD карта с `firmware.bin` в корне; power on с зажатым reset |

## 10. Окружение и физика

### 10.1 Водоход / Конденсат

| Симптом | Исправление |
|---------|-----|
| Туман внутри купола | Пакет силикагеля (меняйте ежегодно); проверьте вентиляционный пробку открытой (Gore-Tex) |
| Вода у входа кабеля | Силикон + прокладка; капающая петля; кондуитное тело |
| Коррозия на RJ45 | Диэлектрическая смазка на контактах; замените коннектор; экранированный кабель |

### 10.2 Насекомые / Пауки / Птицы

| Отпугиватель | Эффективность |
|-----------|---------------|
| Перечная мята (ватушка у камеры) | Высокая (пауки ненавидят) |
| Ультразвуковой отпугиватель | Средняя |
| Анти-перные шипы на кронштейне | Высокая (птицы) |
| Флисовый чехол над IR кольцом | Высокая (пауки строят на теплом IR) |
| Регулярная чистка (ежемесячно) | Необходима |

### 10.3 Вандализм / Кража

- **Torx / Security винты** (в комплекте с большинством про камер)
- **Кондуит** до камеры (нельзя легко перерезать кабель)
- **Высота** >10 ft (нужна лестница)
- **Резервная камера** смотрит на камеру (перекрытие)
- **Tamper Alert** (NVR: Camera → Tamper Detection → Alarm)

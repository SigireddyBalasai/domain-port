## त्वरित निदान फ़्लोचार्ट

```
कैमरा काम नहीं कर रहा?
    │
    ├─ कोई पावर LED नहीं? → पावर सप्लाई / PoE / बैटरी (सेक्शन 1)
    ├─ पावर ठीक, कोई इमेज नहीं? → केबल / लिंक / IP विरोध (सेक्शन 2)
    ├─ इमेज ठीक, कोई रिकॉर्डिंग नहीं? → NVR / स्टोरेज / शेड्यूल (सेक्शन 3)
    ├─ इमेज ठीक, झूठे अलर्ट? → मोशन ज़ोन / संवेदनशीलता / AI (सेक्शन 4)
    ├─ नाइट विज़न खराब? → IR / रिफ्लेक्शन / शटर (सेक्शन 5)
    └─ रिमोट एक्सेस विफल? → पोर्ट फ़ॉरवर्ड / VPN / वेंडर क्लाउड (सेक्शन 6)
```

## 1. पावर और स्टार्टअप समस्याएँ

### 1.1 कैमरा डेड — कोई LED, कोई प्रतिक्रिया नहीं

| चरण | जाँच                    | समाधान                                                                 |
| --- | ----------------------- | ---------------------------------------------------------------------- |
| 1   | **PoE स्विच पोर्ट LED** | बंद = स्विच पोर्ट डेड / बजट खत्म → ज्ञात-अच्छे पोर्ट पर ले जाएँ        |
| 2   | **PoE इंजेक्टर LED**    | बंद = इंजेक्टर विफल / कोई AC नहीं → इंजेक्टर बदलें ($25)               |
| 3   | **कैमरे पर वोल्टेज**    | मल्टीमीटर: 44–57V (PoE), 12V±10% (DC) → कम = वोल्टेज ड्रॉप / खराब केबल |
| 4   | **केबल कंटीन्युइटी**    | परीक्षक: सभी 8 तार → खुला जोड़ा = दोनों सिरों को फिर से टर्मिनेट करें  |
| 5   | **कैमरा रीसेट**         | रीसेट 10–30 सेकंड दबाएँ (मैनुअल देखें) → ऐप/NVR में फिर से अपनाएँ      |

**बैटरी कैमरे:** मृत घोषित करने से पहले USB-C के माध्यम से 24 घंटे चार्ज करें। गहरे डिस्चार्ज के बाद Li-ion BMS लॉक आउट हो जाता है।

### 1.2 कैमरा रात में रीबूट होता है (IR चालू)

**कारण:** जब IR LEDs चालू होती हैं तो वोल्टेज ड्रॉप (1–2W जोड़ता है)।

| निदान                             | समाधान                                               |
| --------------------------------- | ---------------------------------------------------- |
| PoE स्विच बजट सीमा पर             | कैमरे को इंजेक्टर पर ले जाएँ; स्विच अपग्रेड करें     |
| केबल रन >250 फीट Cat6 पर          | रन छोटा करें; Cat6a; कैमरे पर PoE++ इंजेक्टर         |
| CCA (कॉपर-क्लैड एल्युमिनियम) केबल | **सॉलिड कॉपर Cat6 से बदलें**                         |
| बैटरी कैम: Li-ion ठंड में सैग     | LiFePO₄ अपग्रेड; इंसुलेटेड बॉक्स; चार्ज करने घर लाएँ |

### 1.3 PoE स्विच ओवरहीटिंग / पोर्ट फ़्लैपिंग

- **PoE बजट जाँचें:** Σ कैमरा अधिकतम ड्रॉ ≤ स्विच बजट × 0.8
- **थर्मल:** स्विच बंद कैबिनेट में → पंखा जोड़ें या वेंटिलेटेड रैक में ले जाएँ
- **फ़र्मवेयर:** स्विच फ़र्मवेयर अपडेट करें (PoE नेगोशिएशन बग आम हैं)

## 2. नेटवर्क और कनेक्टिविटी

### 2.1 कैमरा ऐप/NVR में "ऑफलाइन"

| चरण | परीक्षण                            | समाधान                                      |
| --- | ---------------------------------- | ------------------------------------------- |
| 1   | PC से कैमरा IP पिंग करें           | टाइमआउट = लेयर 1/2 समस्या (केबल, स्विच, IP) |
| 2   | कैमरे से गेटवे पिंग करें (NVR शेल) | टाइमआउट = VLAN / फ़ायरवॉल / रूटिंग          |
| 3   | `arp -a` → MAC कैमरे से मेल?       | IP विरोध → स्टैटिक IP / DHCP रिज़र्वेशन     |
| 4   | DHCP लीज़ समाप्त                   | स्टैटिक IP या DHCP रिज़र्वेशन (राउटर)       |
| 5   | IP विरोध                           | दो उपकरण समान IP → अद्वितीय IP रिज़र्व करें |

### 2.2 वाई-फ़ाई कैमरा रोज़ डिस्कनेक्ट होता है

| निदान               | समाधान                                                   |
| ------------------- | -------------------------------------------------------- |
| RSSI < -75 dBm      | AP को पास लाएँ; वायर्ड AP जोड़ें; चैनल जाँचें            |
| 2.4 GHz कंजेशन      | केवल चैनल 1/6/11; 20 MHz चौड़ाई; बैंड-स्टीयरिंग बंद करें |
| DFS चैनल (5 GHz)    | 2.4 GHz पर फ़ोर्स करें; 5 GHz रडार = डिस्कनेक्ट          |
| पावर सेव मोड        | कैमरा ऐप में वाई-फ़ाई पावर सेव बंद करें                  |
| राउटर रीबूट शेड्यूल | ऑटो-रीबूट बंद करें; राउटर पर UPS                         |

### 2.3 RTSP स्ट्रीम नहीं खुलती (VLC / Frigate / NVR)

```
टेस्ट URL: rtsp://user:pass@192.168.1.100:554/stream1
```

| त्रुटि           | कारण               | समाधान                                                                                                                       |
| ---------------- | ------------------ | ---------------------------------------------------------------------------------------------------------------------------- |
| 401 अनधिकृत      | गलत उपयोगकर्ता/पास | NVR/ऐप जाँचें: अक्सर `admin` / `123456` या खाली                                                                              |
| 404 नहीं मिला    | गलत पथ             | ब्रांड पथ: Hikvision `/Streaming/Channels/101`, Dahua `/cam/realmonitor?channel=1&subtype=0`, Reolink `/h264Preview_01_main` |
| कनेक्शन अस्वीकृत | RTSP बंद           | कैमरा वेब UI → नेटवर्क → RTSP में सक्षम करें                                                                                 |
| टाइमआउट          | फ़ायरवॉल / VLAN    | NVR VLAN से कैमरा VLAN तक TCP 554 की अनुमति दें                                                                              |

**ब्रांड RTSP पथ (सामान्य):**

- **Reolink:** `/h264Preview_01_main` (मुख्य), `/h264Preview_01_sub` (उप)
- **Dahua/Amcrest:** `/cam/realmonitor?channel=1&subtype=0`
- **Hikvision:** `/Streaming/Channels/101` (मुख्य), `/Streaming/Channels/102` (उप)
- **UniFi:** `rtsp://user:pass@ip:7447/...` (Protect सेटिंग्स → RTSP)
- **Amcrest:** Dahua के समान (OEM)

## 3. रिकॉर्डिंग और स्टोरेज

### 3.1 NVR रिकॉर्ड नहीं कर रहा / टाइमलाइन में गैप

| जाँच                       | समाधान                                                          |
| -------------------------- | --------------------------------------------------------------- |
| HDD स्वास्थ्य (S.M.A.R.T.) | रियलोकेटेड सेक्टर >0 → **तुरंत बदलें**                          |
| HDD भरा / ओवरराइट बंद      | स्टोरेज सेटिंग्स में "सबसे पुराने को ओवरराइट" सक्षम करें        |
| शेड्यूल गलत कॉन्फ़िगर      | सत्यापित करें: मुख्य स्ट्रीम पर 24/7 निरंतर; उप पर मोशन         |
| स्ट्रीम बेमेल              | NVR मुख्य चाहता है; कैमरा उप भेज रहा है → एनकोड सेटिंग्स मिलाएँ |
| टाइमस्टैम्प ड्रिफ्ट        | NVR + कैमरों पर NTP सक्षम करें (समान सर्वर: `pool.ntp.org`)     |

### 3.2 microSD रिकॉर्ड नहीं कर रहा / दूषित

| लक्षण                 | कारण                | समाधान                                                                |
| --------------------- | ------------------- | --------------------------------------------------------------------- |
| "फ़ॉर्मेट आवश्यक" लूप | कार्ड विफल / गलत FS | **U3/V30 A2** से बदलें (SanDisk Extreme, Samsung Pro)                 |
| फ़ाइलें 0 बाइट / अचल  | लिखते समय पावर खत्म | ऐप में "सेफ इजेक्ट" सक्षम करें; प्लग-इन कैम पर UPS                    |
| 4 GB पर रुक जाता है   | FAT32 सीमा          | कैमरे को ऑटो-exFAT करना चाहिए; यदि नहीं, तो PC पर exFAT फ़ॉर्मेट करें |
| लूप काम नहीं कर रहा   | कार्ड भरा / लॉक     | लॉक स्विच जाँचें; पुरानी फ़ाइलें हटाएँ; कैमरे में फ़ॉर्मेट करें       |

### 3.3 प्लेबैक फ़्रीज़ / पिक्सेलेटेड / हरी स्क्रीन

| कारण                     | समाधान                                                           |
| ------------------------ | ---------------------------------------------------------------- |
| H.265/HEVC समर्थित नहीं  | प्लेयर को HEVC कोडेक चाहिए (Windows: $0.99 Store; VLC: बिल्ट-इन) |
| उप-स्ट्रीम चयनित         | प्लेबैक में मुख्य स्ट्रीम पर स्विच करें                          |
| कीफ्रेम इंटरवल बहुत अधिक | कीफ्रेम इंटरवल = FPS सेट करें (जैसे, 15 fps → 15)                |
| VBR बिटरेट स्पाइक्स      | CBR या VBR अधिकतम कैप सेट करें (4K के लिए 4–6 Mbps)              |

## 4. मोशन डिटेक्शन और अलर्ट

### 4.1 बहुत अधिक झूठे अलर्ट (स्पैम)

| सेटिंग                    | अनुशंसित                         | क्यों                                          |
| ------------------------- | -------------------------------- | ---------------------------------------------- |
| **डिटेक्शन प्रकार**       | केवल व्यक्ति + वाहन              | जानवरों, पेड़ों, छायाओं, बारिश को बाहर करता है |
| **मोशन ज़ोन**             | केवल लक्ष्य क्षेत्र ड्रा करें    | सड़क, पेड़, पड़ोसी, आसमान को बाहर करें         |
| **संवेदनशीलता**           | मध्यम (शुरू) → समायोजित करें     | अधिक = सब कुछ ट्रिगर                           |
| **रीट्रिगर अंतराल**       | 30–60 सेकंड                      | एक ही कार के 50 क्लिप रोकता है                 |
| **ऑब्जेक्ट आकार फ़िल्टर** | फ्रेम का न्यूनतम 5% / अधिकतम 50% | कीड़े, पक्षी, दूर की कारों को अनदेखा करता है   |
| **AI/डीप लर्निंग**        | सक्षम करें (व्यक्ति/वाहन/पैकेज)  | 90%+ झूठे अलर्ट कमी                            |

### 4.2 छूटी घटनाएँ (जब कुछ हुआ तो कोई अलर्ट नहीं)

| जाँच                              | समाधान                                                                  |
| --------------------------------- | ----------------------------------------------------------------------- |
| ज़ोन घटना क्षेत्र को कवर करता है? | ज़ोन को चौड़ा फिर से ड्रा करें; परीक्षण करके चलें                       |
| संवेदनशीलता बहुत कम?              | एक नॉच बढ़ाएँ; परीक्षण करें                                             |
| रात में "व्यक्ति" का पता नहीं चला | IR रिफ्लेक्शन AI को भ्रमित करता है → कैमरा नीचे झुकाएँ; बाहरी IR जोड़ें |
| कैमरा "स्लीप" मोड में             | बैटरी कैम: स्लीप बंद करें / PIR संवेदनशीलता बढ़ाएँ                      |
| NVR मोशन डिटेक्शन बंद             | NVR साइड पर सक्षम करें (कैमरा-साइड बंद हो सकता है)                      |

### 4.3 अलर्ट में देरी >10 सेकंड

| कारण                            | समाधान                                                    |
| ------------------------------- | --------------------------------------------------------- |
| क्लाउड रिले (Ring, Arlo, Blink) | अंतर्निहित — &lt;1s के लिए लोकल NVR/Frigate का उपयोग करें |
| कमज़ोर वाई-फ़ाई (RSSI < -75)    | सिग्नल सुधारें (सेक्शन 2.2)                               |
| NVR CPU अधिकतम                  | एनालिटिक्स चैनल कम करें; NVR अपग्रेड करें                 |
| MQTT/Webhook विलंबता            | पहले डायरेक्ट ऐप पुश का परीक्षण करें                      |

## 5. इमेज गुणवत्ता और नाइट विज़न

### 5.1 नाइट विज़न व्हाइटआउट / धुला हुआ

**कारण:** IR LEDs पास की सतह (दीवार, ईव्स, कांच, मकड़ी का जाला) से परावर्तित हो रही हैं।

| समाधान                       | कैसे                                                       |
| ---------------------------- | ---------------------------------------------------------- |
| **15–30° नीचे झुकाएँ**       | टिल्ट ब्रैकेट; कैमरा नीचे देखता है, दीवार पर नहीं          |
| **कैमरा बाहर ले जाएँ**       | दीवार/ईव्स से 12–18" दूर (एक्सटेंशन आर्म)                  |
| **बिल्ट-इन IR बंद करें**     | **बाहरी IR इल्युमिनेटर** जोड़ें (अलग, लक्ष्य पर केंद्रित)  |
| **डोम/लेंस साफ़ करें**       | मकड़ी के जाले = IR स्कैटर → माइक्रोफ़ाइबर + लेंस क्लीनर    |
| **स्मार्ट IR (यदि समर्थित)** | दूरी के अनुसार IR तीव्रता स्वचालित रूप से समायोजित करता है |

### 5.2 लाइसेंस प्लेटें रात में अपठनीय

| आवश्यकता        | स्पेक                                                        |
| --------------- | ------------------------------------------------------------ |
| **शटर स्पीड**   | न्यूनतम 1/100s (1/250s आदर्श) — एक्सपोज़र → शटर में सेट करें |
| **रिज़ॉल्यूशन** | 4K/8MP + ऑप्टिकल ज़ूम या 2MP LPR कैमरा                       |
| **IR रेंज**     | प्लेट दूरी ≤ कैमरा IR रेटिंग का 50%                          |
| **WDR**         | 120 dB+ (हेडलाइट्स को संभालता है)                            |
| **कोण**         | लंबवत से ≤30° (प्लेट सामने की ओर)                            |

**समाधान:** शटर बढ़ाएँ → इमेज गहरी लेकिन प्लेट साफ़। बहुत गहरा होने पर बाहरी IR जोड़ें।

### 5.3 दिन की इमेज: धुंधली / कम कंट्रास्ट / रंग कास्ट

| कारण                     | समाधान                                                    |
| ------------------------ | --------------------------------------------------------- |
| डोम गंदा / खरोंच         | माइक्रोफ़ाइबर साफ़ करें; डोम बदलें ($15)                  |
| IR कट फ़िल्टर अटका       | पावर साइकल; फ़र्मवेयर अपडेट; हल्के से टैप करें (मैकेनिकल) |
| व्हाइट बैलेंस गलत        | "आउटडोर" / "ऑटो" / कस्टम केल्विन (5500K) सेट करें         |
| बैकलाइट (फ्रेम में सूरज) | रिपोज़िशन; WDR सक्षम करें; सन शील्ड जोड़ें                |

### 5.4 फ़्लिकर / रोलिंग लाइन्स (50/60 Hz)

| कारण                             | समाधान                                                      |
| -------------------------------- | ----------------------------------------------------------- |
| **पावर लाइन फ़्रीक्वेंसी बेमेल** | कैमरा 60 Hz (US) / 50 Hz (EU) पर सेट — अपने ग्रिड से मिलाएँ |
| **ग्राउंड लूप**                  | सभी उपकरण एक ही ग्राउंड; PoE आइसोलेटर (Transtector)         |
| **फ़्लोरोसेंट/LED फ़्लिकर**      | शटर 1/100 (50 Hz) या 1/120 (60 Hz) पर सेट करें              |

## 6. रिमोट एक्सेस और मोबाइल ऐप

### 6.1 ऐप वाई-फ़ाई पर काम करता है, LTE/5G पर विफल

| आर्किटेक्चर                                   | समाधान                                                                 |
| --------------------------------------------- | ---------------------------------------------------------------------- |
| **वेंडर क्लाउड** (Reolink, UniFi, Eufy, Ring) | फ़ोन पर इंटरनेट जाँचें; वेंडर स्टेटस पेज                               |
| **पोर्ट फ़ॉरवर्ड**                            | **उपयोग न करें।** CGNAT, डबल NAT, ISP ब्लॉक। VPN/क्लाउड पर स्विच करें। |
| **VPN (Tailscale, WireGuard)**                | सुनिश्चित करें फ़ोन VPN सक्रिय; स्प्लिट-टनल ऐप को बाहर कर सकता है      |
| **Cloudflare Tunnel**                         | `cloudflared` सेवा स्वस्थ सत्यापित करें                                |

### 6.2 "कनेक्शन विफल" / रिमोट पर टाइमआउट

| परीक्षण                                 | अपेक्षित  | समाधान                               |
| --------------------------------------- | --------- | ------------------------------------ |
| फ़ोन से NVR LAN IP पिंग करें (VPN चालू) | &lt;50 ms | VPN रूटिंग समस्या                    |
| फ़ोन से `curl http://nvr:80`            | HTML      | पोर्ट ब्लॉक / फ़ायरवॉल               |
| वेंडर क्लाउड स्टेटस पेज                 | हरा       | प्रतीक्षा करें / लोकल VPN उपयोग करें |

### 6.3 लाइव व्यू काम करता है, प्लेबैक रिमोट से विफल

- प्लेबैक अलग पोर्ट/स्ट्रीम का उपयोग करता है (अक्सर HTTPS/443 बनाम RTSP/554)
- **जाँचें:** NVR → नेटवर्क → HTTPS पोर्ट (डिफ़ॉल्ट 443) VPN के माध्यम से सुलभ
- **बैंडविड्थ:** प्लेबैक को स्थायी 4–8 Mbps चाहिए; LTE थ्रॉटल कर सकता है

## 7. PTZ और मैकेनिकल

### 7.1 PTZ हिलता नहीं / अटका / घिसता हुआ

| कारण                          | समाधान                                                          |
| ----------------------------- | --------------------------------------------------------------- |
| मैकेनिकल लिमिट तक पहुँच गया   | प्रीसेट लिमिट जाँचें; होम पर रीसेट करें                         |
| स्लिप रिंग विफलता (360° अनंत) | केबल हार्नेस घिसा → कैमरा बदलें (मरम्मत योग्य नहीं)             |
| बर्फ / मलबा                   | हीटर विफल; मैन्युअल रूप से साफ़ करें; हीटर सर्किट सत्यापित करें |
| प्रोटोकॉल बेमेल               | Pelco-D/P बनाम ONVIF PTZ → NVR प्रोटोकॉल को कैमरे से मिलाएँ     |

### 7.2 प्रीसेट टूर नहीं चल रहा

- शेड्यूल सक्षम? (NVR → PTZ → टूर शेड्यूल)
- ड्वेल टाइम >0? (प्रति प्रीसेट 10–30s)
- होम प्रीसेट परिभाषित? (टूर लूप के बीच होम पर लौटता है)

| जाँच                   | समाधान                                        |
| ---------------------- | --------------------------------------------- |
| ऐप अनुमति              | फ़ोन सेटिंग्स → ऐप → माइक्रोफ़ोन: चालू        |

| हाफ-डुप्लेक्स बनाम फुल | अधिकांश कैम = PTT (हाफ); सुनते समय बात न करें |

| कारण                                 | समाधान                                                                    |

| कोई AEC (एकॉस्टिक इको कैंसलेशन) नहीं | फुल-डुप्लेक्स कैम को AEC चाहिए; हाफ-डुप्लेक्स = बोलते समय माइक म्यूट करें |
| माइक पर हवा                          | विंड फ़िल्टर सक्षम करें; फोम विंडस्क्रीन; रिपोज़िशन                       |

## 9. फ़र्मवेयर और अपडेट

### 9.1 सुरक्षित फ़र्मवेयर अपडेट प्रक्रिया

1. **चेंजलॉग पढ़ें** — यदि "केवल सुरक्षा" है और आप स्थिर हैं तो छोड़ें
2. **कॉन्फ़िग बैकअप** — NVR: कॉन्फ़िग निर्यात करें; कैमरा: सेटिंग्स का स्क्रीनशॉट
3. **पहले NVR अपडेट करें** — फिर कैमरे (NVR संगत फ़र्मवेयर पुश कर सकता है)
4. **एक बार में एक कैमरा** — अगले से पहले प्रत्येक को सत्यापित करें
5. **पावर बंद न करें** — प्रति कैमरा 5–10 मिनट
6. **सत्यापित करें** — लाइव व्यू, रिकॉर्डिंग, मोशन, PTZ, ऑडियो

### 9.2 विफल फ़र्मवेयर को रोलबैक करें

| ब्रांड              | विधि                                                               |
| ------------------- | ------------------------------------------------------------------ |
| **Reolink**         | फ़र्मवेयर पेज → "पिछला वर्शन" बटन                                  |
| **Dahua/Hikvision** | TFTP रिकवरी (बूट + रीसेट दबाएँ + TFTP सर्वर)                       |
| **UniFi**           | Protect → डिवाइस → कैमरा → "फ़र्मवेयर रोलबैक"                      |
| **जेनेरिक**         | रूट में `firmware.bin` के साथ SD कार्ड; रीसेट दबाकर पावर चालू करें |

Każdy system kamer ma problemy. Ten przewodnik obejmuje 25 najczęstszych problemów z diagnostyką krok po kroku i naprawami — pogrupowanych według objawów, aby szybko znaleźć rozwiązanie.

<Badge variant="outline">Metoda diagnostyczna</Badge> **Zacznij prosto:**
Zasilanie → Kabel → Sieć → Ustawienia → Sprzęt. 80% problemów to zasilanie lub
kabel.

---

## Szybki schemat diagnostyczny

```
Kamera nie działa?
    │
    ├─ Brak diody zasilania? → Zasilacz / PoE / Bateria (Sekcja 1)
    ├─ Zasilanie OK, brak obrazu? → Kabel / Połączenie / Konflikt IP (Sekcja 2)
    ├─ Obraz OK, brak nagrywania? → NVR / Pamięć / Harmonogram (Sekcja 3)
    ├─ Obraz OK, fałszywe alarmy? → Strefy ruchu / Czułość / AI (Sekcja 4)
    ├─ Słabe widzenie nocne? → IR / Odbicie / Migawka (Sekcja 5)
    └─ Zdolny dostęp zawodzi? → Przekierowanie portów / VPN / Chmura (Sekcja 6)
```

## 1. Zasilanie i uruchamianie

### 1.1 Kamera martwa — brak diody, brak reakcji

| Krok | Sprawdź                          | Naprawa                                                                      |
| ---- | -------------------------------- | ---------------------------------------------------------------------------- |
| 1    | **Dioda portu przełącznika PoE** | Wyłączona = martwy port / przekroczony budżet → Przenieś do sprawnego portu  |
| 2    | **Dioda injektora PoE**          | Wyłączona = uszkodzony injektor / brak zasilania → Wymień injektor ($25)     |
| 3    | **Napięcie przy kamerze**        | Multimetr: 44–57V (PoE), 12V±10% (DC) → Niskie = spadek napięcia / zły kabel |
| 4    | **Ciągłość kabla**               | Tester: wszystkie 8 żył → Otwarta para = zakończ oba końce                   |
| 5    | **Reset kamery**                 | Przytrzymaj reset 10–30s → Ponownie dodaj w aplikacji/NVR                    |

**Kamery bateryjne:** Ładuj 24h przez USB-C przed uznaniem za martwą. BMS Li-ion blokuje się po głębokim rozładowaniu.

### 1.2 Kamera restartuje się nocą (włączone IR)

**Przyczyna:** Spadek napięcia przy włączaniu diod IR (dodaje 1–2W).

| Diagnoza                                  | Naprawa                                                   |
| ----------------------------------------- | --------------------------------------------------------- |
| Przełącznik PoE na limicie budżetu        | Przenieś kamerę na injektor; wymień przełącznik           |
| Długość kabla >250 stóp na Cat6           | Skróć; Cat6a; injektor PoE++ przy kamerze                 |
| Kabel CCA (aluminium platerowane miedzią) | **Wymień na lity miedziany Cat6**                         |
| Kamera bateryjna: zimny spadek Li-ion     | Wymiana na LiFePO₄; ocieplana obudowa; wnieś do ładowania |

## 2. Sieć i łączność

### 2.1 Kamera "offline" w aplikacji/NVR

| Krok | Test                                | Naprawa                                           |
| ---- | ----------------------------------- | ------------------------------------------------- |
| 1    | Ping IP kamery z komputera          | Timeout = problem Warstwy 1/2 (kabel, switch, IP) |
| 2    | Ping bramy z kamery (powłoka NVR)   | Timeout = VLAN / firewall / routing               |
| 3    | `arp -a` → MAC zgadza się z kamerą? | Konflikt IP → Statyczne IP / rezerwacja DHCP      |
| 4    | DHCP lease wygasł                   | Statyczne IP lub rezerwacja DHCP (router)         |
| 5    | Konflikt IP                         | Dwa urządzenia to samo IP → Rezerwuj unikalne IP  |

### 2.2 Kamera Wi-Fi rozłącza się codziennie

| Diagnoza                     | Naprawa                                                    |
| ---------------------------- | ---------------------------------------------------------- |
| RSSI < -75 dBm               | Przybliż AP; dodaj przewodowy AP; sprawdź kanał            |
| Zatłoczenie 2.4 GHz          | Tylko kanał 1/6/11; szerokość 20 MHz; wyłącz band-steering |
| Kanał DFS (5 GHz)            | Wymuś 2.4 GHz; 5 GHz radar = rozłączenie                   |
| Tryb oszczędzania energii    | Wyłącz oszczędzanie Wi-Fi w aplikacji kamery               |
| Harmonogram restartu routera | Wyłącz auto-restart; UPS na routerze                       |

## 3. Nagrywanie i przechowywanie

### 3.1 NVR nie nagrywa / luki w osi czasu

| Sprawdź                        | Naprawa                                                             |
| ------------------------------ | ------------------------------------------------------------------- |
| Dysk HDD (S.M.A.R.T.)          | Przydzielone sektory >0 → **Wymień natychmiast**                    |
| Dysk pełny / nadpisywanie wył. | Włącz "Nadpisuj najstarsze" w ustawieniach pamięci                  |
| Nieprawidłowy harmonogram      | Sprawdź: Ciągły 24/7 na strumieniu głównym; Ruch na pomocniczym     |
| Niezgodność strumienia         | NVR oczekuje głównego; kamera wysyła pomocniczy → Dopasuj kodowanie |
| Dryf czasu                     | Włącz NTP na NVR + kamerach (ten sam serwer: `pool.ntp.org`)        |

## 4. Detekcja ruchu i alerty

### 4.1 Zbyt wiele fałszywych alarmów (spam)

| Ustawienie                        | Zalecane                    | Dlaczego                                   |
| --------------------------------- | --------------------------- | ------------------------------------------ |
| **Typ detekcji**                  | Tylko osoba + pojazd        | Wyklucza zwierzęta, drzewa, cienie, deszcz |
| **Strefy ruchu**                  | Rysuj TYLKO obszar docelowy | Wyklucz ulicę, drzewa, sąsiada, niebo      |
| **Czułość**                       | Średnia (start) → dostosuj  | Wysoka = wszystko wyzwala                  |
| **Interwał ponownego wyzwalania** | 30–60 sekund                | Zapobiega 50 klipom tego samego samochodu  |
| **Filtr rozmiaru obiektu**        | Min 5% / Maks 50% kadru     | Ignoruje owady, ptaki, dalekie samochody   |
| **AI/Deep Learning**              | Włącz (Osoba/Pojazd/Paczka) | Redukcja fałszywych alarmów o 90%+         |

## 5. Jakość obrazu i widzenie nocne

### 5.1 Widzenie nocne białe / wypłukane

**Przyczyna:** Diody IR odbijają się od pobliskiej powierzchni (ściana, okap, szkło, pajęczyna).

| Naprawa                          | Jak                                                               |
| -------------------------------- | ----------------------------------------------------------------- |
| **Nachyl w dół 15–30°**          | Pochyl wspornik; kamera patrzy w dół, nie na ścianę               |
| **Odsuń kamerę**                 | 12–18" od ściany/okapu (ramię przedłużające)                      |
| **Wyłącz wbudowane IR**          | Dodaj **zewnętrzny iluminator IR** (oddzielny, skierowany na cel) |
| **Wyczyść kopułę/obiektyw**      | Pajęczyny = rozpraszanie IR → mikrofibra + środek do czyszczenia  |
| **Smart IR (jeśli obsługiwany)** | Automatycznie dostosowuje intensywność IR do odległości           |

## 6. Zdalny dostęp i aplikacja mobilna

### 6.1 Aplikacja działa na Wi-Fi, nie działa na LTE/5G

| Architektura                                       | Naprawa                                                                  |
| -------------------------------------------------- | ------------------------------------------------------------------------ |
| **Chmura producenta** (Reolink, UniFi, Eufy, Ring) | Sprawdź internet w telefonie; stronę statusu producenta                  |
| **Przekierowanie portów**                          | **Nie używaj.** CGNAT, podwójny NAT, ISP blokuje. Przejdź na VPN/Chmurę. |
| **VPN (Tailscale, WireGuard)**                     | Upewnij się, że VPN aktywny; split-tunnel może wykluczać aplikację       |

## 7. PTZ i mechanika

### 7.1 PTZ nie rusza / zaciął się / zgrzyta

| Przyczyna                     | Naprawa                                                    |
| ----------------------------- | ---------------------------------------------------------- |
| Osiągnięto limit mechaniczny  | Sprawdź limity presetów; zresetuj do pozycji domowej       |
| Awaria pierścienia ślizgowego | Zużyta wiązka kablowa → wymień kamerę (nie naprawialne)    |
| Lód / zanieczyszczenia        | Uszkodzona grzałka; wyczyść ręcznie; sprawdź obwód grzałki |
| Niezgodność protokołu         | Pelco-D/P vs ONVIF PTZ → Dopasuj protokół NVR do kamery    |

## 8. Dźwięk (dwukierunkowy)

### 8.1 Brak dźwięku / tylko jednostronny

| Sprawdź                | Naprawa                                                       |
| ---------------------- | ------------------------------------------------------------- |
| Uprawnienia aplikacji  | Ustawienia telefonu → Aplikacja → Mikrofon: WŁ.               |
| Dźwięk kamery włączony | Web UI kamery → Audio → Wejście/Wyjście: WŁ.                  |
| Niezgodność kodeka     | G.711 (PCM) najbardziej kompatybilny; spróbuj G.726 / AAC     |
| Półdupleks vs pełny    | Większość kamer = PTT (półdupleks); nie mów podczas słuchania |

## 9. Firmware i aktualizacje

### 9.1 Bezpieczna procedura aktualizacji firmware

1. **Przeczytaj changelog** — Pomiń jeśli "tylko bezpieczeństwo" i jesteś stabilny
2. **Utwórz kopię konfiguracji** — NVR: Eksport konfiguracji; Kamera: Zrzut ekranu ustawień
3. **Najpierw zaktualizuj NVR** — Potem kamery (NVR może wypchnąć kompatybilny firmware)
4. **Jedna kamera na raz** — Sprawdź każdą przed następną
5. **Nie wyłączaj zasilania** — 5–10 min na kamerę
6. **Sprawdź** — Podgląd na żywo, nagrywanie, ruch, PTZ, dźwięk

## 10. Środowisko i fizyczne

### 10.1 Woda / kondensacja

| Objaw                   | Naprawa                                                                    |
| ----------------------- | -------------------------------------------------------------------------- |
| Mgła wewnątrz kopuły    | Saszetka osuszacza (wymień rocznie); sprawdź otwór wentylacyjny (Gore-Tex) |
| Woda przy wejściu kabla | Silikon + dławica; pętla kroplowa; przepust                                |

### 10.2 Owady / pająki / ptaki

| Środek odstraszający                | Skuteczność                |
| ----------------------------------- | -------------------------- |
| Olejek miętowy (wacik przy kamerze) | Wysoka (pająki nienawidzą) |
| Kolce na ptaki na wsporniku         | Wysoka (ptaki)             |

### 10.3 Wandale / kradzież

- **Śruby Torx / zabezpieczające** (dołączone do większości profesjonalnych kamer)
- **Przepust** do kamery (trudno przeciąć kabel)
- **Wysokość** >10 stóp (wymaga drabiny)
- **Kamera zapasowa** obserwująca kamerę (nakładanie)
- **Alert naruszenia** (NVR: Kamera → Detekcja naruszenia → Alarm)

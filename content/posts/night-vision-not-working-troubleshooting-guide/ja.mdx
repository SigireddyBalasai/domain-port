## 1. IR LEDが点灯しない（赤い光がない）

### 1.1 IR状態の確認

<Accordion type="single" collapsible>
  <AccordionItem value="check-ir">
    <AccordionTrigger>IR LEDの確認方法</AccordionTrigger>
    <AccordionContent>

      **目視:** 夜間にカメラの前に立つ → **かすかな赤い光**（850 nm）または**目に見えない光**（940 nm「隠密」）を確認。**スマートフォンカメラテスト:** スマホのカメラをカメラに向ける → IRが画面上で**明るい紫/白**に表示（スマホセンサーは近赤外線を認識）。**アプリステータス:** カメラのWeb UI / NVR → 「IRステータス」/「ナイトモード」→ 「オン」または「オート」と表示されるはず。

    </AccordionContent>

  </AccordionItem>
</Accordion>

### 1.2 原因と修正

| 原因                                     | 診断                                                 | 修正                                                                    |
| ---------------------------------------- | ---------------------------------------------------- | ----------------------------------------------------------------------- |
| **IRカットフィルター固着（デイモード）** | 夜間に画像がカラー、IR光なし                         | 電源再投入（12V/PoEを30秒オフ）。カメラを軽く叩く。ファームウェア更新。 |
| **フォトレジスタ/LDR故障**               | IRが自動で有効にならない; 手動「オン」は動作         | LDR（小さな透明窓）を覆う → IRが作動すれば、LDR（$2）またはカメラを交換 |
| **IR LEDドライバー基板不良**             | 12V/PoEは供給中、LEDが冷たい、手動「オン」で動作せず | カメラを交換（基板は修理不可）                                          |
| **IRに電源が不十分**                     | 昼間はOK、夜間に再起動/IRが点滅                      | IR負荷時のカメラ電圧を確認（§1.3参照）                                  |
| **ファームウェアバグ**                   | 最近のアップデートで自動IRが壊れた                   | ファームウェアをロールバック; ベンダーサポートチケット                  |
| **隠密IR (940 nm)**                      | 赤い光なし、スマホテストでは紫表示                   | 正常 — 940 nmは目に見えない                                             |

### 1.3 IR負荷時の電圧降下（重要）

**IR LEDは追加で2–5Wを消費します。** 昼間は動作するカメラが夜になるとクラッシュします。

<Callout type="warning">

**テスト:** カメラ電源入力でのマルチメータ（PoE: PoEテスターを使用）。昼間の電圧 vs 夜間（IRオン）の電圧。**0.5V以上（12V）または2V以上（PoE 48V）の低下 = ケーブルが長すぎる / ゲージが細すぎる / スイッチの余裕を超えている。**

</Callout>

| 構成                       | 最大IR負荷                       | 修正                                               |
| -------------------------- | -------------------------------- | -------------------------------------------------- |
| **PoE (802.3af 15.4W)**    | ~4W IR                           | PoE+ (30W) インジェクター/スイッチにアップグレード |
| **12V DC (18 AWG, 50 ft)** | 2A IR                            | 配線を短く / 14 AWG / ローカル12V電源              |
| **バッテリーカメラ**       | IRによりバッテリーが数時間で消耗 | IRを無効化; 周囲光 / 外部IR照明器を使用            |

## 2. IR白飛び / 白っぽい / 灰色の画像

**症状:** 夜間画像 = 明るい白/灰色の塊、詳細なし。IRが近くの表面で反射。

<Accordion type="single" collapsible>
  <AccordionItem value="whiteout-causes">
    <AccordionTrigger>一般的な反射源</AccordionTrigger>
    <AccordionContent>

| 表面 | 距離 | 修正 | |---------|----------|-----| | **壁 / サイディング / 軒先** | 6–24" | カメラを**15–30°下向き**に; 12–18" 離す（延長アーム） | | **軒天井** | 0–12" | 軒天井に**面して**ではなく、**下に**取り付ける | | **窓 / ガラス** | 任意 | **絶対に**ガラス越しに狙わない（IRは100%反射） | | **クモの巣 / ドームのほこり** | レンズ上 | 月1回清掃; IRリングにフリースカバー | | **雨 / 雪 / 霧** | 空中 | 外部IR照明器（レンズから離して設置） | | **白いトリム / 雨どい** | 12–36" | フラットブラック（BBQ用塗料）で塗装またはテープでマスキング | | **ナンバープレート（再帰反射）** | 20–50 ft | 下向き角度; 外部IR; 偏光フィルター付きLPRカメラ |

    </AccordionContent>

  </AccordionItem>
</Accordion>

### 2.1 白飛び修正ステップバイステップ

1. **反射体を特定:** 夜間に外に出て、表面でのIRの光を確認。スマホカメラでIRの広がりを確認。
2. **角度を調整:** 水平から**15–30°下向き**に傾ける。水平器＋分度器を使用。
3. **距離を確保:** 延長ブラケット（12–24"）でIRコーンを近くの物体の先に移動。
4. **外部IR照明器:** 10–20 ft 離して設置、対象ゾーンに向ける。カメラIRは**オフ**。ナンバープレート、ゲートに最適。
5. **スマートIR（対応の場合）:** カメラが近くの物体に対してIR強度を自動低減。設定で有効化。
6. **IRカットフィルター確認:** 「アウト」（ナイトモード）で固着＝昼間のようなカラー画像。「イン」（デイモード）で固着＝IRなし。電源再投入。

## 3. 黒い / ノイズの多い / ざらついた夜間画像

**症状:** 暗く、斑点状の「雪のような」映像。詳細なし。

| 原因                                         | 診断                         | 修正                                                            |
| -------------------------------------------- | ---------------------------- | --------------------------------------------------------------- |
| **IRが距離に対して弱すぎる**                 | 対象が定格IR範囲の50%以上    | 外部IR照明器; カメラをより近くに移動                            |
| **シャッターが遅すぎる（モーションブラー）** | ゴーストの軌跡、明るい縞模様 | **最小シャッター: 1/100s** に設定（ナンバープレートは1/250s）   |
| **ゲイン/ISOが最大になっている**             | ノイズ＝「雪」               | 光を追加（外部IR、街灯）; 最大ゲインを下げる                    |
| **WDR（ワイドダイナミックレンジ）がオフ**    | 明るい光がカメラを盲目に     | **WDR: 120 dB+** を有効化（DWDRではなくTrue WDR）               |
| **ドームが汚れている / 傷ついている**        | 光の周りにハロー             | ドームを交換（$15）; マイクロファイバー＋レンズクリーナーで清掃 |
| **センサーの経年劣化**                       | 年々進行性                   | カメラを交換（CMOSは経年劣化、ホットピクセル増加）              |
| **圧縮率が高すぎる**                         | 夜間にブロックノイズ         | ビットレート: CBR 4–6 Mbps（4K）/ 2–3 Mbps（2K）                |

### 3.1 シャッター速度早見表

| 対象                 | 最小シャッター      | 備考                    |
| -------------------- | ------------------- | ----------------------- |
| 一般エリア           | 1/50s（1/60s US）   | 明るさとブレのバランス  |
| 歩行者               | 1/100s              | 手足が鮮明に            |
| **ナンバープレート** | **1/250s – 1/500s** | **重要** — 手動設定必須 |
| 高速車両             | 1/500s – 1/1000s    | 外部IRが必要            |

<Callout type="note">

**設定方法:** カメラWeb UI → 画像 → 露出 → シャッター速度 → 手動 →値を入力。「オート」シャッターを無効化。

</Callout>

## 4. カラーナイトビジョンの不具合

**カラーナイトビジョン（CNV）** = 大型センサー（1/1.8"、1/1.2"）+ 低照度ISP + **周囲光必須**。

| 故障モード                | 原因                      | 修正                                                                                                          |
| ------------------------- | ------------------------- | ------------------------------------------------------------------------------------------------------------- |
| **白黒のまま（IRオン）**  | 周囲光 &lt;1 lux          | CNVには~3–5 lux必要（街灯、玄関灯）。10–20 ft 離して**暖色（3000K）LED** を追加。                             |
| **カラーノイズ/ざらつき** | ゲインが高すぎる          | 最大ゲインを下げる; 光を追加; 白黒IRで妥協                                                                    |
| **色ずれ（紫/緑）**       | ホワイトバランスが混乱    | WBを「屋外」/「街灯」/ カスタム3000Kに設定                                                                    |
| **IR + CNVの競合**        | 両方オン＝奇妙な色        | CNV有効時にIRを無効化（良いカメラでは自動）                                                                   |
| **非対応**                | カメラにCNVセンサーがない | 仕様確認: 1/1.8"+ センサーが必要（Reolink ColorX、Hikvision ColorVu、Dahua Full-Color、Axis Lightfinder 2.0） |

## 5. ナンバープレート撮影 — 夜間特有の修正

**ナンバープレート撮影が夜間に失敗する90%の原因:** プレートからのIR反射＋低速シャッター＋IR不足。

### 5.1 ナンバープレート撮影 夜間チェックリスト

- [ ] **カメラ:** 4K/8MP＋光学ズーム（バリフォーカル 2.8–12mm または 5–50mm）または専用LPR（2MP、100+ fps、グローバルシャッター）
- [ ] **シャッター:** **最低1/250s**（手動）
- [ ] **IR:** 外部照明器（850 nm、30–50°ビーム）を**カメラから離して**、プレートゾーンに向ける
- [ ] **角度:** 垂直から≤30°; 高さ3–4 ft（プレートレベル）
- [ ] **WDR:** 120 dB+ True WDR（ヘッドライト対応）
- [ ] **IRカット:** 切替確認（昼間カラー / 夜間白黒）
- [ ] **ゾーン:** モーションゾーン＝プレート経路のみ（誤検知を減らす）
- [ ] **録画:** CBR 6–8 Mbps（4K）/ 4 Mbps（2MP LPR）

| ---------------------------------- | ------------------- | ---------------------------------------------------- | ------------------------------ |

| **Dahua ITC237-PW6M-IRLZF**        | 2MP LPR             | 60 fps、140 dB WDR、164 ft IR                        | 駐車場、ゲート入口             |

## 6. ドアベルカメラの夜間問題

| 問題                         | 原因                    | 修正                                                              |
| ---------------------------- | ----------------------- | ----------------------------------------------------------------- |
| **訪問者の顔が白飛び**       | IRがポーチ/ひさしで反射 | ウェッジキット（15–25°下向き）で角度調整; IRを無効化→玄関灯を使用 |
| **夜間暗視不能**             | バッテリー節約モード    | 有線接続（16–24V AC）→ 常時IRが有効に                             |
| **カラーナイトビジョンオフ** | 周囲光が低すぎる        | ポーチ照明に**3000K LED**を追加（クールホワイトは不可）           |
| **夜間に動体検知しない**     | PIR範囲 < IR範囲        | PIR ~25 ft、IR ~30 ft → IR感度を下げる / ゾーンを狭くする         |

## 7. バッテリー/ソーラーカメラ夜間の注意点

| 問題                                  | 原因                              | 修正                                                                 |
| ------------------------------------- | --------------------------------- | -------------------------------------------------------------------- |
| **IRによりバッテリーが1週間で切れる** | IR = 2–4W 連続                    | IRを無効化; 周囲光＋モーションスポットライトに頼る（Eufy、Ring）     |
| **冬に夜間暗視不能**                  | リチウムイオン &lt;32°Fで充電停止 | LiFePO₄ アップグレード（Reolink Go PT Plus）; 断熱ボックス＋ヒーター |
| **動体検知が昼間のみ動作**            | PIRが窓/ガラスでブロック          | ガラスの**外側**に取付; PIRは温度差が必要                            |
| **スポットライトが点灯しない**        | モーションゾーンが間違っている    | ゾーンをアプローチ経路をカバーするよう再設定; テスト歩行             |

### 8.1 暗視テストカード（印刷してラミネート）

│  ░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░   │  ← 50% グレー

│  プレート: ABC-1234（20/40/60ftでテスト）│
│  顔: [輪郭]                    │
└─────────────────────────────────────┘
```

**手順:** 対象距離に設置 → 録画 → プレート/顔の明瞭度を確認。

### 8.2 IR電力測定

| ツール                  | 測定内容                | 合格基準                            |
| ----------------------- | ----------------------- | ----------------------------------- |
| **PoEテスター**         | カメラ部の電圧（夜間）  | >48V（PoE+）、>52V（PoE++）         |
| **マルチメータ（12V）** | カメラ部の電圧（夜間）  | >11.5V（12V 定格）                  |
| **クランプメータ**      | 12Vラインの電流（夜間） | &lt;2A（カメラあたり）              |
| **サーマルカメラ**      | IR LED温度              | 40–60°C（動作中）、&lt;30°C（故障） |

### 8.3 ファームウェアと設定監査

```
☐ ファームウェア: 最新安定版（ベータ版不可）
☐ IRモード: オート（24時間「オン」不可）
☐ スマートIR: 有効
☐ シャッター: 手動 最低1/100s（ナンバープレートは1/250s）
☐ WDR: True WDR オン（120 dB+）
☐ ゲイン最大: 制限あり（「オート最大」不可）
☐ ビットレート: CBR 4–6 Mbps（4K）
☐ IRカット: 切替確認済み（昼/夜）
☐ タイムゾーン/NTP: 正しい（スケジュールIR用）

## 9. 交換 vs 修理の判断

| 症状                                   | 修理可能？                     | 交換の目安                                         |
| -------------------------------------- | ------------------------------ | -------------------------------------------------- |
| IR光なし、電源OK                       | IR基板（$50）/ LDR（$2）       | カメラ3年以上、水の侵入                            |
| IRカットフィルター固着                 | タップ/電源再投入（一時的）    | メカニズム摩耗; 2年以上                            |
| 角度調整で白飛びが改善しない           | N/A                            | 場所に合っていないカメラ（外部IRを使用）           |
| ノイズの多いセンサー（ホットピクセル） | ピクセルマッピング（一部対応） | 5%以上の死滅ピクセル; 4年以上                      |
| カラーナイトビジョン故障               | センサー/ISP問題               | センサーが小さすぎる（&lt;1/2.8"）; アップグレード |
| 断続的な夜間再起動                     | 電圧降下                       | ケーブル/電源を修正; 持続する場合は交換            |
